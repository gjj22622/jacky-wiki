## 閱讀後的處理流程

```
讀完文章／書／影片
   ↓
[影響我做事的方式？]
   ├ 是 → 寫進對應 ailab 頁面（concepts / tools / patterns），並在本索引備註
   └ 否 → 只列在本索引（不寫個人心得）
```

> 不是每篇都值得寫心得。**改變行為才寫**——這是門檻。

---

## 心得格式（如果要寫）

當某篇真的改變做事方式，獨立成 page：

```
ailab/reading/<日期>-<主題簡稱>.md
```

YAML：
```
---
title: <一句話>
domain: ailab
source: <出處 URL / 書名 / 作者>
read_date: <YYYY-MM-DD>
type: 部落格 | 論文 | 書 | 影片 | 文檔
impact: 改變了 <X 工具選擇 / Y 工作方式>
updated: <YYYY-MM-DD>
---
```

內容：
- 一段：作者主張什麼
- 一段：我為什麼覺得對（或哪裡部分對）
- 一段：對我的具體改變
