## 核心信條

把一個目標，用 **SOSTAC 當引擎**做成一個會自己跑的迴圈：每個流程是一個 Agent，互相合作、互相監控，一圈一圈繞著目標跑，達標才停。正著跑＝建迴圈，逆著查＝除錯。**永遠圍繞目標 O，不越跑越遠。**

SOSTAC 六格 → 六個 Agent/步驟：
- **S 現況**：現況收集 Agent（抓起點事實、上一輪交付）
- **O 目標**：人給；可檢查；迴圈的圓心
- **S 策略**：拆解任務、擬定策略 Agent（plan）
- **T 工具**：執行工具與環境確認
- **A 行動**：執行，產一版
- **C 控制**：交付＋對 O 打分；**C 成為下一輪的 S**

## `/迴圈 拆 <目標>`

1. 若目標太空泛（「提升知名度」「做得更好」），**先逆向工程**：請使用者給一個成功範例，倒推 3–5 個成功特徵，把 O 改寫成可檢查的具體版（見「逆向工程定 O」）。
2. 產出 SOSTAC 六格拆解表：

| 格 | 這格在本任務是什麼 | 人/Agent | 輸入 | 輸出（＝下一格輸入）|
|---|---|---|---|---|
| S 現況 | … | Agent | … | … |
| O 目標 | （可檢查的具體目標）| 人 | — | — |
| S 策略 | … | Agent | 現況 | 計畫 |
| T 工具 | … | Agent＋人 | 計畫 | 工具就緒清單 |
| A 行動 | … | Agent | 工具+計畫 | 一版產出 |
| C 控制 | 交付＋對 O 打分 | Agent＋人 | 產出 | **下一輪的現況** |

3. 鐵律：**一格一個決策**（判斷規則 >10 條就拆子 Agent）；**上一格輸出格式＝下一格輸入格式**。

## `/迴圈 設計`

產一張 **loop 藍圖**，含四件事：
1. **骨架**（三選一，每格內部協作可混用 L5 三協作）：
   - 接力迴圈：A→B→C 串行＋退件重做（固定流程）
   - 主管迴圈：主管拆派→工人做→主管收驗→沒過再派（orchestrator-workers）
   - 出稿-審稿迴圈：產生→評分→改→再評（evaluator-optimizer，**最常用**）
2. **終止條件**（至少設兩個）：達標（評審全過）／上限 N 輪／沒進展（產出不再變）／成本預算／求救人類。
3. **監控**：一個**評審 Agent**，每輪拿 C 對 O 打分（寫 3 條可檢查標準）；**護欄**兩條（絕對不做的、碰到就停找人的）。
4. **鐵律**：Agent 不能改自己的 System Prompt；改規則要人點頭＋留版本。

## `/迴圈 跑 <藍圖>`

用 Claude Code subagent 編排執行（可用 Workflow 工具或逐一 Agent 呼叫）：
1. 跑一輪 S→S→T→A，產出一版。
2. **評審 Agent** 拿產出對 O 打分（3 條標準），輸出：分數＋達標與否＋差在哪。
3. 達標或命中終止條件 → **停，交付**。否則把回饋當新現況，**再跑一輪**（不超過上限）。
4. 連續沒過 → 進**逆推除錯**（見下），找出病根那一格再跑。
5. 回報：跑了幾輪、每輪評審、最終產出、若有逆推附紀錄。

## 逆推除錯（跑不到目標時）

**倒著沿 SOSTAC、從便宜改起、不行才往根本爬**，每層約試 3 次再上爬：

```
A 行動錯？ →(3次↑)→ T 工具/環境錯？ →(3次↑)→ S 策略錯？ →(3次↑)→ O 目標太大太空泛？ → S 現況根本錯(AI 幻覺)
```
- 兩個終極病根：**O 訂壞** 或 **現況 S 是幻覺（起點就假）**。
- O 太空泛 → 回去用**逆向工程**重新定 O。
- 每次失敗都拉回「跟目標的距離」逐層查 → 迴圈永遠繞著 O，不越跑越遠。

---

## 逆向工程定 O（O 太空泛時必做）

拿一個**成功的成品**倒推它為什麼成功 → 把「好」變成可列舉的特徵 → 寫成具體 O。
六原則 ≈ 跑一輪 SOSTAC：目標導向=O／系統拆解=S策略／觀察收集=S現況／假設驗證=評審／模式識別=經驗存檔／重建理解=建 Agent。

## 紅線

- ❌ 不幫使用者把 Agent 自己改自己的 prompt（人工 gate）。
- ❌ 不設無上限迴圈：一定有終止條件。
- ❌ O 空泛就先逼具體（逆向工程），不帶著空目標硬跑。
- ❌ 跑不到目標時不原地重跑：一定逆推往上爬。
- ✅ 每輪都拿 C 對 O 打分，永遠繞著目標。
