# AGENTS® 認證考核手冊

> **版本**：v1.0（2026-04-06）｜**用途**：講師／考核官的操作手冊，每位學員一份考核表
> 來源：`11_AGENTS_知識體系/02_AGENTS認證考核手冊.md`

> 方法論本身（六字訣／三級認證概念）見 [`AGENTS知識體系.md`](AGENTS知識體系.md)。

---

## 1. 考核流程總覽

```
學員完成課程 → 繳交作業 → AI 自動評分 → 同儕互評 → 實戰驗證 → 講師面審 → 頒發認證
```

**考核週期**

| Level | 週期 |
|---|---|
| L1 | 約 1 週 |
| L2 | 約 2 週 |
| L3 | 約 4 週 |

> 週期含實戰驗證時間——學員必須在真實工作中使用 Agent 並提供紀錄。

| Q | 題目 |

### 2.5 同儕認可（15 分）

至少 2 位同儕填寫，每位 3 題（每題 1-5 分）：

1. 這位學員建的 Agent，我看過或用過
2. 這個 Agent 對團隊的工作確實有幫助
3. 我願意在自己的工作流程中使用這個 Agent

**得分公式**：(同儕 A 三項平均 × 3 + 同儕 B 三項平均 × 3) / 2

**小計：_____ / 15**

## 5. 認證有效期與續證

| 等級 | 有效期 | 續證條件 |
|---|:---:|---|
| L1 | 1 年 | 過去一年至少 5 次 Agent 使用紀錄 |
| L2 | 1 年 | 過去一年至少 1 條運作中的工作流 + 進化紀錄 |
| L3 | 2 年 | 過去兩年的營運數據 + 至少 1 次系統升級紀錄 |

到期前 30 天自動提醒，到期後 90 天內可補件續證，超過 90 天需重新考核。

## 6. 設計重點摘要

### 為什麼 L3 的「實戰」佔比最高（35%）

到工廠經理層級，「**做出來別人真的能用**」比「自己懂理論」更重要。L1 的知識考核佔 25%，L3 只佔 15%——對應這個原則。

### 為什麼「Agent 不能自行修改自己的 System Prompt」是鐵律

進化機制可以**產出建議**，但變更必須由 Jacky（或指定管理者）確認。這是 Agent 工廠的安全紅線。

### 為什麼有「同儕認可」維度

防止「自己很懂但同事用不了」的盲點。L1 同儕佔 15%，L3 提高到 25%——位階越高，影響越多人，同儕意見越關鍵。
