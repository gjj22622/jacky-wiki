# 入口導覽頁 SOP

> Jacky 的固定產出方式：每個專案完成後，**根目錄產一份 `index.html`**，把分散的資料夾／檔案整合成一頁式入口。

---

## 一句話定位

**入口導覽頁不是文件，是 AGENTS 流程的標準收尾**——當多個 Agent / 多個資料夾完成各自的產出後，需要一個視覺整合層，給協作者、客戶、未來的自己一個切入點。

> 「資料夾結構是給檔案系統看的，入口導覽頁是給人看的。」

---

## 為什麼每個專案都要有

| 場景 | 沒有入口頁 | 有入口頁 |
|---|---|---|
| 兩週後自己回看 | 點進資料夾再點進子資料夾，找不到主成品 | 一鍵跳到核心產出 |
| 給協作者／指導老師 | 寄一個 zip，對方不知從何開起 | 寄一個 HTML，對方依進度總覽循線追蹤 |
| 對外發布 | 必須再做一次「對外簡介」 | 入口頁本身就是簡介 |
| 工作流可回溯性 | 各階段成品散落，難證明流程完整 | 進度總覽即流程證明 |

---

## 標準章節（從 6 個實例萃取）

```
專案根目錄/
├── index.html              # 入口導覽頁（必有）
├── share_cards/            # 社群擴散圖卡（選配，FB / IG 用）
├── A_xxx/  B_xxx/  C_xxx/  # 階段資料夾
└── ...原始檔案
```

`index.html` 的標準章節：

| 章節 | 內容 | 範例 |
|---|---|---|
| **主標題** | 中文標題 + 英文副標 | 「森林數位孿生作為下一世代地上生物量與碳儲量估算技術 / Forest Digital Twins as a Next-Generation Technology...」 |
| **進度總覽** | 視覺化進度條／時程／里程碑 | D-60 / D-25 / D-7 / D-day 倒推時程 |
| **各階段資料夾** | 卡片式列出每個階段資料夾 + 一句話用途 + 直接連結 | A_資訊收集 / B_專討準備 / C_投稿準備 |
| **核心產出集** | 主成品白名單（不是全部檔案，是篩選後的核心 X 篇／X 個） | 14 篇核心文獻 / 8 個 Skill / 5 大表單 |
| **快速存取** | 最常用的 3-5 個檔案捷徑（講稿、簡報、總表） | 講稿 V3 / 簡報 V3 / 文獻總表 |
| **根目錄路徑** | 顯示本機絕對路徑（方便切到檔案系統） | `C:\...\專題討論\114下學期專討-博二` |

## 設計範式（六實例共通點）

### 兩欄布局（內容專案）
左欄主內容 70% + 右欄 sticky 導覽 30%。常見於需要「邊讀邊跳引用」的場景（→ 見 [來源控制與 script_viewer 模式](來源控制與script_viewer模式.md)）。

### 卡片式階段（流程專案）
每個階段做成一張卡片，顯示完成度、產出檔案數、進入連結。博二專討、双云教育訓練都採此模式。

### 互動式架構圖（方法論專案）
六字訣／九區塊／三色測試這類**結構性方法論**用可摺疊互動卡呈現。雙云 AGENTS 知識體系架構圖.html 是代表作。

### Share Cards（對外擴散）
`share_cards/` 子資料夾放 FB / IG 用的方形圖卡，從入口頁的「核心產出」反推設計。

## 何時不該做

- **單檔成品**：一份簡報、一份論文不需要入口頁，原檔本身就是入口
- **進行中的暫存**：未收束的工作資料夾不必先做入口頁，等成品穩定再做
- **對外受眾不需要**：純內部 ephemeral 任務狀態追蹤（用 todolist.md 即可）

---

## 與 AGENTS 流程的關係

| AGENTS 階段 | 入口頁的角色 |
|---|---|
| A 拆解 | 拆解表本身可內嵌進入口頁的「進度總覽」 |
| G 生成 | 各 Agent 產出 → 入口頁的「各階段資料夾」 |
| E 評測 | 殘酷測試報告連結放入口頁的「快速存取」 |
| **N 串聯** | **入口頁本身就是 N 階段的視覺化收束**——把多 Agent 產出統合 |
| T 追蹤 | 進度總覽 = T 階段的儀表板原型 |
| S 規模 | 對外版（如通用版 index.html）= S 階段對客戶的接口 |

> **入口頁是 N 階段「Agent 工作流設計圖」的人類版本**——設計圖給工程師看，入口頁給協作者與未來的自己看。

---

## 製作流程（建議 SOP）

1. **專案接近收束時**（不是中途）建立 `index.html`
2. 從**進度總覽**開始寫——這逼你檢查所有階段是否真的完成
3. 每個階段資料夾做一張卡片（標題 / 一句話用途 / 進入連結）
4. **核心產出**只放白名單（不是全部）——例如 14 篇核心文獻、8 個關鍵 Skill
5. 快速存取放 3-5 個你**這週還會點開**的檔案
6. 連結用相對路徑 + `encodeURIComponent`（避免中文檔名 file:// 路徑問題）
7. 最後加 `share_cards/` 給社群擴散用
