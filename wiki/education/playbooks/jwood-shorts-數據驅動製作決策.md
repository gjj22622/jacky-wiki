# Jwood Shorts 數據驅動製作決策 SOP

> **給另一個 AI 的 brief**：你接手 Jwood YouTube Shorts 製作決策。本文教你怎麼從數據讀出問題、做出正確的選題與格式決策，並避免踩過的坑。

---

## 情境說明

Jwood 是台灣手工木藝按摩棒品牌（綠檀掌心雷），透過 YouTube Shorts 穴道教學吸引 55–65 歲以上男性觀眾，轉換路徑是 LINE OA。

每週需製作 5 支 Shorts，決策依據是 `gdrive:Jwood/strategy/strategy.json`。

## 前置條件

- `rclone` 已設定 `gdrive:` remote
- 工作目錄：`/home/jacky/claude/projects/Jwood`
- 已有 `data/shorts/agent_sop.md`（製作技術 SOP，本文只談決策）

## Step 1：讀取策略資料

```bash
rclone copy gdrive:Jwood/strategy/strategy.json /tmp/jwood_strategy/
```

讀 `strategy.json` 的三個關鍵區塊：

| 欄位 | 意義 | 如何用 |
|------|------|--------|
| `goal.gap_multiplier` | 目標觀看數的差距倍數 | >10x 拚速度，3–10x 拚品質 |
| `next_videos[]` | 依評分排序的優先穴道清單 | 照順序製作，不要自己選 |
| `audience.dominant_age` | 主力年齡層 | 決定 hook 語言風格 |

**關鍵判斷**：`next_videos[].score` 低於 30 的穴道不要製作，等演算法更新資料。

---

## Step 2：診斷現有影片為什麼觀看低

如果你發現最近幾支影片觀看數從 1,000+ 直接掉到 100–200，不要先改選題，**先比較影片結構**。

### 診斷指令

```bash
for ep in 33 34 35 40 41 42 43; do
  echo "=== EP${ep} ==="
  python3 -c "
import json
data = json.load(open('output/shorts/scripts/scene_series_A_ep${ep}_short1.json'))
for s in data['scenes']:
    print(f\"  [{s['type']}] {s.get('duration_sec','?')}s\")
"
done
```

### 判讀標準

| 結構 | 時長 | 預期表現 |
|------|------|---------|
| hook + demo + cta | 28–36s | ✅ 1,000+ 觀看 |
| hook + demo + **product** + cta | 38–50s | ❌ 100–200 觀看 |

### 斷點案例（2026-05-18 改版）

EP33–EP40：`hook(3s) + demo(22s) + cta(5s)` = **30s，無 product 場景** → 平均 1,200–1,700 觀看  
EP41–EP45：`hook(3s) + demo(20s) + product(12s) + cta(5s)` = **40s，加了 product 場景** → 平均 100–200 觀看

**原因**：product 場景在第 23 秒出現，65 歲觀眾看完穴道就划走，完播率在 23s 斷崖。YouTube 讀到「這支留不住人」，停止推送，連帶壓制頻道後續影片的起始流量。

### 結論規則

> **Series A 絕對不加 product 場景。** Series A 的唯一功能是建立觀眾信任，轉換交給 Series B。

## Step 3：選題邏輯

### 用 score 排序，不用直覺

`next_videos[]` 已按加權分數排好。分數計算包含：人氣、完播率、訂閱轉換、機會值（距上次發的天數）。

**直接照清單順序做，不要問「我覺得哪個比較好」。**

### 哪些穴道不要碰

| 類型 | 例子 | 問題 | 處置 |
|------|------|------|------|
| 背部穴道（自己按不到）| 脾俞、命門 | 65+ 觀眾不會自按背部 → 觀看完馬上離開 | 跳過，選下一個 |
| score < 30 的穴道 | 百會（23.5）、肩井（20.1）| 受眾沒有這個痛點 | 跳過 |
| 已做 4 支的同穴道 | 神門穴（已做 4 支）| 演算法視為同類互相競爭 | 暫停該穴道至少 30 天 |

### 護眼系列是主線

攢竹穴（1,734 觀看）、四白穴（1,504）、睛明穴（1,435）全是護眼穴道，且訂閱轉換率 2–2.5/千次。

**原因**：主力受眾是 65+ 男性，最大的焦慮是視力退化、老花眼。這是剛需，不是想要。

每次選題優先看 `next_videos[]` 裡有沒有護眼穴道，有就先做。

## Step 4：Hook 語言必須對到 65 歲男性

### 受眾數據（2026-05-20）

| 年齡段 | 佔比 |
|--------|------|
| 65 歲以上 | 33.2% |
| 55–64 歲 | 18.5% |
| 合計 55+ | **51.7%** |

男性 59%，女性 41%。

### 錯誤 vs 正確語言對照

| ❌ 年輕人語言 | ✅ 65 歲男性語言 |
|-------------|----------------|
| 眼睛疲勞 | 老花眼越來越嚴重、看字要靠近 |
| 睡眠品質不好 | 半夜醒來睡不回去 |
| 肩頸痠痛 | 頸椎退化、後頸一直緊 |
| 消化問題 | 飯後脹氣、胃越來越差 |
| 沒有精神 | 體力越來越差、走路就喘 |

### Hook 結構公式

```
症狀問句（他們正在經歷的）？
同類症狀問句（確認這就是他）？
解法句（試試這裡）。
```

範例：
- ✅「後頸一直緊？低頭久了頭越來越重？試試這裡。」
- ❌「頸椎穴道有助改善頸部循環，建議按摩大椎穴。」

**前者說出他的感受，後者像醫學廣告。**

## Step 5：格式規則（Series A）

```
hook  →  3 秒
demo  →  22 秒（穴道位置 + 按法 + 適用症狀）
cta   →  5 秒（固定文字）

總長：28–36 秒
```

### CTA TTS（固定，不改）

```
覺得有效，先收藏起來。訂閱 J Wood Shorts，每週穴道不漏接。
```

注意：「J Wood」不是「Jwood」，TTS 讀法不同。

### Demo TTS 要含三件事

1. **穴道位置**：解剖定位（幾指寬、哪個凹陷、哪條筋旁）
2. **按法**：用哪根手指、施多少力、每次幾秒
3. **適用症狀 + 使用時機**：什麼情況按、何時按效果最好

---

## Step 6：同穴道做第二支的規則

若 `next_videos[]` 推薦的穴道已有一支影片，v2 必須換症狀角度：

| v1 角度 | v2 角度 |
|---------|---------|
| 用眼疲勞、盯螢幕 | 老花眼、視力退化、長期保養 |
| 肩頸痠痛、急性 | 頸椎退化、慢性改善 |
| 壓力大、睡前 | 長期體質調理 |

**不要只換標題，hook 和 demo TTS 要真的換角度。**

## 採坑點

### 坑 #1：product 場景殺完播率
- **症狀**：改版後所有影片觀看從 1,000+ 掉到 100–200
- **誤區假設**：加 product 場景可以同時教育 + 轉換
- **真實原因**：65 歲觀眾看完穴道就滑走，product 場景讓第 23 秒出現斷崖，YouTube 停止推送
- **解法**：Series A 永遠不加 product 場景，轉換靠 Series B
- **發生於**：2026-05-18 EP41–EP45 改版

### 坑 #2：背部穴道觀看極低
- **症狀**：脾俞穴 32 觀看、命門穴 251 觀看
- **誤區假設**：背部穴道也是常見保健需求
- **真實原因**：65+ 觀眾自己按不到背部，看完影片沒辦法馬上做，直接離開
- **解法**：選穴道優先選「自己能看到 + 能按到」的部位（臉、手、腳、頸後）
- **發生於**：2026-05-14 批次

### 坑 #3：同穴道重複太多支稀釋流量
- **症狀**：神門穴做了 4 支，平均觀看從原本的 1,586 掉到 795
- **誤區假設**：熱門穴道做更多支可以複製成功
- **真實原因**：演算法把同穴道多支影片視為互相競爭，分流量
- **解法**：同穴道間隔至少 30 天，第二支必須換症狀角度
- **發生於**：2026-05-18 神門穴分析

### 坑 #4：用年輕人語言對 65 歲受眾說話
- **症狀**：CTR 低，影片曝光夠但點進來的少
- **誤區假設**：「眼睛疲勞」「肩頸痠痛」是通用語言
- **真實原因**：65 歲男性的自我認知是「退化」不是「疲勞」，語言要對到焦慮而非症狀
- **解法**：hook 用「老花眼」「頸椎退化」「體力越來越差」等衰老語言
- **發生於**：受眾分析後發現，2026-05-21

### 坑 #5：高觀看穴道 ≠ 高評分
- **症狀**：勞宮穴觀看 1,518 但評分只有 62.8，聽宮穴觀看 1,613 但評分 57.9
- **誤區假設**：觀看數高的就優先做第二支
- **真實原因**：勞宮穴和聽宮穴訂閱轉換率極低（0.66–0.67/千次），帶進來的觀眾不黏，做再多也是過路客
- **解法**：依 `next_videos[].score` 選題，不要只看 `avg_views`
- **發生於**：2026-05-21 數據分析

## 驗收標準

製作完成後確認：
- [ ] 影片時長 28–36 秒
- [ ] 無 product 場景（type: product）
- [ ] hook TTS 有症狀問句
- [ ] demo TTS 含位置、按法、症狀三要素
- [ ] cta TTS 含「J Wood」（非 Jwood）
- [ ] 穴道在臉/手/腳/頸後（非背部）
- [ ] strategy.json 的 next_videos score > 30
