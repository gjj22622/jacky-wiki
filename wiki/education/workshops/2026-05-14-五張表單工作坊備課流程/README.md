# TBSA 五張表單工作坊備課流程（內訓版）

> 把 Jacky 接到 TBSA 五張表單工作坊邀請後，**從 0 到上台**的備課流程整理給團隊：怎麼分階段、五大產出長什麼樣、七個關鍵設計決策怎麼選、採坑點怎麼避。
>
> 同件事重複做才省力 — 這套流程目標是讓**第二場、第三場備課時，團隊接得起來**（含 AI 助手）。

---

## 🎯 學習目標

讀完這份教材你會：

1. 看懂一場 TBSA 五張表單工作坊備課要產出哪 **5 大成品**（規劃／主辦文件／簡報／AI 設定／範例）
2. 知道備課分**五個階段**（A 建檔 → B 設計 → C 通知 → D 素材 → E 上場前）哪個階段該做什麼
3. 能解釋**七個關鍵設計決策**（為什麼三選一、為什麼全雲端、為什麼決策的數學頁⋯⋯）
4. 避開 **5 個常見採坑點**
5. 能用[AI 餵食檔](AI餵食檔-備課助手.md)讓自己的 AI 助手扮演備課助手

## 章 1. 為什麼需要這套備課流程？

Jacky 每年大約接 6–10 場類似的工作坊。每場備課如果都從零開始，會有三個痛點：

| 痛點 | 場景 |
|---|---|
| **產出散落** | 規劃文件、簡報、AI 設定、範例 docx 各自一份，下次找不到對應檔 |
| **品質不穩** | 第一場想得很細的決定（如 SMART 原則的解釋方式），第二場想不起來 |
| **教學模式漂移** | 「三選一」「全雲端」這類核心 framing，沒寫下來就會自己改 |

所以團隊要把**這套備課流程**像生產線一樣固化：每場新案來，照五階段跑、生五大產出。

## 章 3. 備課的五個階段

```
A 建檔啟動  →  B 課程設計  →  C 通知工程  →  D 素材備製  →  E 上場前
（1 天）      （2-3 天）      （半天）       （3-5 天）      （半天）
```

| 階段 | 主要任務 | 產出 |
|------|---------|------|
| **A 建檔** | 確認課程資訊、開資料夾、寫工作流程記錄 | `工作流程記錄.md`（持續更新）|
| **B 設計** | 寫課程規劃 + 助教 SOP | `課程規劃_v?.md`、`助教SOP.md` |
| **C 通知** | 給主辦三份文件、發學員課前通知 | 3 份正式文件、Line 群組通知 |
| **D 素材** | 簡報架構 → PPTX build、AI 設定檔、範例 docx | 簡報 PPTX、AI 設定檔、範例 |
| **E 上場前** | 確認雲端 Doc、設備、列印備援 | 課堂 Checklist |

詳細步驟參考：[playbook 版 SOP](../../playbooks/五張表單工作坊備課SOP.md)

## 章 4. 七個關鍵設計決策（Jacky 的方法觀）

這 7 個決策是這套備課流程的**靈魂**，照抄就能複製品質：

### 決策 1｜為什麼是「三選一」？

學員用 AI 時，預設 AI 給「一個答案」會讓學員照抄。
改為「給三個方向，學員選一個 + 寫理由」，**選擇本身就是企劃力**。

數學佐證（簡報 Slide 06「決策的數學」）：12 個決策點 × 3 選 1 = `3¹² = 531,441` 種可能企劃。

> 評審不看你用了哪個 AI，他看你選了哪一個 + 為什麼。

### 決策 2｜為什麼全雲端 Google Doc？

舊版（v1）用紙本表單 + 海報 + 便利貼；新版（v2）全部進 Google Doc：

| 對比 | 紙本（v1）| 雲端（v2）|
|------|----------|----------|
| 講師監控 | 走動看 | 同時開 5 個 tab 即時看 |
| 學員產出保存 | 拍照、容易遺失 | 自動雲端、版本史 |
| 隔天 Demo | 重 key 入 PPT | 直接打開 Doc 當提詞機 |
| 決策紀錄 | 寫在便利貼上 | 每張表單一個欄位、課後可檢索 |

### 決策 3｜四段式時間結構

每張表單 = **概念說（10min）→ AI Demo（10min）→ 組內實作（30-50min）→ Checkpoint（5-8min）**。

- 概念說：建立評審視角（最常扣分理由）
- AI Demo：Jacky 現場跑 prompt → 選一個 → 講為什麼
- 實作：5 組同步，講師巡 5 個 Doc tab
- Checkpoint：1-2 組 60 秒口頭說明 → 練明天 Demo 開場白

### 決策 4｜Checkpoint = 明天 Demo 的預演

每張表單最後的 60 秒口說，**不是隨機抽人懲罰**，而是「今天練的就是明天說的」。
五場 Checkpoint = 明天 5 個發表段落的骨架。

### 決策 5｜AI 設定檔讓全班 AI 同步

學員一進教室，第一件事不是聽課，是**把 AI 設定檔貼進自己的 AI 對話視窗**。
這份設定告訴 AI：你今天是 TBSA 競賽策略夥伴、你要扮演三角色（研究員／顧問／評審）、回答必須三選一格式。

→ 全班 AI 同步進入「同一個工作模式」，講師示範一次 prompt 全班都能跟著用。

### 決策 6｜助教定位＝「名詞翻譯員」

亞洲科大／明志科大老師擔任助教，他們不熟五張表單。
助教 SOP 明確切出邊界：

| 學員問什麼 | 助教做什麼 |
|----------|-----------|
| 名詞定義（什麼是 SWOT）| **答**（對照 SOP 第三節）|
| 方向判斷（這個策略對嗎）| **不答**，引導學員自己想，方向題等 Checkpoint Jacky 處理 |

→ 助教不會被問倒、不會教錯方向，講師專注關鍵節點。

### 決策 7｜決策的數學頁（v2.1 新增）

簡報第 6 張：`3¹² = 531,441 種可能` 的視覺衝擊頁。
用大數字告訴學員「你今天做的不是填表，是從 53 萬種裡選你們組的那一個」。

→ 把「選擇 + 寫理由」的價值感拉到滿。

## 章 5. 採坑點（最珍貴）

### 採坑 #1：簡報 PPTX 沒分版本就改

**症狀**：v1 簡報架構改到一半發現新點子，直接改下去 → 想對照差異時找不到 v1。
**解法**：**架構檔 + build script 都打版本號**（`主講簡報_架構_v2.md`、`主講簡報_架構_v2.1.md`、`build_main_deck_v1.js.bak`）。每次重要改動 +0.1。

### 採坑 #2：插入新 slide 後編號順移做不乾淨

**症狀**：在原 Slide 05 後插入新 Slide 06 → 後面 26 張要全部 +1 編號。手動改容易漏。
**解法**：用 PowerShell 反向 loop 替換（`for i=31 to 6: Slide i → Slide i+1`），再個別處理範圍格式（`Slide 07–10`）和總頁數。
**範例**：`02_課前/pptx_build/build_main_deck_v1.js.bak`（v1 版備份）+ 新版 32 頁建置完成。

### 採坑 #3：助教不熟表單，現場被問倒

**症狀**：助教當下不知道怎麼回 → 學員失去信心。
**預防**：開課前 1 週把 [助教 SOP](#) 寄給助教，列「最常被問的 10 句 + 標準回答」。
**邊界**：助教不做方向判斷，只做名詞翻譯。

### 採坑 #4：學員 AI 出英文回答

**症狀**：AI 預設輸出英文，學員不會修。
**解法**：AI 設定檔開頭明寫「請永遠用繁體中文回答」+ 助教 SOP 列「AI 出英文怎麼辦」標準應對。

### 採坑 #5：課前通知忘了發

**症狀**：開課當天才發現學員沒帶電腦、沒申請 AI 帳號。
**預防**：在 todolist.md 把「課前 1 週通知」設成🔴 高優先；建檔 A 階段時就把通知模板寫好，到時直接複製貼上 Line 群組。

## 章 6. 怎麼複製到下一場

新案進來，三步驟：

1. **複製專案資料夾骨架**：`cp -r 06_20260517_台中科大五張表單 0?_<新日期>_<新場名>`
2. **更新工作流程記錄.md**：填新課程資訊、清空進度表
3. **依五階段執行**：照 [playbook SOP](../../playbooks/五張表單工作坊備課SOP.md) 逐步跑

**省力來源**：
- 簡報架構 v2.1 可重用，只改場名／日期／講師字段
- AI 設定檔可重用，幾乎不用改
- 助教 SOP 可重用
- 範例 docx 用 [tbsa-forms skill](#) 生新案
