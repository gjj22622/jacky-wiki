# 備課品質 Gate

| Gate | 通過條件 | 失敗時回退 |
|---|---|---|
| G0 Intake | 日期、受眾、時長、場合、可檢查目標、限制已知 | 補資料或重寫 O |
| G1 Route | 主課型唯一；必需、選配、禁止模組明確 | 重判五維，不以主題猜課型 |
| G2 Retrieve | 1–3 前例及可借／不可借項；或明示無前例 | 擴查索引或採最近骨架 |
| G3 Design | 記憶錨點、時間表、互動、收束與交付定義一致 | 先修策略，不做投影片 |
| G4 Validate | 關鍵假設有證據、主辦確認或明示風險 | 補最低成本驗證 |
| G5 Build | 路由要求的交付齊；來源、版本、最終版清楚 | 修缺件，不擴張範圍 |
| G6 Rehearse | 時間、視覺、內容、權限、Demo、備援、隱私全過 | A→T→S 逆推修正 |
| G7 Deliver | 偏差與現場證據已記錄 | 課後補記，不改寫事實 |
| G8 Review | 結果對 O、根因及下一輪改動有證據 | 區分感想與因果 |
| G9 Promote | 有兩場證據或一次高風險明確因果；已去重 | 留候選，不升格 |

## 共同紅線

- 演講與工作坊素材不得直接互套。
- 客戶、學員個資與未授權案例不得進公開教材。
- 重大改動後必須全程串讀；T-24 小時凍結非必要視覺改動。
- Demo 沒有導航與備援，不得視為通過。
- 自動生成檔若已被人工修改，重生前必須確認或另存新版本。
- 迴圈必須有至少兩個終止條件，且不得自行修改規則。

## 逆推除錯

從成本最低處倒查：`A 行動 → T 工具／環境 → S 策略 → O 目標 → S 現況真偽`。同層最多重試三次；沒有新證據不得原樣重跑。
