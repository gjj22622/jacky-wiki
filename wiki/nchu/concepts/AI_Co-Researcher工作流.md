# AI Co-Researcher 工作流

> **這套工作流不是「文獻管理」，是「AI 共同研究者」**。
>
> 核心差異：
> - 文獻管理：人類定方向、AI 執行搜尋
> - AI Co-Researcher：**AI 自主累積對博論的理解**、自主判斷下一步、自主教 Jacky

---

## 為什麼建立這套流程

### 問題（傳統「人類每天搜文獻」的三個失敗）

1. **疲勞性失敗**：每天找 3-5 篇 → 一週後就重複，人類吃不下、放棄
2. **方向死板**：人類定的關鍵字一旦設定就不太動，但博論方向會隨見識變化
3. **看完就忘**：論文翻譯了還是要時間消化，沒有「持續吸收 + 教學回饋」的閉環

### 解法（三個翻轉）

| 傳統 | AI Co-Researcher |
|---|---|
| 量大、求覆蓋 | **質精、求 insight**（最多 1 篇/天，可以跳過） |
| 人類負責調方向 | **AI 負責調方向**（人類只在偏太遠時糾偏） |
| AI 翻譯交給人類消化 | **AI 自己消化** → 教學包餵給人類（含 mp3） |

## 四個動作（AI 的日常行為）

### 動作 1：選（State-driven Selection）

**輸入**：
- `博論研究狀態.md`（當前 5 軸狀態 + insight + gap + trend）
- `下一步候選池.md`（前一天 AI 推的候選）
- `每日學習日誌.md` 最近 7 天

**處理（reflection layer，「彈性」所在）**：
```
1. 上週累積的 insight 是否指向新的子軸？（→ 開新軸）
2. 哪個子軸最近沒進度但仍 hot？（→ 補進度）
3. 候選池有沒有「重複性高」的論文？（→ 跳出舒適圈，找跨領域）
4. 用 arXiv API 搜尋當前最匹配的論文，自評質量
5. 若沒有夠好的候選 → 決定今天 pass（質 > 量）
```

**輸出**：今日讀的論文 DOI / arXiv ID（或 "pass"）

---

### 動作 2：學（Deep Read + Internalize）

**輸入**：當日選定的論文

**處理**：
- 下載 PDF（命名照第三次專討規律：`{Chx}_{Author}_{Year}_{slug}.pdf`）
- 抽取 Figure / Table（用 pymupdf 或 pdf2image）
- 用 Claude API 做：
  - 逐字翻譯（保留 `--- Page X-Y ---` 標記）
  - 每個 Figure / Table 的中文解讀
  - 對應到博論哪個子軸 / 填了什麼 gap / 帶來什麼 insight

**輸出**（3 個檔，存在本機）：
```
博士論文方向思考/文獻吸收_daily/{YYYY}/{MM}/{MMDD}_{軸}_{Author}_{Year}/
├── 原文.pdf
├── 中文翻譯.md
├── 圖表說明.md
└── 主軸對應.md
```

---

### 動作 3：更新研究狀態（Self-Evolving State）

**輸入**：動作 2 的 3 個產出檔

**處理**：
- 把這篇的 insight 寫進 `博論研究狀態.md` 對應子軸的「最新 insight」「gap」「熱度」
- 更新 `文獻追蹤總導覽.md`（新增條目 + 重排序 / 標 hot / 標 cold）
- 在 `每日學習日誌.md` 加一筆（含「狀態怎麼從 X 變成 Y」的反思）
- 動態調整 `下一步候選池.md`（推下一步該讀什麼）

**輸出**：4 個檔的更新版

---

### 動作 4：教（Teach Jacky in 5 min）

**輸入**：動作 2、3 的成果

**處理**：
- 產出 `教學包.md` — **不是翻譯，是「5 分鐘給 Jacky 的精華」**
  - 結構：今天讀什麼 → 核心 insight 3 句 → 對博論的意義 → 推薦下一步
- 用 edge-tts 生成 `教學.mp3`（雲哲男聲，~5 min）
- 通知（Email / 桌面通知 / Claude Code session brief）

**輸出**：
```
{MMDD}_{軸}_{Author}_{Year}/
├── 教學包.md
└── 教學.mp3
```

## 隱私與安全（依 jacky-wiki/CLAUDE.md）

| 入 wiki | 不入 wiki |
|---|---|
| ✅ 研究方向、insight、引用文獻、與主軸對應分析 | ❌ 未發表的研究計畫具體執行細節 |
| ✅ 已發表論文的閱讀心得 | ❌ 跟指導老師私下討論的回饋 |
| ✅ 領域 gap 分析（公開資訊）| ❌ 論文初稿、原始實驗數據 |

**本機儲存原則**：PDF + 翻譯 + mp3 太肥不能上 wiki；只把「腦」（狀態、導覽、log）push 到 wiki。

## 驗收標準

- **跑滿 7 天連續**：AI 自主選篇沒選到重複 / 不相關的
- **每天 mp3 Jacky 聽得進去**（不會 skip）
- **一週後翻 `每日學習日誌.md`** 能看出「研究狀態怎麼演化」
- **Jacky 自評**：「我感覺 AI 比上週更懂我的博論」

## 跟 ailab 的分工

| ailab 域 | nchu 域（本系統） |
|---|---|
| 個人 AI 實踐的活知識（捕事件） | 博論文獻 + 研究狀態（特定學術主題） |
| 對象：未來的自己 | 對象：博論進度本身（AI 是參與者） |
| 跨專案、跨主題 | 鎖在博論上下文 |

兩者**互不重疊**：ailab 是工具觀，nchu 是學術根基。
