## 🔗 視角 3：主題軸 ↔ 章節對應矩陣

> 一個軸通常會貢獻到多個章節（特別是文獻回顧 Ch2 與方法 Ch3）。
> 一個章節通常需要多軸支援。

| | Ch1 緒論 | Ch2.1 異速 | Ch2.2 光學 | Ch2.3 LiDAR | Ch2.4 ML/DL | Ch2.5 融合 | Ch2.6 FDT | Ch2.7 SIF | Ch3 方法 | Ch4 論文1 | Ch5 論文2 | Ch6 論文3 | Ch7 討論 |
|---|:-:|:-:|:-:|:-:|:-:|:-:|:-:|:-:|:-:|:-:|:-:|:-:|:-:|
| **A FDT** | ● | | ● | | ● | ● | ⭐ | | | | ⭐ | ⭐ | ● |
| **B SIF** | ● | ⭐ | | | | | | ⭐ | ● | | | ⭐⭐ | ● |
| **C GEDI** | ● | | | ⭐ | | ● | | | ● | ⭐ | ⭐ | | ● |
| **D FM/SSL** | ● | | | | ⭐ | ● | | | ● | ⭐⭐ | | | ● |
| **E 台灣 MRV** | ⭐ | | | | | | | | | | | ● | ⭐ |

**符號**：● = 一般引用、⭐ = 主要支撐、⭐⭐ = 核心貢獻

**讀法**：
- **直看**（一個軸貢獻給幾章）：軸 A 貢獻 8+ 章（最廣）；軸 B 貢獻 5 章但 Ch6 是「核心貢獻」
- **橫看**（一章需要幾軸）：Ch6 需要 A+B+C（最複雜，也是最大亮點）；Ch4 主要靠 D+C
