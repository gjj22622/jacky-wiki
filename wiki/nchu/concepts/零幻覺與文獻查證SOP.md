# 零幻覺與文獻查證 SOP

> 真實事故觸發的方法論：2026-03-12 檢查 26 個帶作者名的檔案，發現 **19 個錯誤（73% 錯誤率）**，2 個 PDF 的內容與檔名完全不一致。
>
> 這個事故反推出 [來源控制與 script_viewer 模式](../../cross-domain/來源控制與script_viewer模式.md)——AGENTS E 階段的**事前品管**第二軌。

---

## 一句話定位

**零幻覺 = AI 寫學術文件時引用必須可追溯到具體 PDF 頁碼**——
任何不能對應到 PDF 第一頁作者列表、具體段落、明確數字的引用，都視為「**幻覺嫌疑**」，必須標記 `[⚠️核對]` 直到驗證為止。

---

## 事故重演（2026-03-12）

### 發現過程
專討準備期間用 AI 整理文獻並命名檔案。命名規則：`Ch{N}_{第一作者}_{年份}_{標題關鍵詞}.pdf`。某次 review 偶然發現一個檔名作者與 PDF 內容對不上，啟動全面查核。

### 結果
| 指標 | 數值 |
|---|---|
| 受查檔案 | 26 個 |
| 作者名錯誤 | 19 個（73%）|
| PDF 內容與檔名完全不一致 | 2 個 |
| 查核耗時 | 約 1 個工作日（人工逐個翻 PDF 第一頁）|

### 為什麼會發生
1. AI 在「批次重命名」「批次摘要」時會**內插出聽起來合理但不存在的作者**
2. 中文混英作者名容易出錯（e.g. Tian L. vs Tian X. 是兩個不同人，AI 容易混）
3. 共同作者多時，AI 抓的不一定是真正的第一作者
4. 沒有 viewer 工具可以一鍵驗證

## 八項零幻覺原則

從事故反推 + seminar-helper 系統化後的核心紀律：

| # | 原則 | 操作方法 |
|:---:|---|---|
| 1 | **零幻覺原則** | 所有數字、作者、結論可追溯到 PDF 具體頁碼；不能確認標 `[⚠️核對]` |
| 2 | **單一資料來源** | 核心文獻收在 `專討文獻集/` 一個資料夾，禁止外引 |
| 3 | **作者名 PDF 查核** | 對照 PDF 第一頁作者列表，不靠 Google Scholar 或 metadata |
| 4 | **導讀禁止重做** | 導讀 PPTX 完成後只能引用／複用，避免重新生成時又產生新幻覺 |
| 5 | **檔名規則嚴格** | `{編號}_{章節}_{第一作者}_{年份}_{原文｜導讀}.{pdf｜pptx}` |
| 6 | **數字必須帶頁碼** | 引用任何 R²、RMSE、像素數、誤差 → 必須附 `(p.X, Table Y)` |
| 7 | **引用查核表優先於講稿** | 先鎖定每段引用的數字／頁碼，再寫口白 |
| 8 | **白名單硬邊界** | script_viewer.html 的 REFS 物件 hard-code，引用必須匹配才能 hover |

## 雙軌品管（與 AGENTS E 階段對應）

| 軌 | 名稱 | 時機 | 工具 |
|---|---|:---:|---|
| 軌 1 | **事前可追溯性** | 產出當下 | 引用查核表 + script_viewer.html + REFS 白名單 |
| 軌 2 | **事後殘酷測試** | 產出之後 | 三色情境 + 演練 R1-R4 + Q&A 模擬 |

> AGENTS v1.0 只定義了軌 2（殘酷測試三色情境）。NCHU 場景反推出軌 1——**這是 v1.1 應該補的核心章節**。

## 實際 SOP（從文獻收集到上場）

### 階段 1：文獻收集時
- 用 NCHU IP / VPN 從 Springer / Elsevier / Wiley / T&F 下載 PDF
- 命名前**先翻 PDF 第一頁**確認第一作者
- DOI 驗證：把 DOI 貼到瀏覽器確認連結到正確期刊頁
- 建 Excel 文獻總表（# / 章節 / 第一作者 / 年份 / 期刊 / 標題 / DOI / 主題標籤 / PDF 下載狀態 / 備註）

### 階段 2：核心精選
- 從 30-40 篇選 10-14 篇進「專討文獻集」
- 進去就是白名單，後續禁止外引
- 每篇 1 份導讀 PPTX + 1 份中文翻譯 .md（建議）

### 階段 3：講稿引用
- 每段引用標 `[{編號}]` + 頁碼（e.g. `[05] (p.42, Table 3)`）
- 先寫**引用查核表**：段落 / 時長 / 使用文獻 / 引用頁碼 / 引用具體內容（含數字）/ 視覺素材出處 / 核對狀態
- 查核狀態：`[⚠️核對]` → 翻 PDF 確認 → `[✅ 已驗證]` 或 `[❌ 不一致]`

### 階段 4：用 script_viewer 巡視
- 把講稿 .md 轉成 `script_viewer.html`
- 每個 `<cite data-num="XX">` 標籤對應 REFS 白名單
- 上場前最後一次 hover 巡視所有引用，確認都能跳到原文

### 階段 5：上場後若投稿
- 從專討文獻集擴展到投稿用的更全文獻集
- 同樣紀律：可追溯、白名單、查核表

## 工具支援

### 已實作
- `seminar-helper` 的 `/seminar review <file>` 指令——讀檔案、抓引用、對照 `專討文獻集/` PDF 逐一查核、產出 `[⚠️核對]` `[❌ 不一致]` `[✅ 已驗證]` 報告
- `script_viewer.html` 模式——視覺化引用 → 來源映射

### 規劃中
- `literature-verifier` Skill（**最急**）——獨立的文獻驗證 Skill，串接 script_viewer 的 REFS 白名單做自動比對

詳見 [學術 Skill 體系](學術Skill體系.md)。

## 反模式（不要這樣做）

| 反模式 | 問題 | 正確做法 |
|---|---|---|
| 信任 AI 給的作者名 | 73% 錯誤率 | 必翻 PDF 第一頁 |
| 信任 Google Scholar 的 metadata | 也可能錯（中英混名）| 以 PDF 為準 |
| 引用「其他資料夾」的文獻 | 突破單一資料來源原則，未來無法復現 | 全部進專討文獻集 |
| 講稿先寫完再查引用 | 已經把錯誤的引用寫進敘事邏輯了 | 引用查核表先於講稿 |
| 重做導讀 PPTX | 會產生新的幻覺 | 只能引用／複用 |

## 跨場景擴展

零幻覺紀律不只學術——任何「事實宣稱必須可追溯」的場景都適用：

| 場景 | 「白名單」是什麼 | 「PDF 頁碼」相當於什麼 |
|---|---|---|
| 学術 | 14 篇核心文獻 | PDF 頁碼 + Figure / Table 編號 |
| 行銷（双云客戶內容）| 客戶品牌資料庫 | 品牌定位文件具體段落 |
| 木藝（Jwood 影片）| 穴道知識來源 + 醫學文獻 | 解剖位置與按摩手法依據 |
| 教學（TBSA）| 兩本 TBSA 教材 | 教材章節頁 |

> 詳見 [來源控制與 script_viewer 模式 §跨場景應用](../../cross-domain/來源控制與script_viewer模式.md)。
