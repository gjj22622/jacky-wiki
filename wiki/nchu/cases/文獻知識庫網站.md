# J博Team 文獻知識庫網站

> 把博士研究累積的森林數位孿生（FDT）／森林碳估算文獻整理，從私人資料夾升級成**公開、有發布節奏的內容產品**。一邊學、一邊跟趨勢、一邊讓有興趣的人一起學，並逐步發展為創業產品。

---

## 一句話定位

> **這個網站是臺北市公有林數位孿生展示系統的「方法論來源端」。** 展示系統秀成果，文獻庫秀支撐成果的文獻證據。兩邊是獨立網站，互相連結。

---

## 線上位置

| 網站 | 網址 | 角色 |
|---|---|---|
| 文獻知識庫（本案）| https://forest-lit.zeabur.app | 方法論／文獻證據端 |
| 臺北市公有林 FDT 展示系統 | https://tpe-forest-dt.zeabur.app | 成果展示端 |

- 同一個 Zeabur 專案 `Forest_FDT` 下兩個 Git 自動部署服務，各綁各的既有 GitHub repo，push main 自動重部署。
- 兩站雙向互連（文獻站導覽列＋關於頁 → FDT；FDT 工具列＋頁腳 → 文獻站）。
- 部署 ID/URL 操作細節記在 repo 內 `部署到Zeabur.md`，wiki 不重複。

---

## 雙層架構

| 層 | 給誰看 | 內容 |
|---|---|---|
| 前台 **學習筆記** | 一起學習的人 | 敘事好讀：勾子 → 問題 → 材料方法 → 結論 → 討論 → 重要科學發現 → 博士生 takeaway → 短引用＋原文 DOI |
| 後台 **審核卡** | 查核與引用 | 完整書目、結構式摘要、證據表、批判評估（沿用既有嚴謹格式）|

兩層共用同一份 JSON 為單一事實來源（git 可追溯，符合零幻覺原則）。

## 四層產品規劃（實作分階段）

1. **前台內容層**（已實作 MVP）：品牌首頁時間軸（最新在前）＋學習筆記文章頁＋RSS＋關於頁。
2. **後台製程層**（scaffold）：`/article` skill 產學習筆記草稿＋編輯佇列 draft→review→scheduled→published＋發布前查核。
3. **資料庫層**（scaffold）：JSON 為事實來源，Postgres `articles_meta` 鏡像供查詢／會員閘控。
4. **付費訂閱層**（scaffold）：magic-link 認證、綠界 ECPay 金流、paywall 反灰、會員分級。

## 發布節奏

- 開站庫存先回填既有導讀素材，目前 **12 篇**（11 篇導讀 PPTX ＋ Sasaki）。
- 每 3 天一篇：未來日期的文章用 `status_pub=published` ＋未來 `publish_date`，靠 build 機制自動 drip 上線。

---

## 原則（與專討一脈相承）

- **零幻覺**：所有數字／作者／年份／頁碼／Table 編號可追溯到來源，無法確認標「待查」，不編造。
- **單一資料來源**：學習筆記只從既有導讀素材轉述，不重新製作截圖或中英對照。
- **原創導讀＋短引用（≤25 字）＋連 DOI**：全文中文翻譯只限 CC-BY 開放取用論文。
- **中文標題／副標／摘要不用冒號**。
- 詳見 [零幻覺與文獻查證 SOP](../concepts/零幻覺與文獻查證SOP.md)。
