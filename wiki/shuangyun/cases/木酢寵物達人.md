## SOSTAC 拆解

### S — Situation（2026 Q1 現況）

**雙通路結構**：蝦皮為主（65.9%）+ 官網為輔（34.1%），尚未形成多通路均衡。

| 指標 | 數據 |
|------|------|
| 月均總收入（Q1'26）| 29.6 萬 TWD |
| 月均淨收益（Q1'26）| 11.8 萬 TWD |
| 蝦皮 Q1'26 月均 GMV | 19.5 萬（YoY −40%）|
| 官網 Q1'26 月均收入 | 10.1 萬 |
| 官網平均客單 | 963 TWD（高於蝦皮 66%）|
| 蝦皮平均客單 | 578 TWD |
| 蝦皮轉換率 | 5.45%（電商均值 2-3%，表現優異）|
| 總會員數 | 17,090 人 |
| 會員回購率 | 13.8%（1萬買家中僅 1,522 回購）|

**警訊**：蝦皮 Q1'26 vs Q1'25 GMV 下滑 39%（95.9萬 → 58.4萬），訪客數萎縮為主因（月均 3,227 vs 5,682），轉換率仍穩定，問題在流量獲取。

### O — Objectives（目標）

- **短期**：月均收入達 50 萬 TWD（缺口 20.4 萬）
- **通路重心轉移**：降低對蝦皮的單一依賴，官網比重從 34% 提升至 45%+
- **回購率提升**：13.8% → 18%（多推播分群驅動）

### S — Strategy（策略方向）

1. **LINE 推播為最高 ROI 槓桿**：10.7% sessions 貢獻 33.5% 官網收益（ROI 3.13×）
2. **官網高客單 × 會員深度經營**：客單比蝦皮高 66%，精準推播可快速放大
3. **蝦皮流量診斷 × 廣告補強**：轉換率健康，流量端補廣告
4. **沉睡會員喚回**：6,098 人從未購買，低成本高潛力

### T — Tactics（戰術）

| 戰術 | 預估月增量 | AI 可執行比例 | 啟動時序 |
|------|------|------|------|
| LINE 推播加速 × 精準分群 | +3 萬 | 90% | Week 1 |
| 蝦皮廣告 × 流量復甦 | +2.5 萬 | 75% | Week 2 |
| 沉睡會員喚醒（6,098人）| +1.5 萬 | 85% | Week 1 |
| 官網 AOV 提升（組合 + 訂閱）| +1 萬 | 80% | Week 3 |
| Google SEO × 內容行銷 | +0.8 萬 | 80% | Month 2 |
| B2B / ESG 企業採購 | +2 萬 | 70% | Month 2 |

前三策略合計 **+7萬/月**，Month 1 可執行，平均 AI 比例 83%。

### A — Action（行動重點）

- **LINE**：月推播從 1-2 次 → 4-6 次；依品類分群（除臭液 / 洗毛精 / 肌膚保護）；30天未購買者自動推補貨提醒
- **蝦皮**：診斷 Q1'26 衰退原因（關鍵字競爭 / 廣告預算 / 搜尋排名）；開啟關鍵字廣告（臭臭退散、寵物除臭液、木酢）
- **沉睡會員**：Facebook 再行銷 Custom Audience；LINE 推播喚回序列（第1週折扣 → 第2週新品）
- **訂閱制**：臭臭退散每 45 天自動出貨，95折免運

### C — Control（管控指標）

- 月均 GMV（目標 50萬）
- 蝦皮 vs 官網收入比例（目標 55:45）
- LINE 推播開信率 / 點擊率 / 每次推播收益
- 回購率（月追蹤，目標 18%）
- 官網行銷費 ROI（S4 = 官網總花費，Y4 = 官網收益公式）

## AI 數據顧問（兩個賣點）

**1. 直接回答（➤ 按鈕）**
- 模型：Claude Sonnet 4.6
- 架構：**Tool Use（agentic loop）** — 8 個工具讓 Claude 主動查詢
- 工具：`get_ga4_summary` / `get_ga4_sources` / `get_ga4_funnel` / `get_line_campaigns` / `get_meta_summary` / `get_meta_campaigns` / `get_monthly_trend` / `get_crm_snapshot`
- SSE 串流回應，前端顯示「🔍 查詢中」進度

**2. 生成指令（📋 按鈕）⭐ 產品定位翻轉**
- **不呼叫 Claude API**，只抓資料
- 把即時數據格式化成「含數字的 prompt」，使用者複製貼到自己的 Claude.ai Pro / ChatGPT Plus / Gemini Advanced 用
- 使用者不需要再付一次 AI 費用，平台不需要負擔他們的 token 用量
- 平台核心價值定位：「**整合數據成可被 AI 理解的指令**」，而不是「自己當 AI」

## 技術棧（零依賴原則）

- Node.js 24 ESM + 純 HTML/Vanilla JS + 不裝任何 npm 套件
- 用 `node:http` + 原生 `fetch` + `node:crypto`（JWT 簽名 + token 產生）
- 部署：Zeabur 自動 build，環境變數注入
- Token 儲存：`data/users.json`（純 JSON 檔，不需要資料庫）

## 三個核心架構決策

1. **HTML cookie + API Bearer 雙軌認證** — 前台頁面 server-side 強制 redirect、API 用 Bearer header
2. **AI Agent Tool Use 而非 Context Injection** — 之前把所有數據塞進 prompt 是錯的，應該讓 Claude 主動呼叫工具
3. **「生成指令」按鈕** — 平台從「AI 助手」升級成「數據準備層」，使用者帶著結構化的 prompt 去用自己的 AI
