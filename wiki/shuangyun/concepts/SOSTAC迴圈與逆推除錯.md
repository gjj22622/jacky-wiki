# SOSTAC 迴圈與逆推除錯

> 双云 AI School **L6「迴圈」章節**的核心方法論。一句話：**把學員在 L2 學過的 SOSTAC，變成一個會自己跑到完成的 AI 工作迴圈**；正著跑＝建迴圈，逆著查＝除錯，全程繞著目標 O，不空泛、不失控、不漂移。

## 1. SOSTAC 本身就是一個迴圈

六格各自變成一個 Agent/步驟，C（交付）回頭成下一輪的 S（現況），繞著 O 反覆直到達標：

| 格 | AI 迴圈裡是什麼 | 誰做 |
|---|---|---|
| S 現況 | 現況收集 Agent：抓這輪起點事實、上一輪交付 | Agent |
| O 目標 | 人給；可檢查；**迴圈的圓心** | 人 |
| S 策略 | 拆解任務、擬定策略 Agent（plan mode） | Agent |
| T 戰術 | 執行工具與環境確認 | Agent＋人 |
| A 行動 | 執行，產一版 | Agent |
| C 控制 | 交付＋對 O 打分；**C 成為下一輪的 S** | Agent＋人 |

對應 AGENTS 六字訣的 **N（串成迴圈）＋T（自我監控直到達標）**深化；見 [AGENTS知識體系](AGENTS知識體系.md)。

## 2. 兩種「逆」：建構用 × 修正用

| | 逆向工程（建構用） | 逆推除錯（修正用） |
|---|---|---|
| 何時 | 設計迴圈、定目標時 | 迴圈跑不到目標時 |
| 怎麼做 | 從一個**成功成品**倒推結構與成功邏輯 → 提煉可複製流程 | 從失敗端 **A→T→S→O→S** 逐層往回查 |
| 解決的痛 | 打掉「**O 太大太空泛**」（目標從成功案例倒推，自然具體） | 防止 loop 越跑越遠 |

逆向工程借鏡 Jacky 2025-08《數位行銷AI逆向工程》；其六原則（目標導向／系統拆解／觀察收集／假設驗證／模式識別／重建理解）≈ 跑一輪 SOSTAC。

## 3. 逆推除錯階梯（最有價值的一招）

跑不到 O，**倒著沿 SOSTAC、從便宜改起、不行才往根本爬**，每層約試 3 次再上爬：

```
A 行動錯？ →(3次↑)→ T 工具/環境錯？ →(3次↑)→ S 策略錯？ →(3次↑)→ O 目標太空泛？ → S 現況根本錯(AI 幻覺)
```

- 兩個終極病根：**O 訂壞** 或 **現況 S 是 AI 幻覺（起點就假）**。
- 每次失敗都被拉回「跟目標的距離」逐層查 → 迴圈永遠繞著 O，這是迴圈的**煞車＋方向盤**。

## 4. 監控與終止（讓它安全自走）

- **評審 Agent**：每輪拿 C 對 O 打分（殘酷測試 3 色可升級成打分標準）；見 [殘酷測試](AGENTS知識體系.md)。
- **護欄**：絕對不可做的事＋碰到就停下找人的事。
- **鐵律**：Agent 不能改自己的 System Prompt（人工 gate＋留版本）。
- **終止條件**：達標／上限 N 輪／沒進展／成本預算／求救人類（至少設兩個）。
