# 2026-05-07｜木酢平台從零到上線（3 天）+ AI Agent 升級到 Tool Use

## 事件
3 天內為双云客戶（木酢寵物達人）打造完整的「品牌指揮中心 + AI 顧問」平台並上線給團隊使用。
網址：https://muzopet-dashboard.zeabur.app

## 為什麼值得記

這次有兩個 AI 實踐上的突破：

### 突破一：Tool Use 取代 Context Injection

**先前認知**（v1）：「給 Claude 越多資料越好」→ 把所有 GA4/Meta/歷史數據塞進 system prompt。
**問題**：使用者問「這週 vs 上個月」時，Claude 答「我只有一週的資料」。
**現在認知**（v2）：定義 8 個工具（`get_ga4_summary` / `get_monthly_trend` / ...），讓 Claude 自己決定查什麼。

**關鍵啟發**：
- Anthropic Tool Use 不需要任何 SDK，原生 `fetch` 就能做（Node.js 24）
- Agentic loop = `while (response 含 tool_use) { 執行工具 → 結果回送 → 繼續 }`
- 同一回合可平行呼叫多個工具，比 sequential 快得多
- SSE 串流可以混合 tool_use + text，前端可顯示「🔍 查詢中」進度
- 5 個 round 的 MAX_ROUNDS 上限避免無限迴圈

### 突破二：「生成指令」按鈕——產品定位翻轉

**起點**：使用者觀察「我已經付月費給 Claude.ai Pro，為什麼還要再付平台的 API 費？」
**翻轉**：平台不該是「自己當 AI」，而是「**幫使用者把數據準備好**」，讓他們帶著結構化的 prompt 去用自己的 AI 工具。

**做法**：
- ➤ 按鈕：呼叫 Anthropic API 直接回答（消耗平台 API 費）
- 📋 按鈕：跑同樣的工具抓資料 → 組合 prompt → 彈出 modal → 一鍵複製

**結果**：平台 API 成本接近零，使用者用熟悉的介面，AI 答案品質取決於使用者自己付費的工具。

## 技術實踐紀錄

### Tool Use 程式碼骨架

```js
// 工具定義（用 Anthropic 標準 schema）
const AGENT_TOOLS = [{
  name: 'get_ga4_summary',
  description: '取得 GA4 官網近 N 天的 KPI...',
  input_schema: {
    type: 'object',
    properties: { days: { type: 'number', enum: [7, 14, 30, 90] } },
    required: ['days']
  }
}, ...];

// Agentic loop
async function streamAgentWithTools(res, systemPrompt, history, question) {
  const messages = [...history, { role: 'user', content: question }];
  for (let round = 0; round < 5; round++) {
    // 呼叫 Anthropic 串流 API（含 tools）
    const upstream = await fetch('https://api.anthropic.com/v1/messages', {
      body: JSON.stringify({ model, max_tokens, stream: true, system, tools: AGENT_TOOLS, messages })
    });
    // 解析 SSE，組裝 assistantContent[]，含 text 和 tool_use blocks
    // ...
    messages.push({ role: 'assistant', content: cleanContent });

    const toolUses = assistantContent.filter(b => b.type === 'tool_use');
    if (toolUses.length === 0) break;  // 沒呼叫工具 = 結束

    // 平行執行所有工具
    const toolResults = await Promise.all(toolUses.map(t => executeTool(t.name, t.input)));
    messages.push({ role: 'user', content: toolResults });
  }
}
```

### 採坑：`_raw` 欄位污染 messages

Streaming 時為了累積 tool input 的 partial JSON，我把字串存在 `tool_use._raw` 欄位上。
但發回 Anthropic 時沒清掉 → `Extra inputs are not permitted` 錯誤。
**修法**：發送前解構排除：`const { _raw, ...clean } = b;`

## 對 Jacky 工具觀的影響

### 過去
- AI Agent = 把資料給 Claude 然後讓他答
- 平台 = 包裝 AI 給使用者用

### 現在
- AI Agent = 給 Claude 工具，他自己判斷需要什麼
- 平台 = **幫使用者把零散數據變成結構化指令的中介層**

> 「平台不是 AI 的容器，是數據去 AI 之前的加工廠。」

這個視角影響後續所有客戶平台的設計：能讓使用者帶走的東西優先做，能省掉平台 API 費的功能優先做。
