# 採坑點——Gmail 量寄與跨客戶洩漏掃描

> 做活動後的批量跟進信，一路踩到的雷。每筆統一欄位，照著避。

## 採坑 #1：「看板沒資料」≠「沒發生」
- **症狀**：演講後看 CRM 看板與站台 leads buffer，幾乎看不到名單，差點對外回報「演講零轉換」。
- **誤區假設**：看板查不到 = 名單沒進來 = 演講沒成果。
- **真實原因**：73 個 lead 全都寫進磁碟 `case.json`，但建案順序是「寫檔 → git commit → 投影 read model」，而 `clients/**` 退版控後 commit 報錯 → `ok:false` → **檔案已落地卻從沒投影**。看板/buffer 是衍生層，壞了。
- **解法**：磁碟才是真相來源；用 `rebuildReadModel()` 從磁碟全量重建。git-store 對被 gitignore 的路徑用 `check-ignore` 優雅跳過。
- **預防**：下任何「沒成果」結論前，先問「是真的沒有，還是我查的那層（read model/buffer/快照）壞了？」先驗觀測管道。
- **發生於**：2026-06-13 演講成果盤點。

## 採坑 #2：空頭承諾——不問就說「我幫你撈」
- **症狀**：診斷跟進信寫「我會用明察為您的產業撈一份情報」，但根本沒問對方是什麼產業、什麼關鍵字。
- **誤區假設**：先承諾，細節之後再說。
- **真實原因**：明察撈標案是 `標案名稱 LIKE '%關鍵字%'`，沒有具體產品關鍵字根本撈不到；硬做就得編 → 踩反幻覺紅線。
- **解法**：把承諾改成「先問再做」——當場補建 `/diagnosis` 結構化問卷頁，收「產品關鍵字／是否做政府生意／想看哪種情報」等明察真正需要的輸入，再進產線。
- **預防**：寫任何「我幫你做 X」前先問「我手上有沒有做 X 的必要輸入與資料？」沒有就先收集。
- **發生於**：2026-06-15 審到診斷批。

## 採坑 #3：用 import「測試」有副作用的腳本 = 誤觸真寄
- **症狀**：想驗證寄信腳本的標頭剝除，跑了 `node -e 'import("./send-followup.mjs")'`，結果直接啟動真寄迴圈（每 2 分一封）。
- **誤區假設**：import 只是載入，不會執行。
- **真實原因**：ESM `import` 會執行模組**頂層程式碼**；該腳本頂層就是寄送主流程。
- **解法**：立刻 `pkill`；幸好當下洩漏掃描把全部擋下、**0 封真的寄出**。
- **預防**：驗證有副作用的腳本一律用 `--dry-run` 旗標，**絕不**用 import/require 去「測」。
- **發生於**：2026-06-15 建寄送器時。

## 採坑 #4：跨客戶洩漏掃描誤擋「自己人/場地/公開產品名」
- **症狀**：每封跟進信都被 leak scan 擋下，理由是「含其他客戶『鐘基啟』『彰師大』『Google』」。
- **誤區假設**：掃描器報的都是真外洩。
- **真實原因**：署名本人（鐘基啟，且某自家 case 的 name 欄就是它）、演講場地（彰師大，某 lead 的 name 欄）、信裡「Google Meet」撞到名叫 Google 的潛客——都是公開/自家字，不是他人機密。
- **解法**：在掃描器 STOP 白名單加入自己人/公開場地/公開平台名（鐘基啟/jacky/彰師大/google/meet…）。
- **預防**：加防呆前先問「這真的是別家客戶的機密，還是公開/自家的字？」白名單要含「自己」。
- **發生於**：2026-06-15 全批寄前掃描。

## 採坑 #5：同公司不同寫法被當成跨客戶
- **症狀**：寄給「陳景崧／宗珀股份有限公司」被擋，理由「含其他客戶『宗珀』」（另一位同事填的是「梁先生/宗珀」）。
- **誤區假設**：名字 token 完全相等才算同一人/同公司。
- **真實原因**：自己人名單存的是「宗珀股份有限公司」，別人填「宗珀」，字串不相等 → 誤判跨客戶。
- **解法**：比對 self 時，剝掉公司後綴（股份有限公司/有限公司/企業/實業/工業/科技…）取「公司核心」再比，同核心＝同公司。
- **預防**：中文公司名比對要正規化後綴；分詞正則也要含**全形**分隔符（`／｜`），否則「人名／公司」切不開。
- **發生於**：2026-06-15 全批寄前掃描。

## 採坑 #6：免費 Gmail 量寄會進垃圾黑名單
- **症狀**：打算一個早上連寄 66 封內容相近、都帶同網域連結的信。
- **誤區假設**：沒超過每日上限就沒事。
- **真實原因**：免費 Gmail 上限 500/天（66 遠低於沒問題），但**真正的雷是「突發爆量 + 無大量寄信歷史」**會觸發外寄垃圾啟發式，輕則進對方垃圾匣、重則帳號限流。Gmail 嚴格大量寄件者規範（SPF/DKIM/DMARC、List-Unsubscribe、投訴率<0.3%）門檻是 5,000/天，我們用不到，但 spike 風險仍在。
- **解法（事後證明不足）**：原本以為「分批跨天 30–40 封 + 間隔 2 分 + 內容客製」就夠——⚠️ **實測仍被擋（見 #7）**。免費 Gmail 連發相同主旨，第 17 封就觸發封鎖。分批間隔只降速度，沒解決「免費帳號 + 無寄件信譽」的本質。**真正解法＝改品牌網域（#7）。**
- **預防**：outreach 規模化從第一天就走有 SPF/DKIM/DMARC 的品牌網域，別用免費 Gmail 群發；免費 Gmail 只留 1:1 個人往來。
- **發生於**：2026-06-15 排程寄送前（J董喊停要求先查規範）。

## 採坑 #7：免費 Gmail 連發相同主旨，第 17 封就被 Google 擋下（support 69585）
- **症狀**：batch1 寄到第 17 封，Google 回 `Message blocked / Message rejected`（support 69585），最後 5 封全退；前 16 封寄達。
- **誤區假設**：分批 + 間隔 + 溫名單就安全了（#6 的緩解措施）。
- **真實原因**：免費 Gmail 帳號**無大量寄件信譽**，短時間連發**完全相同主旨**「接續 6/13 的課——為您安排下一步」累積到啟發式閾值就開鍘。本質是帳號信譽 + 重複指紋，不是寄信速度。
- **解法**：改走**品牌網域 Resend**——airun.tw 已驗 SPF/DKIM/DMARC，經部署站 `/api/admin/send-email` → Resend 從 `noreply@airun.tw` 寄。被擋的 5 封 + 後續全用此路，0 被擋。認證網域寄相同主旨是正常的（電子報天天做）。
- **預防**：任何「對外群發」一律走品牌認證網域；免費 Gmail 不做 outreach 群發。
- **發生於**：2026-06-15 batch1 寄送。

## 採坑 #8：寄送腳本回「成功」≠ 真的寄達（退信是非同步的）
- **症狀**：寄送 log 整排 `✓ 已寄`，看起來全成功；J董卻發現信在退。
- **誤區假設**：寄送程式回 exit 0 / message_id = 對方收到了。
- **真實原因**：`gog`/SMTP 回成功只代表「Gmail 接受了送出請求」；被擋的退信是**事後非同步**回到寄件匣，寄送當下的 log 看不到。
- **解法**：判斷「真寄達」要查寄件帳號收件匣 `from:mailer-daemon`／`subject:(Delivery Status Notification)`；被擋的執行緒會是 `[2 msgs]`（原信 + 退信）。
- **預防**：群發後例行掃一次退信，把「寄送器回成功數」和「實際無退信數」分開看，別只信寄送 log。
- **發生於**：2026-06-15 J董發現信在退、要求暫停。

## 採坑 #9：品牌信回覆要回正確信箱，靠 `reply_to`，不是「Resend 帳號用誰申請」
- **症狀**：擔心「Resend 不是用 J董帳號申請，回信會不會回不到 J董」，想重新申請 Resend。
- **誤區假設**：回信去哪，由 Resend 帳號 owner 決定。
- **真實原因**：回信去哪**只由信的 `reply_to` 標頭決定**，跟 Resend 帳號無關；重申請是白繞。而且 airun.tw **沒有 MX**，`from:noreply@airun.tw` 的回覆無處可去；寄信函式預設 `reply_to` 還是另一個信箱（双云），客戶回覆會跑錯地方、CRM 收不到。
- **解法**：寄品牌信務必帶 `replyTo=agentflowjdong@gmail.com`（CRM mail-poll 在看的信箱），回覆才進跟進流程。`sendEmail` 本已支援 replyTo，admin 端點補透傳即可，不必動 Resend 帳號。
- **預防**：上線品牌寄信前，先寄一封給自己、確認收到信的「回覆地址」真的是你要的那個。
- **發生於**：2026-06-15 切品牌網域時（J董點出回信去向）。

## 採坑 #10：降主旨重複，用「分眾 + 輪換句型」，不要塞收件人名字
- **症狀**：同主旨 26 封被擋後，直覺想「把每封主旨加上對方稱謂，保證不一樣」。
- **誤區假設**：主旨塞名字＝唯一＝最安全。
- **真實原因**：收件人名字進主旨是 mail-merge／垃圾信的典型指紋，過濾器會扣分，收件人一看也知是群發，反降開信率。「我是誰（品牌）」進主旨才加分。
- **解法**：按收件人**勾選的需求分眾**（明腦內訓／明察診斷／學院／諮詢）配主旨池、每段**輪換 2–3 句型**，皆帶品牌「AIRUN」、對齊本文重點——自然不重複又更相關（26 封同主旨 → 18 種、最大重複 3）。
- **預防**：主旨變化走「分眾 + 句型輪換 + 品牌識別」，別走「機械式塞個資」。
- **發生於**：2026-06-15 規劃剩餘 45 封寄送策略。
