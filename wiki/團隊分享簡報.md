# jacky-wiki

## 從個人知識庫到團隊 AI 大腦

2026-06-25 最新狀態

---

## 一句話

jacky-wiki 不是單純筆記庫。

它現在是 Jacky 的「AI 作業系統知識層」：

- 內部：整理 Jacky 的方法論、案例、流程、工具與實踐事件
- 外部：透過 AI Brain API 與 MCP，讓團隊在 Claude Code / Codex 直接查詢
- 管理：用 key、tier、topic 控制誰能查到什麼

---

## 這套系統解決什麼問題？

| 舊狀態 | 新狀態 |
|---|---|
| 知識散在腦中、LINE、簡報、課程、專案資料夾 | 統一沉澱到 wiki |
| AI 每次都要重新餵背景 | AI 可查 Jacky wiki |
| 團隊要問 Jacky 才知道脈絡 | 團隊可用 API / MCP 查授權知識 |
| 知識分享容易變成整包外洩 | 只交付答案包，不開原始庫 |
| 權限靠人工判斷 | key 對應 tier/topic 自動控管 |

## AI 大腦：最新狀態

目前線上服務：

```txt
https://jacky-aibrain.zeabur.app
```

有效端點：

```txt
GET  /health
POST /order
```

最新 build：

```txt
filterfix-v3
```

根路徑 `/` 回 404 是正常的，因為這不是網頁前端，是 API-only 服務。

---

## /order 已修好的問題

原本成員查詢一直得到：

```json
{"fulfilled": false}
```

已排除並修正：

- 歷史 thread 污染 -> 每次 fresh thread
- Zeabur CLI TLS 攔截 -> 改走 GitHub 自動部署
- Zeabur root dir 錯誤 -> 指到 `services/fulfillment`
- filter 誤殺答案 -> 改用 `NO_MATCH` sentinel

目前同題已可回：

```json
{"ok": true, "fulfilled": true, "tier": "L3"}
```

## AI 大腦的供應鏈

```txt
jacky-wiki
  -> publisher 分級發佈
  -> AnythingLLM workspace
  -> fulfillment API
  -> API key 授權路由
  -> answer package / fulfilled:false
  -> MCP tool
  -> Claude Code / Codex
```

這不是「把 wiki 公開」。

這是「把 wiki 包成受控的知識服務」。

---

## 權限控管邏輯

```txt
x-api-key
  -> member
  -> tier
  -> topics
  -> allowed workspaces
  -> answer package
```

權限不是只看登入。

而是同時看：

- 這個人是誰
- 他是哪個 tier
- 他能查哪些 topic
- 對應 workspace 是否有命中

---

## Workspace 分層

示意：

```txt
shuangyun-l1
shuangyun-l2
shuangyun-l3

education-l1
education-l2
education-l3

cross-domain-l1
cross-domain-l2
cross-domain-l3
```

publisher 決定內容進哪裡。

fulfillment API 決定 member key 能查哪裡。

## 為什麼不用前端網頁？

這次需求不是「讓人打開網站查」。

真正需求是：

> 讓團隊在自己的 Claude Code / Codex 介面裡，直接把 Jacky wiki 當工具使用。

所以正確解法不是做 UI。

正確解法是 MCP。

## MCP 接入

新增工具包：

```txt
tools/jacky-ai-brain-mcp/
```

提供工具：

```txt
ask_jacky_wiki(question)
```

內部呼叫：

```txt
POST https://jacky-aibrain.zeabur.app/order
Header: x-api-key = JACKY_AI_BRAIN_KEY
```

key 只放在成員本機環境變數，不進 repo、不貼對話。

---

## 團隊使用方式

在 Claude Code / Codex 裡問：

```txt
請用 ask_jacky_wiki 查詢：
請說明双云 AI Agent 課程拆建修串管交六步框架分別在教什麼。
```

AI 會在背後呼叫 MCP tool。

成員不用打 curl。

成員也不需要知道 AnythingLLM、workspace、publisher 的細節。

## 團隊專案建議指令

可以放進成員專案的 `AGENTS.md` / `CLAUDE.md`：

```md
當問題涉及双云、AI Agent 課程、SOSTAC、TBSA、
Jacky wiki 或內部知識時，優先使用 MCP 工具
`ask_jacky_wiki` 查詢。

若工具回傳 fulfilled:true，以工具回傳的知識包回答。
若 fulfilled:false，不要編造；說明知識庫沒有命中，
需要轉人工或補資料。
不要要求使用者提供或顯示 API key。
```

## 安全原則

| 原則 | 做法 |
|---|---|
| 不開原始庫 | 不提供 raw browse/list |
| 不暴露 admin | AnythingLLM admin key 不進 repo |
| 不共用 key | 每位成員一組 `jaibrain_...` |
| 可撤銷 | key 可 rotate / revoke |
| 可審計 | orders.log 記錄查詢摘要 |
| fail closed | 無 key、錯 key、無授權都不回內容 |

---

## 這次學到的部署坑

- PowerShell 直接塞中文 JSON 容易壞，測 API 用 UTF-8 檔案 + `--data-binary @file`
- Zeabur CLI 在 TLS 攔截環境會 x509 失敗，改走 GitHub 自動部署
- Zeabur root dir 設錯會變 Caddy 靜態站，不是 Node API
- RAG 無命中不要靠自然語言判斷，應用 `NO_MATCH` sentinel
- MCP tool 不要把 API key 設成 tool argument
