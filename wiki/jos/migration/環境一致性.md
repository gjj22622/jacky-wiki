# 環境一致性比對（多機）

> Jacky 有多台機器（Windows 主機 / G2 mini Ubuntu / WSL2）。它們的工具、skill、設定**可能不一致**。
> 本頁負責回答：**兩邊環境一樣嗎？哪裡不一樣？怎麼補齊？**

---

## 怎麼做一致性比對（流程）

```
[在每一台機器]
1. /移機 capture        # 各機把自己狀態寫進 manifest 對應區段，push
[在任一台]
2. git pull --rebase    # 收齊所有機器的 capture
3. /移機 diff           # 比對各機區段，列出差異
```

> 關鍵：`capture` **只寫自己那段、不覆蓋別台**（見 [manifest](manifest.md) 多機規則）。所以兩台各跑一次 capture + push，資料才會齊。

## 預期會不一致的地方（常見）

- **新建的 skill**：在 A 機建的 skill，B 機要 `git pull` + 裝執行版才有。
- **CLI 子版號**：各機更新節奏不同，小版號常不同（多半無害）。
- **plugin**：A 機裝的 plugin，B 機要 `/plugin` 重裝。
- **WSL2 vs Windows 主機**：兩個 `~/.claude/` 是分開的，skill 要兩邊各裝。
- **自建程式**：`~/bin/`、`settings.json` 不進 git，各機自管。
