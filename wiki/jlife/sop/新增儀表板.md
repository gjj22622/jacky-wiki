# 新增儀表板 SOP（含 Zeabur 部署）

> 從擷取貼文到線上部署的完整流程。涵蓋本機建置 + GitHub 同步 + Zeabur 自動部署。
> 來源：`CLAUDE.md`、`travel-longstay-dashboard.skill`、`同步到線上.bat`

---

## 完整流程概覽

```
本機建置（3 步驟）              GitHub 同步              Zeabur 部署
────────────                  ─────────                ──────────
Step 1: extract                同步到線上.bat            自動觸發
Step 2: day_meta               ├ git pull               build + deploy
Step 3: build_dashboard        ├ robocopy /XO 增量      （3-5 min）
                              ├ 變更摘要
                              └ git push
```

## 本機建置（3 步驟）

### Step 0：先讀 Skill SKILL.md

> ⚠️ **2026-03-22 教訓**：開始任何儀表板任務前，**先讀 `skills/travel-longstay-dashboard/SKILL.md` 再動手**，不要從零造輪子。

### Step 1: extract — 從 Facebook 擷取貼文

```bash
python scripts/extract_trip_posts.py \
  --start-date YYYY-MM-DD \
  --end-date YYYY-MM-DD \
  --semantic-filter
```

**`--semantic-filter` 的作用**：自動排除時間段內跟主題無關的貼文（純工作、投資、轉發連結）。

**輸出**：`data.json`（擷取的貼文資料）

### Step 2: day_meta — 整理每日摘要

人工或 AI 整理 `day_meta.json`：

```json
{
  "2023-08-15": {
    "summary": "抵達鳴子町，第一天適應日式溫泉旅館",
    "cover_photo": "20230815_001.jpg",
    "coordinates": { "latitude": 38.7, "longitude": 140.6 }
  }
}
```

### Step 3: build_dashboard — 產生 index.html

```bash
python scripts/build_dashboard.py \
  --start-date YYYY-MM-DD
```

**`--start-date` 的作用**：控制 Day 1 = 旅行起始日（即使某天沒有貼文也會跳號）。

**輸出**：`index.html`（三頁籤儀表板）

## 部署到 Zeabur SOP

> 觸發條件：當 Jacky 說「**幫我部署**」。

### Step 1：確認本機儀表板已就緒

檢查 `人物儀表板/` 和 `旅行儀表板/` 資料夾內有新增或修改的內容。

### Step 2：若有新增儀表板，更新 app.py 的 META

#### 新增旅行儀表板

在 GitHub 上編輯 `app.py`，在 `TRIP_META` 列表加入：

```python
{'folder': '資料夾名', 'title': '標題', 'tag': '標籤', 'days': '天數', 'photos': '照片數'},
```

#### 新增人物儀表板

在 `PEOPLE_META` 列表加入：

```python
{'folder': '資料夾名', 'name': '顯示名', 'meta': '描述文字'},
```

#### 編輯方式

透過瀏覽器在 `https://github.com/gjj22622/jacky-memoir/edit/main/app.py` 修改，直接 commit 到 main。

可用 JavaScript 存取 CodeMirror 編輯器：

```javascript
const view = document.querySelector('.cm-content').cmTile.view;
const doc = view.state.doc.toString();  // 讀取
view.dispatch({ changes: { from: idx, to: idx + old.length, insert: newText } });  // 替換
```

### Step 3：執行「同步到線上.bat」

請 Jacky 在 Windows 檔案總管雙擊 `同步到線上.bat`。腳本會：

1. `git pull` 拉取最新版本（含 Step 2 的 app.py 修改）
2. `robocopy /XO` 增量同步（**只複製新增或修改過的檔案**）
3. 顯示變更摘要（哪些資料夾有更新、共幾個檔案）
4. 詢問 `Y/N` 確認後 `git push`

> 如果電腦沒有 clone 過 repo，腳本會**自動 clone 到 `%USERPROFILE%\Desktop\jacky-memoir`**。

#### 首次使用需設定 git 身分

```cmd
git config --global user.email "gjj22622@gmail.com"
git config --global user.name "Jacky"
```

### Step 4：確認部署完成

push 完成後**等 3-5 分鐘**，到 [jacky-memoir.zeabur.app](https://jacky-memoir.zeabur.app) 確認。

若超過 5 分鐘仍未更新：

1. 到 Zeabur 後台 → `jacky-mem` → 服務狀態 → 點「重新部署」
2. 或檢查 Zeabur 後台的 build log 是否有錯誤

## GitHub repo 結構

```
jacky-memoir/
├── app.py                 ← Flask 主程式（TRIP_META、PEOPLE_META、路由）
├── requirements.txt       ← Flask + markdown2
├── zbpack.json            ← Zeabur 設定
├── templates/
│   └── home.html          ← 首頁模板（卷、旅行、人物三區）
├── content/
│   └── volumes/           ← 書稿 .md 檔
└── dashboards/
    ├── travel/            ← 旅行儀表板
    └── people/            ← 人物儀表板
```

### app.py 關鍵路由

| 路由 | 功能 |
|---|---|
| `/` | 首頁，讀取 TRIP_META + PEOPLE_META 渲染卡片 |
| `/travel/<folder>/` | 旅行儀表板入口 |
| `/travel/<folder>/<path>` | 旅行靜態檔案 |
| `/people/<folder>/` | 人物儀表板入口 |
| `/people/<folder>/<path>` | 人物靜態檔案 |

## 已知陷阱（10 條）

> **新增儀表板或修改前端時務必檢查以下項目，避免重蹈覆轍。**

| # | 陷阱 | 原因 / 解法 |
|:--:|---|---|
| 1 | 切換版本（D/E）時不重新 fetch 資料 | JS 必須重新向 API 請求；不能只切換變數就 re-render 舊陣列。**2026-03-19 E 版載入失敗的根因** |
| 2 | 解析檔名的正則只認數字開頭 | D 版 `01_`、E 版 `E01_`。正則用 `^[A-Za-z]?\d{1,2}` 兼容兩種 |
| 3 | 改 serve.py 後沒重啟 | Python 後端不會熱更新 |
| 4 | 二女兒鐘心云出生時間標錯 | 正確：**2016-01**（不是 2019）。博士班入學是 **2025**。寫書稿時務必核實 |
| 5 | 書稿圖片用相對路徑 | Facebook 照片分散 `_01`、`_02` 兩目錄。**必須用 `/api/media/`**，會自動搜尋所有目錄 |
| 6 | JSON 匯出 encoding 沒修復 | UTF-8 中文以 Latin-1 byte 存進 JSON。**所有讀取程式都必須加 `fix_fb_encoding()`** |
| 7 | album JSON 結構當 posts 解析 | album 是 `{name, photos: [{uri, creation_timestamp}]}`。用 `creation_timestamp` 而非 `timestamp`，用 `uri` 而非 `attachments.data.media.uri` |
| 8 | 儀表板用 client-side JS 動態渲染 | **HTML 必須靜態渲染**，所有卡片直接寫在 HTML。照片保留 Facebook **原始檔名**，不要重新命名。**2026-03-22 林小眼儀表板顯示異常的根因** |
| 9 | 儀表板放錯資料夾 | 不論主題（旅行 / 品牌 / 人物），統一放 `旅行儀表板/`。**不要在專案根目錄另建資料夾** |
| 10 | 從零造輪子 | 開始任何任務前，先檢查 `available_skills`。有對應 Skill 就先讀 SKILL.md 再動手。**2026-03-22 犯過的錯** |

## 注意事項摘要

| 項目 | 規則 |
|---|---|
| Cowork VM 無法 git push | VM 沒有 GitHub 認證，所有 push 必須由 Jacky 在 Windows 上執行 |
| GitHub 網頁編輯器可改 app.py | 小改動（如新增 META entry）可直接在瀏覽器完成 |
| 大量檔案（照片）只能用 git push | GitHub 網頁上傳限制 ~100 檔案 |
| base64 縮圖 fallback | 若儀表板沒有 `thumbs/` 資料夾，可在 META 加 `cover_uri` 欄位放 base64 data URI |
| 自動部署可能不觸發 | push 後 5+ 分鐘沒更新，手動到 Zeabur 後台點「重新部署」 |
| 首次 push 600MB+ 大 repo 很慢 | 正常，等它跑完就好 |
