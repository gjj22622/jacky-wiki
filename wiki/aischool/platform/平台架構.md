# AI School 平台架構

> repo `gjj22622/shuangyun-claude-code-training`｜線上 `2clouds-claude-code-training.zeabur.app`｜TypeScript · Node 24 · SSR+vanilla JS · Zeabur（Docker）

## 五個權限分區（zones）

平台依角色開放不同分區，伺服器端依角色移除無權的 `<section data-zone>`：

| zone | 名稱 | 對象 |
|---|---|---|
| `ai-os` | AI 作業系統 | 全員（四層架構、搭建環境、換電腦轉移） |
| `course` | 影片課程 | 全員（L0–L6 共 42 支，見[課程地圖](../curriculum/課程地圖.md)） |
| `tbsa` | TBSA 種子教師 | TBSA 線 |
| `newcomer` | 新人專區 | 新人 |
| `core` | 核心成員 | 核心團隊（含 suite 工具軌 / agents / token / dev-env / map / workflow / deploy 等子段） |

## token 後台

- **發 token**：本機 `npm run issue-token -- --label=X --days=N`；或部署後 `POST /api/admin/tokens`（需 `ADMIN_SECRET`）
- **安全**：token 明碼只在發行時出現一次；伺服器只存 hash；`data/tokens.json` 已 gitignore；session 用 HttpOnly cookie
- **持久化**：Zeabur 硬碟 ID `data` 掛 `/data`，`DATA_DIR=/data`，token 檔 `/data/tokens.json`
- 詳細權限管理設計法見 `/admin後台` skill（兩層權限／去個資化行為追蹤／四來源成本歸戶）

### 環境變數

| 變數 | 必填 | 用途 |
|---|---|---|
| `ADMIN_SECRET` | 是 | 管理者發 token 密鑰（`.zeabur-secrets.local.txt`，gitignore） |
| `SESSION_SECRET` | 是 | 簽署登入 session cookie |
| `BOOTSTRAP_TOKENS` | 否 | 首批登入 token（逗號分隔） |
| `DATA_DIR` | 否 | token 儲存目錄，Zeabur 設 `/data` |
| `PORT` | 否 | 本機 3000，Zeabur 自動注入 |

## 影片管線

- **單一真相**：`course.json`（產生器 `scripts/build-course-json.js`，母源 `双云AI_School/01_影片課程` 底線版，未進 git）
- **串流**：`/api/course` ＋ `/videos` Range 串流；影片在 Zeabur `/data/videos` 持久碟
- **對齊鐵則**：講稿「一段 `#` 對一頁」才能真對齊；`_realign_all.sh` 標準重出片（音畫等長差 0s）
- **配音克隆**：`swap-clone-voice.sh <單元> <克隆.mp3>`（冪等、`--revert`）逐單元換 Jacky 克隆聲音；工具在 `C:\AI\voxcpm2-voice-cloner`（CPU 版）

## 部署流程

1. push 到 GitHub repo
2. 本機 `docker build` + `docker run` 跑過一次
3. Zeabur 建 service 接 repo，讀 `Dockerfile`，start command `npm start`
4. 設 `ADMIN_SECRET` / `SESSION_SECRET` / `DATA_DIR=/data`
5. 掛硬碟到 `/data`，重新部署一次確認 token 仍在

> **上線手法（本輪實證安全）**：dev 提交 → cherry-pick 單一 commit 到 main → push，避開分支分歧互蓋。
