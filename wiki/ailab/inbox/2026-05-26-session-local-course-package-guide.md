## 最終做法（What Worked）

- 把整包教材設計成「下載 zip → 解壓縮 → 打開根目錄 index.html」的本機教材包，而不是讓使用者在檔案總管裡找檔案。
- `index.html` 不只是檔案清單，而是任務導覽：快速入口、五張表單流程地圖、角色導覽、六份必讀、5/26 分享包、資料夾地圖。
- 文件型檔案不直接連檔案，改連所在資料夾，並用 `data-file` 顯示要找的檔名。這解決瀏覽器把包內 Word、PPT、Excel、Markdown 重複下載的問題。
- HTML 頁面保留直接連結，例如學生導覽頁、助教工具、課後問卷儀表板；音訊用 `<audio>` 原生播放器嵌入。
- 導覽頁音訊放在六格快速入口上方，文字稿與音訊分離：文字稿在 `00_工作坊總覽/導覽頁導覽口稿.md`，音訊在 `00_工作坊總覽/audio_導覽頁導覽口稿/導覽頁導覽口稿_完整朗讀.mp3`。
- 建立「打包排除區」作為非破壞式精簡策略：正式包只放使用者需要的檔案，建置腳本、備份、抽文字暫存、Jacky 自用口稿、音訊製作中間檔移到同層 `_打包排除_...`，不直接刪除。
- 最後做連結檢查：4 個 HTML、57 個相對連結與媒體來源、失效路徑 0、直接連到非 HTML 文件檔 0、音訊來源 1。

## 繞路紀錄（Detours）

- 一開始導覽頁直接連到 `.md`、`.docx`、`.pptx`、`.xlsx`。後來發現本機瀏覽器可能把檔案再下載一份，導致解壓包內出現重複檔案，因此改成資料夾型連結。
- 一開始把「台中科大空白範本」當成空白工作區入口。後來讀取 docx 文字後確認它是 `TBSA商務企劃能力進階檢定` 空白檔案，不是台中科大空白範本，也不是學生共編工作區，因此修正標籤並另建真正的工作區空白範本。
- 一開始口稿音檔與建置腳本都在正式包內。後來依打包邏輯，將 Jacky 自用口稿、口稿音檔、建置腳本與暫存檔移到排除區；但導覽頁音訊是使用者需要的，所以只保留導覽音訊 mp3。
- 音訊播放器第一次放在主內容區，後來移到六格快速入口上方，符合使用者「先聽導覽再操作」的流程。
- RWD 原本只有基本 grid 斷點，最後補上手機專用覆寫：音訊、快速入口、流程圖、角色卡、資料夾地圖全部可單欄閱讀。

## 錯誤與失敗（What Failed）

- PowerShell 讀寫 UTF-8 時曾造成中文亂碼，後續以明確 UTF-8 讀寫與檢查修正。教訓：Windows PowerShell 處理中文 HTML/Markdown 時要特別注意 encoding。
- 曾用文字模糊搜尋判斷排除區是否被引用，結果會把普通文字提及也算成引用。後來改用實際 HTML `href/src` 解析，才是判斷打包後是否失效的正確標準。
- `todolist.md` 一度被移到排除區，後來收工時重新寫回正式工作目錄，確保下次 `/開工` 能讀到正確進度。
- 瀏覽器不允許本機 HTML 直接呼叫 Windows 檔案總管或 Mac Finder，因此「點一下直接打開資料夾視窗」不可作為通用設計，只能用資料夾連結加檔名提示作為跨平台折衷。

## 升格候選

- 值得升格為 ailab pattern：`本機教材包導覽頁交付模式`。核心是「入口 HTML + 相對路徑 + 文件型連結改資料夾型 + 音訊導覽 + 排除區 + RWD + 打包前連結檢查」。
- 值得升格為 cross-domain 或 internal-training SOP：`可下載教材包交付檢查清單`，可用於 TBSA、双云內訓、Jwood 教材包、博士班專題教材包。
- 只留 inbox 的細節：本次特定檔名、台中科大資料夾路徑、5/26 種子教師分享檔名，屬於個案脈絡。

## 待延伸（Next）

- 在 Mac 實機測試：Safari/Chrome 開 `index.html` 是否能讀中文相對路徑、資料夾連結是否可用。
- 壓縮正式 zip 後，解壓到另一個路徑測試 `index.html`，確定相對路徑不依賴原 OneDrive 位置。
- 若下載對象包含外部學員，需確認台中科大學生實作檔是否應去識別或移出正式包。
- 可將這套模式抽象成通用模板：`教材包 index.html 生成器` 或 `downloadable-course-pack checklist`。
