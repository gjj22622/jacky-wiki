# 双云 AI 大腦 — 核心成員使用說明書

> 歡迎使用双云 AI 知識大腦。這份說明帶你 5 分鐘上手。

---

## 1. 這是什麼？

它是双云累積的**知識與判斷邏輯的大腦**。你**問它一個問題**，它回你一份為你量身提取的包裹，內含：

- **【對應知識】**：針對你問題的重點答案。
- **【判斷邏輯】**：「如果 A 則 B」式的決策規則 + 何時該回報人類。

你不是在「搜尋資料庫」，而是在**下一張訂單**——大腦進倉庫幫你提取、合成、送達。

---

## 2. 心智模型：下訂單，不是逛倉庫

```
你的問題（訂單） → 大腦提取對應知識+判斷邏輯（履約） → 回你一份包裹（送達）
```

- 你**看不到**整個倉庫，也不能瀏覽或下載原文——只會收到為這個問題提取的答案。這是刻意的設計（保護知識資產）。
- 你能提取到多深，由你的**層級**（L1/L2/L3）和**授權主題**決定。問超出你範圍的東西，會回「倉庫中無此資訊」——**這不是壞掉，是權限邊界**。

---

## 3. 怎麼用（API）

你會拿到一把**專屬 api_key**（管理員私下給你，請妥善保管）。

### 端點
```
POST https://jacky-aibrain.zeabur.app/order
Header:  x-api-key: <你的key>
Header:  Content-Type: application/json
Body:    {"question": "你的問題"}
```

### curl 範例
```bash
curl -X POST https://jacky-aibrain.zeabur.app/order \
  -H "x-api-key: 你的key" \
  -H "Content-Type: application/json" \
  -d '{"question":"賣點要怎麼命名才有記憶點？"}'
```

> Windows PowerShell 提醒：中文問題請把 body 存成 UTF-8 檔案再用 `--data-binary "@檔名"` 送，直接打字串可能變亂碼。

### 回應格式
```json
{
  "ok": true,
  "fulfilled": true,
  "tier": "L2",
  "package": "【對應知識】…\n\n【判斷邏輯】…"
}
```
- `fulfilled: true` → `package` 就是你的包裹。
- `fulfilled: false` → 「倉庫中無此資訊（或不在你的授權範圍）」：你的層級/主題沒有這塊，或大腦確實沒這內容。

### 在程式裡用
任何能發 HTTP POST 的語言都行（Python `requests`、JS `fetch`、n8n、Make…）。把 `x-api-key` 放 header、問題放 `question` 即可。

## 4. 使用技巧（問得好，答得準）

1. **問具體、單一主題**：「演講後跟進信怎麼分批寄、避免被 Gmail 擋？」比「行銷怎麼做？」好太多。
2. **想要決策規則就明說**：問題帶「…的判斷標準／什麼時候該 X／如果遇到 Y 怎麼辦」，會逼出更扎實的【判斷邏輯】。
3. **一次一個問題**：把複合問題拆開分別下單，每份包裹更聚焦。
4. **0 命中時換個問法**：可能是用詞沒對上。換關鍵字、講具體場景再試一次。
5. **它只根據倉庫內容回答**：不會幫你瞎掰；答不出來它會老實說「無此資訊」或「回報人類」——這是優點，代表答案可信。

---

## 5. 你會遇到的情況

| 情況 | 意思 / 怎麼辦 |
|---|---|
| `fulfilled: false` 倉庫中無此資訊 | 不在你的授權範圍，或大腦沒這內容。換問法；真的需要就找管理員。|
| `401 unauthorized` | key 錯了或沒帶。檢查 `x-api-key`。|
| `429 rate_limited` | 太快了（每分鐘上限 30 張）。稍等再送。|
| 答案不夠深 | 可能你的層級到這。需要更深內容找管理員談層級。|

---

## 6. 紅線（請務必遵守）

- **你的 api_key 等於你的身分，絕不外流、不共用、不貼到公開地方。**
- 懷疑 key 外洩 → 立刻通知管理員換新 key。
- 大腦的內容是双云知識資產，**包裹內容請依你的職務合理使用，勿外傳客戶或第三方**。

- 覺得某類問題常 0 命中、希望補進大腦 → 回報管理員，可安排入庫。
