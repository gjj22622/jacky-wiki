# Week 5｜通 — 腦跟手「串」起來，打造你的 AI 行銷部

> **時長**：170 分鐘（含四週回顧）
> **使用版本**：v3（取代 v2 / 廢棄 v1 API 技術版）
> **核心語言**：絲滑 / 摩擦力 / 串 / AI 行銷部
> 來源：`02_課程架構與教案/05_Week5_課程架構_通_AI行銷部_v2.md`（內容版本標為 v3）

> **v3 設計原則**：移除技術性 API 內容。「API」這個詞不在課堂出現，取而代之的是：**串、絲滑、摩擦力、腦跟手**。
> 「串」= 程式跟程式間的絲滑連結，**不是技術概念，是流程設計哲學**。
>
> 廢棄的 v1（API 串接版）本來要教 `shuangyun-api-gateway` 路由表 + 現場跑 `content.generate`，但決策上不在課堂帶夥伴看技術。技術內容轉為 Jacky 後台維護，課堂主軸換成「老闆 1-4 / AI 5-8」的行銷部模型。

---

## 一句話定位

> Week 1-4，我們學會了打造 Agent 工廠。
> Week 5-8，我們要把這個工廠，變成客戶的 **AI 行銷部**。

| | Agent 工廠 | AI 行銷部 |
|---|---|---|
| 主角 | 夥伴操作 Agent | 老闆說方向，AI 接手 |
| 交付 | 一篇貼文、一份企劃 | 整個行銷部的運作 |
| 感受 | 工具 | 部門 |
| 終點 | 做得出來 | 老闆不需要管細節 |

---

## 課程目標

1. **說清楚** 自己手上品牌的「摩擦點」
2. **理解** AI 行銷部運作模型：老闆做 1-4，AI 行銷部做 5-8
3. **看懂** 腦跟手「串」起來長什麼樣（現場 demo）
4. **設計** 自己品牌的第一條「串」
5. **開始** 幫手上第一個品牌建立 AI 行銷部（課後作業）

---

## Week 5-8 總覽

```
Week 5「通」→ 找摩擦力，設計第一條串
Week 6「控」→ 4 Agent 委員會上線，品管自動跑
Week 7「開」→ Onboarding 真實客戶，Form 串接 live
Week 8「營」→ 驗收：老闆一個指令，AI 行銷部全自動接手
```

**畢業目標**：每個夥伴手上所有品牌，都有一個可以運作的 AI 行銷部。

## 模組 1｜摩擦力診斷（20 min）

### 核心提問

> **「你現在手上最痛的品牌，腦跟手之間最大的摩擦點是什麼？」**

不要說 AI，不要說工具。說「最痛的那一件事」。

### 四個摩擦力來源（白板四格）

| # | 摩擦點 | 常見症狀 |
|---|---|---|
| ① | 老闆意志傳達不準 | 說「有質感」→ 出土味；說「不要賣貨」→ 每篇都在促銷 |
| ② | 策略與執行脫節 | 月曆定好了 → 執行沒看；KPI 衝 ROAS → 貼文沒行動 |
| ③ | 執行細節被遺忘 | 客戶說不要用「！！！」→ 每篇都有 |
| ④ | 品牌一致性崩潰 | 換執行人 → 品牌感覺整個跑掉 |

> 詳細定義 → [AI行銷部定義.md Part A-B](../concepts/AI行銷部定義.md)

## 模組 3｜AI 行銷部模型（15 min）

### 老闆 1-4 / AI 5-8 鐵律

```
老闆做的（不可被取代）：           AI 行銷部做的（自動接手）：
1. 找問題                         5. 想策略
2. 定邊界                         6. 排執行
3. 設目標                         7. 自品管
4. 給方向                         8. 學回饋
```

> **鐵律**：老闆只輸入 1-4，不問「你喜歡哪個版本」。AI 行銷部只做 5-8，不打擾老闆確認每一篇貼文。
>
> 詳見 [AI行銷部定義.md Part C](../concepts/AI行銷部定義.md)

### 4 Agent 委員會 — 一個 Agent 消除一個摩擦

| Agent | 消除的摩擦力 | 守護的邊界 |
|---|---|---|
| 🔴 老闆 Agent | 老闆意志傳達不準 | 品牌願景 / 絕對禁忌 / 重大決策否決 |
| 🟡 主管 Agent | 策略與執行脫節 | KPI 方向 / 策略一致性 / 行銷計畫對齊 |
| 🟢 窗口 Agent | 執行細節被遺忘 | 客戶偏好 / 語氣禁忌 / 溝通習慣 |
| 🔵 品牌 Agent | 品牌一致性崩潰 | CI 規範 / 視覺語調 / 核准例句庫 |

### 「串」是什麼

> 「串」= 程式跟程式之間的絲滑連結。**哪個腦的輸出，要自動成為哪個手的輸入。**

```
老闆說一句話
    ↓ 串
主管 Agent 產策略方向
    ↓ 串
內容 Agent 產貼文
    ↓ 串
4 Agent 委員會審核
    ↓ 串
寫入品牌資料庫（Google Form）
```

**有摩擦** = 中間有一條串斷掉，要人手動補。
**絲滑** = 每一條串都自動流，老闆看結果就好。

## 模組 4｜現場 Demo（25 min）

**情境**：潘潘的保健品品牌，老闆說：「這個月我們的主打是維他命 C，要讓媽媽覺得給家人買最合適。」

| 對比 | 步驟 |
|---|---|
| 傳統做法（有摩擦） | 自己想策略 → 給老闆確認 → 改稿 → 合規再看 → 整個過程 3 天 |
| 今天做法（絲滑） | 一句話觸發，行銷部全自動接手 |

**Demo 平台**：`shuangyun-agent-platform.zeabur.app`

| Step | 動作 | 投影展示 |
|---|---|---|
| 1 | 老闆輸入方向 | 觸發欄位輸入 |
| 2 | 老闆 Agent 確認邊界 + 主管 Agent 轉策略包 | 策略包 JSON 結構 |
| 3 | 內容 Agent 產 FB 貼文 | 完整貼文 |
| 4 | 保健合規 Sub Agent 檢查 | 合規報告 |
| 5 | 4 Agent 委員會會審 | 四個分數 + 綜合判決 PASS |
| 6 | 自動寫入 Google Form | 品牌資料庫已更新 |

### 觀察重點

1. **老闆只說了一件事** → 行銷部自動完成 5 件事
2. **每個 Agent 只消除一個摩擦** → 缺一個就會有地方卡住
3. **串的威力** → 每個輸出自動成為下一個的輸入

## 模組 5｜實作：設計你的第一條串（35 min）

### 任務

為手上一個品牌設計「第一條串」。**不是寫程式，是畫流程。**

### 設計圖模板

```
┌─────────────────────────────────────────────┐
│ 品牌：______   最大摩擦點：① ② ③ ④           │
├─────────────────────────────────────────────┤
│ 老闆說什麼？（一句話）                          │
│ ___________________________________________│
│                                             │
│  ↓ 第一條串                                  │
│ [腦：______] 負責把老闆的話轉成 ______          │
│                                             │
│  ↓ 第二條串                                  │
│ [手：______] 負責產出 ______                  │
│                                             │
│  ↓ 第三條串                                  │
│ [品管：______] 負責確認 ______                │
│                                             │
│  ↓ 落地                                     │
│ 最終產出寫到：______（Form / Drive / 其他）    │
│                                             │
│ 這條串消除的摩擦是：______                     │
└─────────────────────────────────────────────┘
```

### 分組（高階 + 中階 + 起步 混組，助教帶動）

| 組別 | 助教 | 成員 |
|---|---|---|
| A | Sophia | 潘潘 + 政澔 |
| B | Barbie | 藝嘉 + 岱恩 |
| C | 詩媛 | 蕙瑄 |

每組上台展示，Jacky 給回饋：**「這條串消除了哪個摩擦？還有哪個漏電點？」**

## 模組 6｜課後作業（分級）

回應問卷「工作太忙沒時間」的擔憂，作業分三級，**至少完成 Level 1**。

| 級別 | 工作量 | 任務 |
|---|---|---|
| **Level 1（必做）** | 約 1 小時 | 為最熟的品牌完成「第一條串」設計圖（老闆角色 / 4 Agent 邊界 / 第一個串起來的執行環節） |
| Level 2（建議） | 約 2 小時 | 把設計圖輸入平台，跑第一條串，記錄哪一步卡住 |
| Level 3（挑戰，高階組） | — | 幫手上第二個品牌也跑一次，比較兩個「串」的差異 |

**Week 6 前要做好**：Level 1 作業必交（截止：Week 6 前一天）；政澔與 Jacky 約一對一健檢（15 分鐘）。

## 設計哲學注記

> **v3 之所以放棄「教 API」**：技術門檻太高、夥伴擔憂大（問卷 3/6 人「不確定 API 是什麼」/ 3/6「技術門檻太高」）。

> **語言哲學**：用「絲滑 / 摩擦力 / 串 / 腦跟手」取代「API / JSON / 路由」。
