# TBSA 五張表單工作坊 — 實戰採坑點

> 2026-05-17 台中科技大學首場實戰後的 7 個教訓。每個都是「備課時沒發現、上課才爆」的問題。
>
> 配合閱讀：[備課流程 README](../workshops/2026-05-14-五張表單工作坊備課流程/README.md) ← 備課設計的「應然面」

---

## 採坑 #1：STP 的 AI Demo 只教了 1/4 任務

- **症狀**：5 組全數缺感知圖、2 組 Segmentation 只有一句話，但 Persona 全部高品質完成
- **誤區假設**：「STP 理論學生都學過，AI Demo 教 Persona 就夠」
- **真實原因**：Form 3 有 5 個子任務（S/T/P/Persona/感知圖），但 AI Demo 只覆蓋 Persona。學生在時間壓力下只做「有 prompt 範本的事」，其餘直接跳過。對比 Form 1（三選一）和 Form 2（雙 prompt 攻防），AI Demo 覆蓋率直接決定產出品質。
- **解法**：增加 Segmentation prompt Demo（「用 Form 1 消費者洞察 + PEST，給 3 種區隔 + 規模估算」）+ 感知圖軸線 AI 建議 prompt
- **預防**：備課時數「AI Demo 數量 ÷ 表單子任務數」，比值 < 0.5 就要補 Demo
- **發生於**：2026-05-17 台中科大，Form 3 段落（13:00-14:00）

---

## 採坑 #2：表單順序改了，Dataroom 沒跟著改

- **症狀**：簡報教 Form 1 = 企劃情報，但學生打開 Dataroom 的欄位指引看到 Form 1 = 現況分析。AI 設定檔也是舊順序。
- **誤區假設**：「改完簡報和工作區 docx 就好了」
- **真實原因**：表單 1↔2 對調是在備課最後 36 小時才做的結構性變更，影響範圍跨 6+ 個檔案。改了簡報（.pptx）和工作區（.docx），但 Dataroom 的 3 個 .md 檔（AI設定檔 + 2 份欄位指引）被遺漏。
- **解法**：課前最終模擬時發現並修正（重寫 3 檔、改檔名、改內文標題）
- **預防**：任何結構性變更（順序、命名、分組）後，跑一次「grep 舊名稱 across 全資料夾」確認零殘留
- **發生於**：2026-05-17 凌晨 02:30 最終模擬時發現

> 延伸閱讀：[SOSTAC 順序鐵規](../../ailab/concepts/SOSTAC順序鐵規.md)

## 採坑 #3：課程模擬沒考慮「非本科系」學生

- **症狀**：模擬報告預測 STP「節奏適中，無壓力」，實際卻是品質最不均的表單
- **誤區假設**：「模擬跑通 = 學生也跑得通」
- **真實原因**：模擬對象是對 STP 有先備知識的虛擬學生。台中科大可能多為設計系，問卷明確回饋「設計學這個還是太難」「企管系可以多一點」。STP 是行銷系大二才教的概念，設計系學生第一次接觸。
- **解法**：下次模擬加入「對 STP 完全陌生」的虛擬學生角色
- **預防**：模擬前先確認學員科系組成，對非商管科系加入「白話翻譯」和更多 step-by-step
- **發生於**：2026-05-17 課中 STP 段落

## 採坑 #4：「教了」≠「教了怎麼做」

- **症狀**：P26 實作引導寫了「感知地圖 X/Y 軸要明確定義」，5 組 0 組畫了感知圖
- **誤區假設**：「告訴學生『要做什麼』就夠了」
- **真實原因**：學生缺的不是「知道要做感知圖」，是「不知道從哪裡開始」。沒有 prompt 範本、沒有工具指引、AI 也不擅長產圖 → 直接放棄
- **解法**：每個要求附帶「30 秒能啟動的具體第一步」。感知圖的第一步＝「問 AI：根據我的競品分析，建議用什麼 X/Y 軸來做定位感知圖？」
- **預防**：實作引導頁的每一條，都要能回答「學生看到這行字後，下一個動作是什麼？」。如果答案是「想一下」就代表會被跳過。
- **發生於**：2026-05-17 Form 3 實作段

## 採坑 #5：頁碼版本殘留（XX / 42 vs XX / 55 vs XX / 39）

- **症狀**：40 頁簡報中混雜三種頁碼格式（來自不同版本），部分頁面根本沒有頁碼
- **誤區假設**：「每次新增/刪除頁面後記得改」
- **真實原因**：簡報經歷 v1→v2→v3→v4→final 五個版本，每次插入/刪除投影片都改變總頁數。手動維護頁碼不可能不出錯。
- **解法**：最終版 freeze 後，用 python-pptx 一次性「刪掉所有頁碼 shape → 重建於統一位置」
- **預防**：頁碼永遠最後一步做，且用腳本不用手動。腳本偵測模式：`re.compile(r'^\s*\d{1,3}\s*/\s*\d{1,3}\s*$')`
- **發生於**：2026-05-17 凌晨字型統一作業時發現

## 採坑 #6：中文路徑 + OneDrive + python-pptx = 必爆

- **症狀**：`python-pptx` 對含中文的 OneDrive 路徑直接 crash（encoding error 或 file not found）
- **誤區假設**：「Python 3 已經 Unicode 化了，路徑不是問題」
- **真實原因**：OneDrive 路徑含中文 + 特殊字元（如「_」前後中文），Windows cp950 環境下 `print()` 含特殊符號也會炸（如 ✓✗）。`Presentation()` 有時能開、有時不能，取決於 OneDrive 同步狀態。
- **解法**：標準模式 = `shutil.copy2(src, os.path.join(os.environ['TEMP'], 'work.pptx'))`，所有操作在 TEMP 做完再 copy 回去
- **預防**：所有 python-pptx / python-docx 腳本第一行就 copy 到 TEMP，最後一行 copy 回來。print 不用 ✓ 改用 [OK]。
- **發生於**：整個備課過程反覆出現（> 20 次）

> 延伸閱讀：[Claude Code 工具筆記](../../ailab/tools/claude-code.md)

## 採坑 #7：Form 5 時間被前面吃掉

- **症狀**：5 組中 3 組 Form 5 有空白欄位（營收 or ROI），第三組 ROI 寫 1257%（不合理但來不及檢查）
- **誤區假設**：「前面表單教好，Form 5 只是整合，自然會完成」
- **真實原因**：Form 5 是「收斂型」表單（把前四張精華濃縮成一頁），需要回頭翻看所有之前的內容。但放在最後 = 最疲勞 + 最趕時間。且 Form 5 的「所需經費」需要跟 Form 4 的 AIDAS 預算對齊，學生懶得回去對。
- **解法**：Checkpoint 5 增加「跟 Form 4 預算對得起來嗎？」的強制檢核項。或在 Form 4 結束時就先讓學生把預算加總寫在 Google Doc 最底部。
- **預防**：「收斂型表單」不能假設學生會自己回頭查。要在前面表單就埋下收斂的鉤子（如 Form 4 最後一步 = 預算加總）。
- **發生於**：2026-05-17 15:30-16:20 Form 5 段落

## 總結：備課品質指標清單（下次用）

從本次 7 個採坑點歸納的 pre-flight checklist：

- [ ] 每張表單的「AI Demo 數 ÷ 子任務數 ≥ 0.5」
- [ ] 結構性變更後 grep 全資料夾確認零殘留
- [ ] 模擬對象包含「目標科系的真實先備知識水準」
- [ ] 實作引導每一條都能回答「學生下一個動作是什麼」
- [ ] 頁碼用腳本最後一步統一產生
- [ ] python 腳本全走 TEMP 中轉
- [ ] 收斂型表單在前面就埋鉤子
