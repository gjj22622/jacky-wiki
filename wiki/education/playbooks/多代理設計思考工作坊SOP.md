# 多代理設計思考工作坊 SOP

> 把「找一桌人腦力激盪、吵完給我會議紀錄」這件事，變成一個跑 12 分鐘、自動產出具名會議紀錄的 Workflow。
> 適合：解開卡住的設計題、策略發散、痛點找解方、任何「需要多元視角＋互相碰撞」的題目。
> 與 [Subagent 並行 fan-out 交付 SOP](subagent並行fan-out交付SOP.md) 的差別：fan-out 是「N 件**獨立**產物各做各的、不互看」；本法是「N 個視角**互相看、互相戰、再收斂成一份共識**」——重點在碰撞，不在並行加速。

## 什麼時候用這個（而不是自己想 / 不是 fan-out）

- 需求很模糊（「幫我想想這個怎麼解」）、答案空間很寬。
- 你要的是**討論過程與收斂**，不是一堆並排點子。
- 題目會因「不同立場互相挑戰」而變好（設計、策略、定價邏輯、服務流程）。

## 核心骨架：三輪

```
[發想 parallel] → [交鋒 parallel] → [綜合 single]
  各人格獨立提案     互相挺/戰/加碼      引導師寫會議紀錄
```

1. **發想（parallel）**：N 個人格代理各自從自己視角發想，schema 強制結構化（提案名／解決哪個痛點／怎麼運作／可行性）。
2. **交鋒（parallel）** ★最關鍵：把第一輪**全部提案彙成一張白板字串**，回餵給每個人格，要它對別人的提案表態（挺／戰／加碼）＋說「綜合後我會改成…」。
   > 沒有這一輪，產出只是 N 份獨白並排；有這一輪，才有真討論、互相修正、自然收斂。**這是本法和普通 fan-out 的分水嶺。**
3. **綜合（single）**：一個「引導師／首席」代理拿前兩輪全文，寫出**保留具名發言與交鋒**的會議紀錄＋收斂方向＋待決問題。

## 人格怎麼設計（成敗在這）

每個人格 `desc` 要寫足三件事：**信念（他堅持什麼）＋禁忌（他絕不接受什麼）＋他會戰什麼**。人格立得住，交鋒才有火花。一桌人至少放這兩種「特殊座位」：

- **第一性原理人格（Elon 化身）**：灌「質疑每條需求、能刪就刪（the best part is no part）、能自動化別用人、敢碰硬體、用瘋狂時間表逼簡單解」。
  > 它的價值＝把「怎麼教老師傅用電腦」直接改寫成「**把這個需求刪掉**」。逼出別人想不到的解。
- **對抗性用戶代理（真實使用者本人）**：放一個「目標客戶本尊」進團隊（例：62 歲不打字、怕被取代的老師傅），任務是把每個方案放他身上壓力測試「**這我才不要用**」。
  > 它是落地性的守門員。一場會它一票否決了「桌遊卡牌／廣角相機／感測手套」這些聰明但會死的方案。**沒有它，整桌專家會自嗨。**

其餘座位按題目放領域專家（方法論官、前台、財務…），每個一個鮮明立場。建議 4–6 人格。

## 操作步驟

1. **寫 GUARD 常數**：守線（定價停拍板／客戶匿名／30% 揭露／去 AI 腔）寫一次，每個 prompt 都帶（DRY）。
2. **寫 PROBLEM 常數**：把痛點講死（例：P1 不會電腦／P2 沒語料／P3 知識難統整），對映到階段。
3. **定 PERSONAS 陣容**：每人一個 `{key, label, desc}`，desc 照上面「信念＋禁忌＋會戰什麼」寫。
4. **三輪用 parallel / parallel / single**，schema 各自鎖（IDEA / REACT / MINUTES）。
   - 交鋒輪：把第一輪結果 `.map()` 成白板字串，整段塞進每個人格的 prompt。
5. **綜合輪 prompt** 明列會議紀錄結構：會議資訊→按痛點記錄討論（誰提什麼、誰戰誰、用戶代理買不買單、收斂到哪）→解方方向→共識原則→待拍板→MVP。
6. 跑完落檔（會議紀錄寫檔、上 Drive），不要把全文塞回主對話。

## 選型心法（踩過才知道）

- **要「會議紀錄」型產出 → 用交鋒輪，別用打分輪**。同期另一場策略研究用「5 視角發想＋3 評審打分」，產出偏靜態；改成交鋒輪後討論感與收斂品質明顯更好。打分輪適合「選最好的方案」，交鋒輪適合「吵出一個共識」。
- **schema 強制結構化**：合成階段才有乾淨素材，不用解析自由文。
- **背景跑**：用 Workflow（完成自動通知），不要自己開一堆 Agent 手動接力。

## 雷區

- **子代理 WebSearch 在 workflow 內常回傳不可用**：research 型一定要在 prompt 內強制「**查不到就誠實標查無、別編**」，並讓合成階段保留「資料限制聲明」。→ 見 [pitfalls](../pitfalls/)。
- **少了對抗性用戶代理＝專家自嗨**：方案聰明但使用者不買單，這是設計思考最常見的死法。
- **人格 desc 太單薄**：寫成「你是行銷專家」沒用，交鋒會變一團和氣。要寫到「他會為了 X 跟人翻臉」。
- **交鋒輪忘了回餵白板**：忘了把第一輪彙總塞進去，第二輪就變第二次獨白，整套白做。
