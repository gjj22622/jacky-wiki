# 双云客戶腦範本（Brand Templates）

> **層級**：Brain（客戶品牌腦範本）
> **domain**：brand
> 來源：`06_Skills/双云技能包/{brand-beauty-template, brand-ecommerce-template, brand-food-template}/`

> 3 個產業類別的客戶腦範本。每個範本包含品牌定位、目標受眾、語調、內容策略、絕對禁區、語調範例。**幫新客戶上線時，從最接近的範本開始客製化。**

---

## brand-beauty-template｜醫美品牌腦

> 適用於醫美診所、保養品牌

### 品牌定位

- **核心價值**：專業安全、自然美感、客製化療程
- **目標受眾**：25-45 歲女性，注重保養、追求自然美
- **品牌語調**：溫暖專業、不推銷、以教育引導

### 內容策略

| 類型 | 優先順序 |
|---|---|
| 社群貼文 | 素人分享 > 醫師專業知識 > 療程介紹 |
| 影片 | before/after 見證、醫師 Q&A、日常保養教學 |
| 廣告 | 痛點喚起 → 解決方案 → 限時體驗 |

### 絕對禁區

- 不**宣稱療效**（違反醫療廣告法規）
- 不使用「最」「第一」「保證」等絕對用詞
- 不比較競品
- 不使用過度修圖的 before/after
- 不洩露客戶隱私

### 語調範例

| | 句子 |
|---|---|
| ✅ 好 | 「了解自己的肌膚需求，找到適合的保養方式」 |
| ❌ 壞 | 「用了就年輕 10 歲，保證有效」 |

## brand-ecommerce-template｜電商品牌腦

> 適用於線上零售、D2C 品牌

### 品牌定位

- **核心價值**：高 CP 值、獨特設計、快速到貨
- **目標受眾**：22-38 歲，網購重度使用者
- **品牌語調**：年輕有活力、直接利益訴求、有個性

### 內容策略

| 類型 | 優先順序 |
|---|---|
| 社群貼文 | 產品情境照 > 用戶開箱 > 限時優惠 > 品牌故事 |
| 影片 | 開箱評測、穿搭／使用示範、幕後製作 |
| 廣告 | FOMO 限時 → A/B 素材測試 → 再行銷 |

### 轉換重點

- 每則貼文都要有**明確 CTA**
- 追蹤 **UTM 參數**
- 社群導流到官網／蝦皮／Momo

### 絕對禁區

- 不做虛假折扣（**先漲後降**）
- 不抄襲競品文案
- 退換貨政策必須透明
- 不使用與實物嚴重不符的照片

## brand-food-template｜餐飲品牌腦

> 適用於餐廳、飲品店、食品品牌

### 品牌定位

- **核心價值**：美味、新鮮、用心製作
- **目標受眾**：20-40 歲，重視生活品質、愛美食分享
- **品牌語調**：親切有溫度、生活感、不矯情

### 內容策略

| 類型 | 優先順序 |
|---|---|
| 社群貼文 | 美食實拍 > 製作過程 > 食材故事 > 客人好評 |
| 影片 | 料理過程縮時、開箱、吃播風格 |
| 廣告 | 食慾喚起 → 限定優惠 → 到店／外送 CTA |

### 絕對禁區

- 不貶低其他餐廳
- 不誇大食材來源（**有機、進口需有證明**）
- 不使用與實物差異過大的修圖
- 食安相關用詞必須精確

### 語調範例

| | 句子 |
|---|---|
| ✅ 好 | 「每一口都是我們用心的味道」 |
| ❌ 壞 | 「全台最好吃，不吃後悔」 |

## 範本使用方式

| 步驟 | 動作 |
|---|---|
| 1 | 從 3 個範本中找出**最接近**新客戶產業的那一個 |
| 2 | 用 `shuangyun-brand-pack-builder` ([Skill OS](SkillOS系統.md)) 客製化定位、受眾、語調 |
| 3 | 用 `shuangyun-review-rule-builder` 把客戶的禁區、合規要求變成 review rules |
| 4 | 把客製化後的腦變成 `sy.brain.{客戶代號}`，由 `shuangyun-master-content-orchestrator` 呼叫 |

> **如果客戶產業不在這 3 類**：用最接近的範本當基底（例：3C 用 ecommerce、咖啡用 food、SPA 用 beauty），再做客製化。

## 共同設計原則

1. **「絕對禁區」永遠在第三段** — 確保下游 Skill 在任何時候都會看到底線
2. **語調範例必須有「好 vs 壞」對照** — 比口頭描述「自然」「親切」更具體
3. **每個範本都列出 3 種內容類型的優先順序** — 給內容 Agent 明確的選題權重
4. **不在範本裡放具體客戶名稱** — 範本是底版，客戶名稱在客製化階段才填入
