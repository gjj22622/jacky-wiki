## 最終做法（What Worked）

**「真 agent 孿生市場模擬」法** —— 不用合成問卷逼近 agent，直接 spawn 真 subagent 當每個買家：

1. **三個版本化物件**：`personas.v1.json`（市場人口，每個是「人類老闆＋買家 agent」dyad，座標對齊 `/api/assess` 七維）＋ `ideas-27.json`（待測點子）＋ rubric（領域戒律＝《當消費者變成 Agent》第五段五戒律）。roster 是策展、版本化 JSON，**不每次 LLM 隨機生** → 可重現、可比較（同 /api/assess 低變異哲學）。
2. **真實客戶錨定**：6 個真實客戶去識別化當核心 persona ＋ 7 個合成補洞補矩陣空格，`segment_weight` 加總=1。
3. **Workflow 扇出真 subagent**：13 個 buyer-agent 並行，**每個一次評完全部 27 點**（不是每點一呼叫——省成本：13 呼叫而非 351）。schema 強制 `agent_verdict`/`boss_verdict` 封閉列舉 ＋ 自由文字 `reason`（借 SSR「先文字判斷再映射刻度」精神，但用 schema 實現、不裝它的 Python 依賴）。
4. **確定性 JS 聚合**：成功率 = Σ(weight × P(adopt) × P(authorize))，排名與三浮現訊號榜（信任落差／DIY 侵蝕／真實需求拉力）全在 workflow 內用 JS 算，**不讓 LLM 算數**。
5. **交付**：報告 md ＋ 双云品牌 HTML 視覺化（node 從 result.json 生 HTML 保證數字精確、playwright 兩斷點自截圖驗）→ 上 Drive。
6. 整套包成可複用 skill `數位孿生市場模擬`，丟任何點子集都能跑。

> 為什麼穩：把「判斷變異」收進兩處確定性——版本化 roster（誰在市場）＋ JS 聚合（怎麼算分）；只把「每個 persona 對每點的 verdict」交給 LLM，且 schema 鎖死輸出形狀。

## 繞路紀錄（Detours）

- **Workflow `args` 傳資料 → 進不到 script**：第一次把 personas/ideas 當 `args` 傳給 Workflow 工具，script 內 `args.personas` 是空 → workflow 直接 throw。改成**把資料嵌進 script body 的 const**（保留 `args.x || EMBEDDED` 兼容寫法）最穩，再 `resumeFromRunId` 重跑。
- **解析 workflow 輸出檔**：先用 `indexOf('{"ranked"')` 找 JSON → 失敗，因為輸出是 pretty-print（`"ranked": [` 有空格）＋ 檔案結構是 `{summary, agentCount, logs, result:{ranked,...}}` 包一層。改成 `JSON.parse(file).result.ranked` 一行解決。

## 錯誤與失敗（What Failed）

- **假設 Workflow 的 `args` 會原樣注入 script 的 global `args`** —— 實測是空的。教訓：workflow script **無檔案系統存取**、`args` 注入不可靠 → 要餵資料就**嵌進 script const**，別走 args、更別想在 script 內讀檔。
- **差點把模擬結果當定論**：A 類（測量情報層）集體墊底，一度看起來像「點子爛」。細看 reason 才發現買家 agent 一致說「**我是買方不是品牌商**」——A 類的真客戶是**賣家**，而我只孿生了**買方市場**。這是模擬設計的根本陷阱：**先定義你模擬的是市場哪一側**，否則會把「賣錯邊」誤讀成「點子差」。

## 升格候選

- ⭐ **「真 agent 孿生市場模擬」=新 pattern**：spawn 真 subagent 當 persona（不是合成問卷）＋版本化 roster＋領域戒律 rubric＋JS 確定性聚合。已成 skill `數位孿生市場模擬`；方法論本身候選 `patterns/` 或 `concepts/`。
- ⭐ **「先定義模擬的是市場哪一側」**＝模擬設計鐵則：候選 `concepts/`（多 agent 模擬的方法論陷阱）。
- **Workflow args 不可靠 → 資料嵌 script const**：候選 `tools/`（Workflow 工具雷區一條）。
- **省成本扇出法**：N 個 agent 各「一次評完全部 M 項」而非 N×M 呼叫 → `tools/` 或 patterns 補一條。
- 「node 從 result.json 生 HTML 保數字精確 + playwright 自截圖驗」已是既有實踐，不用再升。
