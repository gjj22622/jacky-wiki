# 如何打造一個「品牌指揮中心」平台

> **驗證來源**：[木酢寵物達人案例](../shuangyun/cases/木酢寵物達人.md)（2026-05-04 → 2026-05-07，三天從零到上線給團隊）  
> **適用場景**：客戶的多渠道（GA4 + Meta Ads + 蝦皮 + 官網 + CRM）數據需要整合成一個團隊共用、可分享 Token 的內部平台。

---

## 平台核心定位（先想清楚再寫程式）

> 「每多看一個數字沒有對決策產生價值，就是無用的數字。存起來要看再撈出來就好。」— Jacky

平台不是「秀數據的地方」，是「**幫使用者把數據準備好讓 AI 分析**」。
真正的決策還是在使用者的腦子裡，平台的責任是：

1. **把分散的數據整合到一個地方**（GA4 / Meta / Shopee / WACA / CRM）
2. **每個數字都要驅動一個決策問題**（不是「炫技」用的）
3. **支援使用者帶數據去自己的 AI 工具**（Claude.ai Pro / ChatGPT / Gemini）— 不要強迫他們付兩份 API 費用

---

## 七大架構決策

### 決策 1：技術棧——零外部依賴

**選擇**：Node.js 24 ESM + 純 HTML/Vanilla JS + 不裝任何 npm 套件。

**為什麼**：
- Node 24 已內建 `fetch`、`crypto`、`--env-file`，不需要 dotenv / axios
- HTTP server 用 `node:http` 就夠
- 部署到 Zeabur／Vercel／任何雲都可以，build 時間 < 30 秒
- 沒有依賴 = 沒有資安漏洞、沒有版本衝突、沒有突然棄用

**反例**：用 Next.js / Express + 50 個 npm 套件 = 部署 5 分鐘、3 個月後升級會壞、上 prod 多一倍 bug。

### 決策 2：資料層分兩種——即時 API + Excel fallback

|資料 |做法 |理由 |
|---|---|---|
| GA4 | Service Account JWT → Data API（即時） | 每天都要看，自動化省事 |
| Meta Ads | 系統用戶 Token → Graph API（即時） | 同上 |
| 蝦皮 | 月底匯出 Excel，Python 解析成 JSON | 蝦皮 API 太煩，月度數據夠用 |
| WACA | 月底匯出訂單 Excel，Python 解析 | 同上 |
| CRM | 靜態值寫 hardcode | 月度更新一次，不需要 API |

**關鍵原則**：API 串接需要 30+ 分鐘配置（OAuth、權限、quota）。**只有「每天要看」的數據才值得串 API**，月度更新的東西用 Excel + 月底跑 Python 就好。

### 決策 3：認證——HTML cookie + API Bearer 雙軌

**前台**：HTML 頁面用 cookie（server-side 強制檢查，無 token → 302 redirect 到 login.html）。  
**API**：用 `Authorization: Bearer <token>` header。  
**Admin**：再多一層 `ADMIN_SECRET` 環境變數保護管理 API。

**Token 儲存**：`data/users.json`（純 JSON，`node:fs` 讀寫）。不需要資料庫。
**為什麼不用 OAuth**：團隊內部 10 個人，沒必要弄 SSO。簡單貼 token 登入最有效率。

### 決策 4：AI Agent 用 Tool Use 而非 Context Injection

**錯誤做法**（v1）：把所有數據塞進 system prompt，讓 Claude 分析。
- 問題：上下文不準（一週還是一個月？），token 浪費，無法問「跟去年比」
**正確做法**（v2）：定義工具，讓 Claude 主動呼叫。

```js
// 工具範例（不裝任何 SDK，用原生 fetch 呼叫 Anthropic API）
const TOOLS = [
  { name: 'get_ga4_summary', input_schema: { days: 7|14|30|90 } },
  { name: 'get_monthly_trend', input_schema: { months: 3|6|12 } },
  // ... 8 個工具
];

// Agentic loop
while (round < MAX_ROUNDS) {
  response = await anthropic_messages({ messages, tools, stream: true });
  if (no tool_use) break;
  for (toolUse of response) {
    result = await executeTool(toolUse.name, toolUse.input);
    messages.push(toolResult);
  }
}
```

**好處**：
- Claude 自己決定查 7 天還是 90 天
- 同一回合可平行呼叫多個工具
- Token 不浪費（只查需要的）
- 前端可以顯示「🔍 查詢 GA4 流量 KPI」「✅ 查詢月度歷史趨勢」的進度

### 決策 5：「生成指令」按鈕（產品定位翻轉）

**洞察**：使用者已經付月費給 Claude.ai Pro / ChatGPT Plus，不會想再為平台的 API 付一次錢。

**做法**：除了「直接用我們 API 回答」按鈕（➤），加一個「📋 生成指令」按鈕。
- 點 📋 後：server 跑同樣的 `executeTool()` 抓即時數據 → 組合成含真實數字的 prompt → 回傳前端
- 前端彈出 modal 顯示完整 prompt → 一鍵複製
- 使用者貼到自己的 AI 工具用

**這不是退步，是進步**。平台從「AI 助手」升級成「**幫使用者準備數據的數據層**」。
**好處**：
- 平台幾乎沒有 API 成本（只有抓資料的 API 費用，不呼叫 Claude）
- 使用者用熟悉的介面
- AI 給的答案品質取決於使用者付費的工具，平台不用負責

### 決策 6：頁面只回答決策問題

每一頁都要能 5 秒內回答「我該採取什麼行動」：

| 首頁要回答 | 對應數字 |
|---|---|
| 廣告值得投嗎？ | Meta ROAS（單一大數字 + WoW）|
| 流量在長嗎？ | Sessions + WoW |
| 這週賺了多少？ | 官網收益 + WoW |
| 哪個渠道最值錢？ | 來源 ROI（前 4 名）|
| 收益趨勢對嗎？ | 月度趨勢圖（蝦皮+官網+Meta）|
| 客戶有留下來嗎？ | CRM：回購率 / 沉睡客 / VIP LTV |

**移除的東西**：裝置分布、落地頁 Top10、累計 KPI strip、月份明細表、每日趨勢、詳細欄位。
**理由**：這些放在 `ga.html` / `meta.html` 深層頁面，首頁不要。

### 決策 7：日期參數化 + 多份快取

預設不要寫死 7 天。讓 API 接收 `?days=7|14|30|90`，server 端用一個 `Map` keyed by days：

```js
const _gaCache = new Map();
async function fetchGAData(days = 7) {
  const cacheKey = `ga_${days}`;
  if (cached && fresh) return cached.data;
  const curr = `${days}daysAgo`;
  const prev = `${days * 2}daysAgo`;
  // ... query GA4 API
  _gaCache.set(cacheKey, { data, expiry: now + 30min });
}
```

**好處**：
- 同一個函式回答四個 tab（7/14/30/90 天）
- 各自快取，切換 tab 第二次就快
- AI Agent 也可以用 days 參數呼叫

## 採坑紀錄

### 坑 1：dashboard.html vs overview.html 兩個首頁混淆使用者

**症狀**：使用者問「為什麼有兩個入口」。
**根因**：保留舊的靜態 dashboard 又加新的 overview，導航有兩個「總覽」。
**處理**：直接刪除 dashboard.html，數據（CRM 靜態值）遷移到 overview。

> 教訓：升級時要**狠心砍**舊頁面。保留 = 動線混亂。

### 坑 2：Tool Use 第一次呼叫失敗（`_raw` 欄位錯誤）

**症狀**：`messages.1.content.1.tool_use._raw: Extra inputs are not permitted`
**根因**：streaming 期間累積 tool input 的 JSON 字串存在 `_raw` 欄位，但發回 Anthropic 時沒清掉。
**處理**：發送前 `delete b._raw` 或解構排除。

```js
const cleanContent = assistantContent.map(b => {
  if (b.type === 'tool_use') { const { _raw, ...clean } = b; return clean; }
  return b;
});
```

### 坑 3：Zeabur 部署失敗 — `.env not found`

**症狀**：`/usr/local/bin/node: .env: not found`，container 不停 restart。
**根因**：`package.json` 的 `start` 寫成 `node --env-file=.env server.js`，但 Zeabur 不會放 `.env` 進 container（環境變數是直接注入的）。
**處理**：`start` 改為 `node server.js`（不帶 --env-file）；`dev` 保留 `--env-file=.env` 給本機用。

### 坑 4：Domain 命名衝突

**症狀**：`muzopet-agent.zeabur.app` 被佔用。
**處理**：換成 `muzopet-dashboard.zeabur.app`。

### 坑 5：GA4 數據跟後台對不上

**症狀**：客戶說「Sessions 數字怪怪的」。
**根因**：API 用 `7daysAgo → today`（含今日，數據不完整），GA4 後台預設用「過去 7 天」（不含今日）。
**處理**：統一用 `7daysAgo → 1daysAgo`（7 完整天）。前後對齊後數字一致。

### 坑 6：每次 redeploy admin token 全部消失（**必踩**）

**症狀**：團隊登入金鑰每次 `git push` 後就失效，要重發一輪。
**根因**：
- `data/users.json` 在 `.gitignore` 內（token 是敏感資訊不能進 repo）
- Zeabur 容器是 ephemeral，每次部署重建容器，本地寫入的檔案全部消失

**處理**：在 Zeabur Dashboard **「硬碟」分頁**新增 volume 掛載到容器的 `/src/data`：

| 欄位 | 值 |
|---|---|
| 硬碟 ID | `userdata` |
| 掛載目錄 | `/src/data`（Zeabur Node.js buildpack 的工作目錄）|
| 大小 | 1 GB |

**設定完必須執行完整 `zeabur deploy`**（不是 restart）才會生效。設定第一次後，舊的 `/src/data/` 會被空 volume 覆蓋，需要重新建立一輪 token。之後就永久持久。

**程式碼配合**：在 `loadUsers()` 之前加 `mkdirSync(DATA_DIR, { recursive: true })`，確保 volume 第一次掛載時 `data/` 目錄存在（不然 atomic write 會失敗）。

**驗證**：
```bash
# 寫測試檔
echo "test" > /src/data/.persist
# git push + zeabur deploy
# 部署完進容器
cat /src/data/.persist  # 應該還在
```

> 💡 注意：Zeabur 硬碟用 sidecar 機制處理，`mount` 命令看不到掛載點，但檔案會跨部署保留。不要被 `mount` 結果誤導。

## 可複用結構

下次要再做類似平台（電商客戶 #2、#3）的話，可以直接複製這個結構：

```
my-dashboard/
├── server.js               # 單檔 server（HTTP + auth + API + agent）
├── package.json            # 零依賴
├── .env                    # 本機開發用
├── overview.html           # 首頁（行銷漏斗）
├── ga.html                 # GA4 深層
├── meta.html               # Meta 深層
├── agent.html              # AI 顧問
├── login.html / admin.html # 認證
├── factory.html            # 系統狀態（可選）
└── data/
    ├── users.json          # token list（gitignore）
    └── timeline-offline.json  # 月度歷史（Python 產出）
```

換新客戶要改的：
1. `server.js` GA4 Property ID + Meta act_xxx
2. CRM 靜態值（會員/沉睡客/LTV）
3. UI 配色（CSS 變數）
4. 客戶品牌名
