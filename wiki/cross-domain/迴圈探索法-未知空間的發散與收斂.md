# 迴圈探索法 — 未知空間的發散與收斂

> 一句話：**當你要找的東西「數量未知、邊界不清」（有多少個生意點子？有哪些新需求？有幾個 bug？），用多個正交視角反覆掃，掃到「連續幾輪都掃不出新東西」才停，再用對抗驗證剔掉假貨，最後用程式（不是 AI）算分排序。**
>
> 這是 2026-06-27「賣鏟子的下一個卡位」探索一次跑出 112 個需求、收斂成 5 個可落地卡位點的方法。可複用於任何「窮舉式探索」。

---

## 什麼時候用（與其他多代理模式的分界）

| 你的處境 | 用哪個方法 |
|---|---|
| N 件**彼此獨立**的事要同時做（各做一院視覺、各審一個檔）| [Subagent 並行 fan-out](../ailab/patterns/subagent並行fan-out設計法.md)（求**加速**）|
| N 個**視角互相辯論**收斂成一個共識（一個設計該不該做）| [多代理設計思考工作坊](../ailab/patterns/多代理設計思考工作坊.md)（求**碰撞收斂**）|
| 要找的東西**數量未知、可能很多、會漏**（有哪些生意點子／新需求／edge case／bug）| **本法：迴圈探索法**（求**窮舉不漏 + 去蕪存菁**）|

> 關鍵判準：如果你問的是「**還有沒有？**」而不是「**哪一個比較好？**」——用本法。簡單的「跑一次列清單」會漏掉長尾；本法的價值就在那條尾巴。

---

## 四個階段

```
① 發散（多透鏡）   N 個正交視角各自掃，每個視角只准用自己的角度
        ↓  loop-until-dry：完整性評論者找盲點 → 補新視角 → 再掃
        ↓  連續一輪新增 < 門檻（或滿 K 輪）才停
② 去重（程式）     正規化名稱去重，累積成獨特清單（不讓 LLM 算）
        ↓
③ 對抗驗證（批次） 把清單切塊，每塊一個懷疑者一次審完——真的嗎？已飽和嗎？多快？
        ↓
④ 收斂（程式算分） 確定性公式排序 + 合成「地圖」（分層 / 該卡哪幾個 / 該剔哪幾個）
```

### ① 發散 — 多透鏡，而且要「正交」

不要叫一個 agent「列出所有 X」——它會給你最明顯的前 10 個就停。改成**派多個 agent，每個戴一副不同的眼鏡**，且眼鏡之間角度盡量不重疊（正交），逼出不同的長尾。

賣鏟子那次用的 6 副眼鏡（可借用框架）：
- **瓶頸轉移**：移除一個限制後，需求湧向哪個相鄰限制？
- **選配→必須**：今天 nice-to-have、未來變剛需的是什麼？
- **每單位乘 N**：當 1 變 N，哪些「按單位計價」的東西會爆量？
- **角色專屬**：某個角色（行銷／營運／老闆）會長出什麼專屬需求？
- **鄰接擴張**：現有熱點的上一層／下一層，下一個會熱的是什麼？
- **故障模式**：規模 ×100 時什麼會壞？每個「會壞」＝一個需求。

> 心法：**視角的正交度 = 你能挖多深**。六副重疊的眼鏡不如三副真正不同的。

### ① 的 loop — 掃到「乾」才停（loop-until-dry）

第一輪掃完，**派一個「完整性評論者」**讀全部結果，專門挑「**還沒被任何視角覆蓋到的盲點**」，產出 3-4 個全新切角；用這些新切角再掃一輪。

停止條件：**連續一輪新增 < 門檻（如 < 3 個）**，或**滿 K 輪**（如 4 輪，防失控）。

> 為什麼不用「跑一次就好」：簡單探索會在第一輪的明顯答案停手，**真正值錢的常在第 3-4 輪的盲點裡**。賣鏟子那次，第 1 輪 6 視角後，評論者補出「跨組織 agent 信任」「實體執行介面」「人機交接摩擦」等盲點，第 2-3 輪才挖出實體層與信任層的需求。

### ② 去重 — 用程式，不用 LLM

不同視角會用不同說法講同一件事。**用程式正規化名稱**（小寫、去標點、截長度）做 `Set` 去重，累積成獨特清單。**別叫 LLM「幫我去重」**——那會引入變異且燒 token。near-dupe 留到最後合成階段語意合併。

### ③ 對抗驗證 — 默認懷疑，批次省成本

每個候選都要過一關懷疑者，問四件事：**這是真需求還是空想？已經被大廠吃滿了嗎？多快會發生？我方能不能卡位？** 預設立場是「**寧可殺錯，不可放過空想**」。

省成本關鍵：**不要一個候選派一個 agent**（112 個就是 112 次呼叫）。**切塊批次**——把清單切成每塊 8 個，一個 agent 一次審完一塊，112 個只要 ~14 次呼叫。這是「N 個 agent 各審 M 項」而非「N×M 次」的省錢法。

### ④ 收斂 — 確定性公式算分，再合成地圖

排序**絕不讓 LLM 算數**。用一條透明公式，例如賣鏟子那次：

```
卡位優先序 = 必然性 × 供給缺口 × (0.35 + 0.65 × 我方貼合度) × 時機權重
（剔除 is_real=false 或 already_saturated=true 的）
```

算完才派**一個合成 agent**讀排好的清單，產出「**地圖**」：分層（這些東西怎麼疊成一個 stack）、該現在卡位的 3-5 個（附「為何現在／第一步／護城河」）、暫時觀望的、該剔除的。

> 合成是唯一讓 LLM「總結」的地方，且它讀的是**已被程式排好、被懷疑者篩過**的乾淨清單——變異被擋在前面兩道。

---

## 為什麼這樣穩（變異控制）

把「判斷的變異」收進三處確定性，只留最後一哩給 LLM：
1. **正交透鏡 + 版本化框架**：誰來掃、用什麼角度，是設計好的，不是 LLM 隨機發揮。
2. **schema 鎖死輸出**：每個 verdict 強制封閉列舉（真/假、飽和/否、now/near/far）+ 自由文字理由——借 SSR 精神「先文字判斷再映射刻度」，不直接要分數。
3. **程式去重 + 程式算分**：數量、排序、分層全是 JS，LLM 不碰算術。

→ 同框架重跑，top/bottom 穩定；這是 [指標雜訊底先測](../ailab/patterns/指標雜訊底先測.md) 同源的低變異哲學。

## 兩個真實戰績（2026-06-27 同日）

| 案例 | 探索什麼 | 規模 | 收斂結果 |
|---|---|---|---|
| **數位孿生市場模擬** | 27 個生意點子在「買方／賣方」兩側市場的成功率 | 13+10 真 agent | 揪出「賣錯邊」陷阱：A 類情報層買方 3.5% → 賣方 82% |
| **賣鏟子需求地圖** | 「每人有數隻 agent 後」湧現的新裝備需求 | 6 透鏡 ×4 輪 = 112 候選 → 36 驗證 agent | 5 層裝備堆疊 + 5 個 priority-100 卡位點 |

**最強驗證訊號**：賣鏟子那次，「per-agent 成本歸因+熔斷」從 **3 個不相干的透鏡各自獨立湧現**——獨立路徑撞同一答案，代表那是真瓶頸，不是腦補。**透鏡正交時，重複出現＝高信度。**

## 可複製骨架（Workflow 偽碼）

```
① const seen = new Map()
   let fresh = absorb(await parallel(LENSES.map(掃一個透鏡)))   // 第一輪
   let round = 1
   while (fresh >= 門檻 && round < K) {                          // loop-until-dry
     const gaps = await agent(完整性評論者, 已知清單)            // 找盲點
     fresh = absorb(await parallel(gaps.map(掃新切角)))
     round++
   }
② const list = [...seen.values()]                               // 程式去重後的獨特清單
③ const chunks = 切塊(list, 8)                                  // 批次省成本
   const verdicts = await parallel(chunks.map(一個懷疑者審一塊))
④ const scored = list.map(算確定性優先序).filter(過篩)          // JS 算分
   const map = await agent(合成, scored.top(20))                // 產地圖
   return { live, dropped, synthesis: map }
```

> 完整可跑腳本：`products/strategy/twin-market/`（賣鏟子＝`agent-era-demand-scout`、孿生市場＝`twin-market-sim`）。配對 skill：[數位孿生市場模擬](個人 skill)。

## 採坑點（別重蹈）

- **透鏡不正交＝白燒 token**：六副重疊的眼鏡挖出同一批前 10 名。下手前先檢查每副眼鏡的角度是否真的不同。
- **少了 loop＝漏長尾**：只跑第一輪會錯過盲點層的真金。一定要有「完整性評論者 → 補洞」這一圈。
- **少了對抗驗證＝一堆性感空想**：發散階段 agent 會給你很多「聽起來很潮」的需求。沒有默認懷疑的驗證關，假貨會混進結論。賣鏟子那次 112 個剔掉 40 個（保險精算類／球員兼裁判類／雙邊冷啟動類），**剔除清單本身就是一張「別人會掉的坑」地圖**。
- **讓 LLM 算分／去重＝引入變異**：數量與排序一定走程式。
- **一候選一 agent＝燒爆**：批次切塊，N 個 agent 各審 M 項。
