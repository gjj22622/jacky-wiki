# Week 3｜修 — 殘酷測試（Agent 扛得住真實世界嗎？）

> **時長**：2 小時（110 min 含 10 min buffer）
> **使用版本**：v3（v1/v2 詳見 OneDrive；v3 新增交叉測試 + 上傳機制 + 醫美加碼）
> **上課人數**：7 人（Sophia、潘潘、蕙瑄、岱恩、藝嘉、Barbie、詩媛）
> 來源：`02_課程架構與教案/03_Week3_課程架構_殘酷測試_v3.md`

---

## 一句話定位

> **自己以為做完的 Agent，離「能用」還有多遠。**

殘酷測試不是考倒學員，而是健康檢查。三個關鍵翻轉：

1. **做出來 ≠ 能用** — 判斷邏輯有沒有覆蓋真實情境？
2. **修 Agent 比建 Agent 更重要** — 學習發生在「發現漏洞 → 現場修復」
3. **失敗是資產** — 每個被測出來的 bug 都是 Agent 進化的燃料

> 三色情境定義（🟢 標準 / 🟡 邊界 / 🔴 錯誤）見 [AGENTS方法論落地.md §3.4](../concepts/AGENTS方法論落地.md)。

## v3 核心變更（相對 v2）

| 項目 | v2 | v3 |
|---|---|---|
| 測試模式 | 各測各的 | **Phase A 自測 → Phase B 交叉測別人的** |
| 結果輸出 | 口頭回報 | **每輪上傳 Agent 產出物到共享資料夾** |
| 真實案例 | 全虛構 | **新增醫美診所真實案例作為加碼情境** |
| 時間分配 | 標準 20 + 邊界 30 + 錯誤 20 | **Phase A 30 + Phase B 15 + 錯誤 15** |

## 測試流程：自測 → 上傳 → 交叉測 → 上傳

```
┌─ Phase A：自測 ────────────────────────────────────────┐
│  同一情境，各測各的 Agent → 上傳產出物到共享資料夾       │
│  學習：看見「同一需求，不同 Agent 的處理品質差多少」      │
└────────────────────────────────────────────────────────┘
┌─ Phase B：交叉測 ──────────────────────────────────────┐
│  指定配對，用別人的 Agent 跑同一情境 → 上傳            │
│  學習：用別人的 Agent 看見「自己 Agent 沒想到的」         │
└────────────────────────────────────────────────────────┘
```

## 模組 2｜Phase A 三輪個人化情境

> 每人的測試情境以自己繳交的 Skill 名稱命名，針對 Skill 中的具體判斷規則設計。
> 完整題目見 `07_殘酷測試素材/Week3_個人化殘酷測試_全員.md`。

| 輪次 | 類型 | 設計原則 |
|---|---|---|
| R1 | 🟢 標準 | 測 Skill 的基本功能是否跑得完 |
| R2 | 🟡 邊界 | 測 Skill 中明確寫的判斷規則會不會被觸發（合規三層掃描、emoji 禁令、尊嚴防線、廣告禁詞） |
| R3 | 🔴 錯誤 | 故意踩爆所有攔截機制，看有沒有漏網之魚 |

每輪節奏：投影情境卡（1 min）→ 各自輸入 Agent 測試（5 min）→ 截圖上傳（2 min）。

## 模組 5｜現場修復 + Sub Agent 種子

當學員發現「我的 Agent 想做太多事，結果什麼都做不好」時，**這是引出 Sub Agent 的最佳時機。**

Jacky 用比喻帶觀念：

> 「如果你的主 Agent 可以說：『這個判斷我不夠專業，讓我叫一個專門處理危機的 sub agent 來。』它只負責『判斷要不要叫人』，專業的事讓專業的 Agent 做——這就是 sub agent 的概念。」
>
> 「下週正式教。這週你們先記住：**如果 Agent 做太多事做不好，那不是 Agent 的問題，是你該拆成主 Agent + Sub Agent。**」

不深入教，只種下「Agent 可以呼叫 Agent」的種子。

## 模組 6｜醫美診所加碼任務（課後）

> **背景**：Jacky 和 Sophia 拜訪了一家醫美診所，Sophia 的接案 Agent 已產出完整文件。第一次讓全班 Agent「串起來」。

| 學員 | 任務 | 輸入來源 |
|---|---|---|
| 潘潘 | 用社群貼文 Agent 為醫美診所產出 1 週 FB+IG 貼文 | Sophia 的品牌策略 + 社群排程 |
| 蕙瑄 | 用 3 Agent 接力產出 2 篇貼文 | Sophia 品牌策略 |
| 岱恩 | 用 4 Agent 產出 1 篇貼文 | Sophia 品牌策略 |
| 藝嘉 | 用 Skill 產出 1 篇貼文（**注意醫療廣告法規**） | Sophia 品牌策略 |
| Sophia | 整理接案 Agent 完整產出 + 自評 | 拜訪紀錄 |
| Barbie | 用工作人格 Agent 審查 Sophia Agent 的產出 | Sophia 完整產出 |
| 詩媛 | 用工作人格 Agent 審查策略文件邏輯 | Sophia 策略文件 |

> **教學意圖**：Sophia 接案 Agent 產出 → 其他人 Agent 輸入 → Barbie/詩媛 人格 Agent 品管。**這就是 Agent 工廠的真實運作模式。**

醫美廣告法規提醒：術前術後照片使用規範、療效宣稱限制、醫療廣告法規。

## v3 教學閉環

> Week 1 學「拆」→ Week 2 學「建」→ Week 3 學「修」。**會修的人，才是真正會建 Agent 的人。**
>
> v3 三層遞進：**虛構品牌測基本功 → 交叉測看差異 → 真實客戶測實戰力**。
>
> 醫美診所加碼任務，第一次讓全班 Agent 形成真正的工作流——接案 → 社群接力 → 人格品管。**這就是双云 Agent 工廠的雛型。**
