# 旅行儀表板系統

> 互動式 HTML 儀表板。雖然命名為「旅行儀表板」，**實際容納所有類型的時間軸主題**——旅行、品牌、人物。
> 來源：`travel-longstay-dashboard.skill` + `旅行儀表板/`

---

## 分類邏輯

所有互動式 HTML 儀表板統一放在 `旅行儀表板/` 資料夾下（**不要在專案根目錄另建資料夾**）。

**命名規則**：`{主題}_{描述}_{時間範圍}` 或 `{年份}_{主題}_{地點或天數}`

> ⚠️ 2021 年因 COVID 三級警戒無旅行紀錄，故跳過。

## 每個儀表板的檔案結構

```
{年份}_{主題}/
├── index.html      ← 三頁籤儀表板（摘要、地圖、相簿），直接開啟
├── data.json       ← 擷取的貼文資料（日期、文字、圖片清單）
├── day_meta.json   ← 每日摘要、封面照索引、地點座標
├── photos/         ← 原始照片
└── thumbs/         ← 300×300 縮圖
```

### 三頁籤設計

1. **摘要**：每日卡片（日期 + 封面照 + 摘要）
2. **地圖**：基於 `coordinates` 在地圖上標註點位
3. **相簿**：所有照片瀏覽

## 產生流程（3 步驟）

> 完整步驟見 [新增儀表板 SOP](../sop/新增儀表板.md)。

### Step 1: extract — 從 Facebook 擷取貼文

```bash
python scripts/extract_trip_posts.py \
  --start-date YYYY-MM-DD \
  --end-date YYYY-MM-DD \
  --semantic-filter         # 自動排除無關貼文（純工作、投資、轉發連結）
```

### Step 2: day_meta — 整理每日摘要

人工或 AI 整理 `day_meta.json`：每日摘要、封面照索引、地點座標。

### Step 3: build_dashboard — 產生 index.html

```bash
python scripts/build_dashboard.py \
  --start-date YYYY-MM-DD   # 控制 Day 1 = 旅行起始日
```

## 關鍵設計決策

### 靜態渲染（重要！）

> ⚠️ **HTML 必須是靜態渲染**——所有卡片、相簿直接寫在 HTML 中，**不能用 client-side JS 讀取嵌入的 JSON 再動態渲染**。
>
> （2026-03-22 林小眼儀表板第一版顯示異常的根因。）

### 照片保留 Facebook 原始檔名

> ⚠️ 不要重新命名照片，保留 Facebook 原始檔名。便於日後追溯與資料對應。

### Day 編號照日曆算

> Day 1 = 旅行起始日，**即使某天沒有貼文也會跳號**。日曆連續性 > 貼文連續性。

### 地圖磚

> 使用 **CARTO Voyager**（不要用 OpenStreetMap 原生）。
> `file://` 協定會被 OSM 的 Referer 擋。

### 開啟方式

> **必須從 Windows 檔案總管直接雙擊 `index.html`**——不要從 Cowork 的 `computer://` 連結開，否則圖片不顯示。

## 分享方式

整個資料夾**壓縮成 zip 即可分享**，收到的人解壓後開 `index.html` 就能看（無需伺服器）。

## 線上部署版本（Zeabur）

線上版本透過 Flask app.py 動態路由：

```
/travel/<folder>/        ← 旅行儀表板入口
/travel/<folder>/<path>  ← 靜態檔案（照片、JSON）
/people/<folder>/        ← 人物儀表板入口
/people/<folder>/<path>  ← 靜態檔案
```

GitHub repo 結構：

```
jacky-memoir/
├── dashboards/
│   ├── travel/    ← 旅行（含 Jwood 品牌旅程）
│   └── people/    ← 人物（林小眼）
└── app.py         ← TRIP_META、PEOPLE_META、路由
```

> 新增儀表板時必須同步更新 app.py 的 `TRIP_META` 或 `PEOPLE_META` 列表。詳見 [新增儀表板 SOP](../sop/新增儀表板.md)。
