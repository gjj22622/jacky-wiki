# 簡報大綱：AI 是新作業系統（16 張）

1. **封面** — 「AI 是新作業系統」｜大家在學工具，你在換 OS
2. **痛點開場** — 大家還在焦慮「要不要學 AI」；真正發生的是工作 OS 換代
3. **視角轉換** — 工具（一次呼叫）vs 作業系統（持續運行、有記憶、會調度）對照表
4. **不是修辭** — Karpathy Software 3.0：context=RAM、weights=CPU、prompt=程式語言
5. **業界共識** — 微軟 Copilot「intent as kernel」、a16z/Goldman「2026 代理 OS 時代」
6. **四種介面（一）** — 終端機（最常用）＋ desktop：可程式化的 shell 與長任務
7. **四種介面（二）** — web（發想/起稿）＋ 手機（隨身/續跑）；同一套 OS 四種 UI
8. **四介面實證** — Cursor CLI：同核心、IDE/終端/Web/Mobile 無縫切換
9. **兩大障礙（一）** — 環境建立：終端/CLI/Docker/MCP/權限（46% 整合困難）
10. **兩大障礙（二）** — 使用流程：每日入口、做完流程、驗收（沒流程=只是好玩的對話）
11. **跳級時刻** — 兩件事建好 → 套回原工作流 → 工作流程大升級
12. **架構層對照** — context=RAM、Agent=行程、MCP=syscall、Memory/Wiki=檔案系統｜AGENTS 腦+手
13. **帶新人（核心）** — 新人不是學工具，是進駐一套 OS（像當年學 Windows）
14. **帶新人三步** — 換視角 → 建環境 → 定流程
15. **採坑點** — 逐個教工具越教越焦慮／沒建環境就上工／對話當主場／忽略治理
16. **收束** — 双云 AI School＝這套作業平台的入口；先換 OS 視角，再談工具
