# 自建程式清單

> Jacky 自己寫的、非標準套件能裝回來的東西：自建工具、script、Claude Code 客製設定。
> 這些**換機不會自動回來**，要手動複製或重建。

---

## `~/bin/`（自建/自備工具）

| 檔案 | 用途 | 怎麼補回 |
|---|---|---|
| `ffmpeg.exe` | 影音轉檔（tts-script 等 skill 用到）| 從 https://ffmpeg.org 下載放回 `~/bin/`，或 `winget install Gyan.FFmpeg` |

> `~/bin/` 不在 git 內。換機後把需要的執行檔放回這個資料夾，並確認它在 PATH 裡（或用絕對路徑呼叫）。

---

## Claude Code 客製設定（`~/.claude/`）

| 檔案 | 用途 | 換機怎麼處理 |
|---|---|---|
| `~/.claude/settings.json` | 全域設定：權限模式、hooks（SessionStart/End）、statusLine、啟用的 plugin | **手動複製**（這份不在 wiki，建議自己備份一份）|
| `~/.claude/settings.local.json` | 本機權限白名單 | 不複製，新機重設（含機敏允許清單）|
| `~/.claude/statusline-command.js` | 自訂狀態行（顯示 context/token/成本）| 手動複製到新機同位置 |
| `~/.claude/CLAUDE.md` | 全域 skill 觸發指南 | 手動複製（或從本 wiki 對照重建）|
| `~/.claude/skills/` | 16 支個人 skill | **由 wiki 還原** → 見 [skill總索引](../skills/skill總索引.md) |

> ⚠️ `settings.json` 內可能含本機路徑/設定，複製後檢查一遍。**不要把含機密的設定推進 public repo。**

---

## SessionStart / SessionEnd hooks（已設在 settings.json）

| Hook | 作用 |
|---|---|
| **SessionStart** | 開工自動顯示 `./todolist.md` 前 30 行（進度看板）|
| **SessionEnd** | 收工檢查 git 狀態，有未提交變動就提醒 |

> 這些靠 `settings.json` 帶過去。換機複製 settings.json 後即生效（PowerShell 指令型 hook，Windows 適用）。

---

## 換機重建順序

1. clone wiki → skill 從 wiki 還原（[skill總索引](../skills/skill總索引.md)）。
2. 手動複製 `~/.claude/settings.json` + `statusline-command.js` + `CLAUDE.md`（自己的備份，或從舊機拷）。
3. `~/bin/` 內執行檔（ffmpeg 等）重新下載放回。
4. 跑 [還原檢查清單](../migration/還原檢查清單.md) 驗收。
