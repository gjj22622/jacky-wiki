# /排查 — 簡報最後排查（雙視角體檢）

把一份「自認快定稿」的簡報，當成上場前一晚的最後一道關。
**先機械、後人工；先抓硬傷、再評精采。** 不重做簡報，只出體檢報告＋修補清單。

## 觸發後流程

### 0. 鎖定檔案
- 使用者給路徑就用該檔；沒給就在工作目錄找最新／含 `Final`／最高版號的 `.pptx`，並回報「我體檢的是哪一份」讓對方確認。
- **聖域原則**：只讀不改原檔（見記憶 [[rebuild-never-overwrite]]）。要修補一律輸出清單，由人決定。

### 1. 機械層（先跑，抓機器看得到的硬傷）
```
python "C:\Users\gjj22\.claude\skills\deck-final-check\check_deck.py" "<簡報.pptx>"
```
若該機沒有對應 python-pptx，直接用 Read/Glob 把每頁文字抽出來人工核。機械層查三件：
- **頁碼一致性**：footer 總數是否單一、是否＝實際張數、顯示序號有無倒退/跳號（混版殘留最常見的破綻）。
- **字級紅線 <16pt**：中小企業主受眾鐵則（見記憶 [[sme-audience-visual-rules]]）。
- **「不是…是…」對比句式**：Jacky 已退掉的 AI 腔（見記憶 [[no-ai-contrast-tic]]），列出讓人改肯定句。

### 2. 人工層 — 抽全文
把每頁文字抽成 `P1 … Pn` 清單（python-pptx 或 Read），作為下面雙視角評估的底稿。

### 3. 雙視角六維評估（本 skill 的核心）

對**同一份簡報**跑兩個獨立人格，再合議。每一維都要 **指名頁碼**、給 **嚴重度（🔴必修／🟡建議／🟢加分）**。

**視角 A：Jacky-wiki（講者本人 × 方法論）**
立場：這是我的招牌、我要站著講三小時。依 Jacky 演講結構偏好（見記憶 [[jacky-speech-structure]]：痛點開場→解方收尾、前段先對焦拉齊）與 SPEAKS/SOSTAC/AGENTS 方法論。問：
- 主題對焦了嗎？每一頁都在還「主標題」這筆債嗎，有沒有離題的炫技頁？
- 我的敘事弧線站得住嗎？痛點→地圖→實戰→收尾，轉場順不順、有沒有斷層？
- 招牌爆點（埋梗→揭曉）有沒有踩在情緒高點，還是太晚/太散？
- 講者負擔：哪頁字太多會逼我念稿、哪頁我會忘詞、哪個數字我得背得出來源？

**視角 B：J董-wiki（受眾 × 數位幹部審稿）**
立場：我是台下 40–60 歲、76% 是主管的老闆，也是替 Jacky 把關的數位幹部。問：
- 第一個 20 分鐘抓得住我嗎，還是在看你的證照牆（自介過長／字海）？
- 每一段有沒有「給我一個帶得走的東西」？鉤子（提問、數字、案例、禮物）密度夠不夠、有沒有連續幾頁乾掉？
- 精采度：哪頁我會拿手機拍、哪頁我會放空？真實案例的數字可信嗎？
- 邏輯謬誤獵殺：未證實統計（如「99% 的人…」）、訴諸權威的引文（名人語錄是否真有出處、日期對不對）、假二分（「企業只會分兩種」）、以偏概全、循環論證（「這份簡報是 AI 做的」當證據時，簡報本身不能有破綻，否則反殺）。

### 4. 合議與輸出
產出一份報告，結構固定：

```
# 簡報最後排查報告 — <檔名>（<張數> 張 / <時長>）

## 一句話結論
（能不能上場？最大風險是什麼？）

## 機械層硬傷
- 頁碼 / 字級 / 對比句式（貼 check_deck.py 結果，標頁碼）

## 六維評估
| 維度 | Jacky 視角 | J董 視角 | 嚴重度 |
| 符合主題 | … | … | 🟢/🟡/🔴 |
| 爆點安排 | … |
| 鉤子密度 | … |
| 精采度   | … |
| 連貫一致 | … |
| 邏輯謬誤 | … |

## 修補清單（依嚴重度排序，逐條可執行）
🔴 必修：…（頁碼＋改法）
🟡 建議：…
🟢 加分：…

## 不動的部分（明確背書，免得對方亂改好的）
```

## 紅線與原則
- **只讀不改原檔**；修補給清單，動手前另開時間戳新檔。
- **指名頁碼、給嚴重度、給改法**——不要只說「不夠精采」這種沒有著力點的話。
- **雙視角要真的吵架**：兩個人格看法不同時兩個都寫出來，由使用者裁決，不要自己和稀泥。
- 邏輯謬誤這一維，對「名人引文／統計數字／因果宣稱」一律要求講得出出處，講不出就標 🔴 verify。
- 簡報文字鐵則延用：可編輯、最小 16pt、禁 AI 對比腔（見記憶 [[deck-text-must-be-editable]]）。
- 配對 skill：定稿後要轉語音走 `/tts`；要查 Jacky 方法論底稿走 `/wiki`。
