# L3 建・最小 Agent

**主題**：建（AGENTS 的 G）｜**4 課**｜對應原 Week2「建」後半
**定位**：從「聊天」跨到「Agent」——一個可交接、可檢查、重跑一致的最小單位。

## 課程

### L3-01 聊天 vs Agent（2:02）
> 鉤子：**聊天靠運氣，Agent 可交接可檢查**
聊天問一句答一句，下次要重來：快，但不穩定、不能交接、沒辦法檢查。

### L3-02 只做一個決策（1:51）
> 鉤子：**一個 Agent 只做一個決策**
Agent 設計最重要的鐵則，也是新人最常踩的坑。

### L3-03 最小 Agent 說明書（1:58）
> 鉤子：**讓別人照著跑也能跑出一樣結果**
把範圍寫成說明書，用木酢 IG 貼文初稿檢查 Agent 當範例，一欄一欄填。

### L3-04 驗收重跑一致（1:55）
> 鉤子：**重跑要一致才算數**
Agent 好不好用要驗收過才算數，這是新人變專業的關鍵動作。
