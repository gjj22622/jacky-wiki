# Week 2｜建 — 從工作流程到 Agent 設定檔

> **時長**：2 小時
> **前提**：每位夥伴已完成第一週作業（自己的工作流程拆解）
> **核心動詞**：建 — 把流程拆解表變成 Agent 設定檔
> 來源：`02_課程架構與教案/01_Week2_課程架構_教案設計.md`

---

## 一句話定位

把腦袋裡的工作方式，**寫成一份 Agent 看得懂的設定檔**——一份設定檔，四個平台通用。

## 開場處理：作業必有的兩件事

> **Jacky 教學意圖**：重複不是浪費，是發現「團隊共用流程」的機會；不完整不是失敗，是「隱性知識還沒被問出來」的訊號。

開場用三階段篩選機制把這兩件事當成 feature：

| 階段 | 動作 | 產出 |
|---|---|---|
| A · 課前 AI 自動分類 | 把所有人拆解表丟給 AI，做 ① 分群（同一件事歸一群）② 6 維度完整度評分 | 流程分群表 + 每人完整度報告 |
| B · 課堂公開展示（10 min） | 投影分群結果 — 看見「5 個人都在做月報」<br>現場指定每群的「流程負責人」 | 共用流程清單 |
| C · 補完工作坊（15 min） | AI 讀拆解表 → 主動提問漏掉的判斷邏輯／資料來源 → 學員回答補完 | 完整度從 60 → 85 分 |

---

## 課程架構（6 模組）

### 模組 1｜作業回顧與流程篩選（25 min）

- 1-1 展示分群結果（10 min）— 投影、指定流程負責人
- 1-2 完整度補完（15 min）— AI 對話補完缺漏

> **銜接關鍵**：模組結束時，每人手上要有一份「夠完整的」流程拆解表——判斷邏輯和資料來源不能空白。

### 模組 2｜Agent 設定檔格式教學（15 min）

**核心概念：一份設定檔，四個平台通用**

通用設定檔結構（建議區塊）：

```
[Agent 設定檔]
■ 基本資訊（名稱 / 所屬流程 / 對應步驟 / 功能定位）
■ 角色指令（你是誰 / 任務 / 限制）         ← 核心
■ 觸發條件（什麼時候叫你 / 什麼時候不是你的事）
■ 輸入 → 輸出（需要什麼資料 / 交出什麼 / 格式）
■ 判斷規則（如果 ___ 就 ___ / 不確定怎麼辦）
■ 知識庫需求（參考資料 / 放在哪）
■ 平台部署對照（Gem / GPTs / Skill / NotebookLM）
```

> **NotebookLM 的定位**：不是「執行 Agent」的平台，而是「設定檔的知識庫查詢工具」——可以問它「我的月報 Agent 的判斷規則是什麼」。

### 模組 3｜單一步驟拆 Agent 的原則（15 min）

**為什麼「一步一 Agent」，不要「一個大 Agent 做全部」？**

```
反例：「你是客戶服務 Agent，負責接收需求、分析、撰寫提案、報價、跟催、結案。」
     → 什麼都做 = 什麼都做不好

正例：
  Agent 1：需求釐清 Agent
  Agent 2：提案架構 Agent
  Agent 3：報價試算 Agent
  Agent 4：跟催提醒 Agent
     → 每個都專精 = 每個都可靠
```

**三個拆分原則**

1. 一個 Agent 只做一個決策（兩個判斷點 → 拆兩個 Agent）
2. 輸出要能被下一個 Agent 接住（A 的輸出 = B 的輸入）
3. 可以獨立測試（不依賴前面的 Agent）

> **銜接關鍵**：這個觀念是整堂課的核心。理解了「一步一 Agent」，後面實作就會順。

### 模組 4｜現場實作：寫第一份 Agent 設定檔（30 min）

| Step | 時間 | 動作 |
|---|---|---|
| 1 | 5 min | 從拆解表選一個步驟（最重複、最花時間、最容易出錯的那一步；不要選最複雜的） |
| 2 | 10 min | 用模板填寫設定檔，重點放「角色指令」和「判斷規則」 |
| 3 | 10 min | 選一個平台部署 + 用 3 種測試情境驗證（標準 / 邊界 / 錯誤） |
| 4 | 5 min | 存到指定的雲端資料夾 |

> **銜接關鍵**：實作結束時，每人至少有 1 份設定檔部署成功並通過測試。

### 模組 5｜雲端資料夾結構與命名規範（15 min）

**雲端資料夾建議結構**

```
双云 Agent 庫/
├── _索引表.xlsx                     ← 全公司 Agent 總覽
├── _模板/                           ← 流程拆解表 / 設定檔 / 進化日誌 模板
├── 【共用】/                        ← 團隊共用 Agent
│   └── 客戶月報產出/
│       ├── 01_需求確認Agent.md
│       ├── 02_數據彙整Agent.md
│       ├── _Dataroom/
│       └── _進化日誌.md
├── 【個人】_王小明/                 ← 個人專屬 Agent
└── ...
```

**命名規範**

| 規則 | 格式 | 範例 |
|---|---|---|
| Agent 設定檔 | `序號_功能名Agent.md` | `01_需求確認Agent.md` |
| Dataroom | `_Dataroom/` | 底線開頭，置頂 |
| 進化日誌 | `_進化日誌.md` | 底線開頭，置頂 |
| 流程拆解表 | `流程名_拆解表.md` | `客戶月報_拆解表.md` |

**索引表欄位**：Agent 名稱 / 所屬流程 / 負責人 / 部署平台 / 版本 / 最後更新 / 狀態

### 模組 6｜課後任務與下週預告（10 min）

**本週作業**

1. 完成至少 3 份 Agent 設定檔（涵蓋你最常做的 3 個步驟）
2. 全部存入雲端資料夾，遵守命名規範
3. 每份設定檔至少在一個平台部署並測試過
4. 開始寫進化日誌（至少記錄 3 次使用）

**下週預告**：Agent 間串接與協作、團隊 Agent 庫 Review。
