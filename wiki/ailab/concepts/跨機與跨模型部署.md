# 跨機與跨模型部署 ⭐

> **目的**：讓 Jacky 不管在哪台機器、用哪個 AI 模型／工具，都能用同一份 ailab 協定記錄實踐事件。
> **核心策略**：**協定（markdown）** 是模型無關的真相；**Skill / Gem / Custom Instructions** 是各環境的執行包裝。

---

## 部署矩陣總覽

| 環境 | 觸發方式 | 部署形式 | 持久性 | 詳細 |
|---|---|---|---|---|
| **Claude Code（任機）** | `/ailab capture` `/ailab session` | `~/.claude/skills/ailab/SKILL.md`（symlink/copy 自 wiki）| 永久 | [§1](#1-claude-code多機windows--ubuntu--wsl2--macos) |
| **Codex CLI（GPT-5.4）** | 對話自然語言 + 自動載入 instructions | `~/.codex/AGENTS.md` 加實踐捕手段 | 永久 | [§2](#2-codex-cli) |
| **Gemini（Gem）** | 在 Custom Gem 對話 | Gemini Gem system instructions | 永久 | [§3](#3-gemini-gem) |
| **Gemini Web / ChatGPT / Claude Web** | 複製 prompt 到對話框 | 純 prompt template，不持久 | 臨時 | [§4](#4-web-aigemini--chatgpt--claude-web) |
| **手機／無 AI 環境** | 三行極簡格式 | 直接寫 OneDrive | 臨時 | [§5](#5-手機極簡記錄) |

> **共同回流路徑**：所有環境最後都把事件丟到 OneDrive/ailab-inbox/ 或 wiki/ailab/inbox/，
> 回 Claude Code 跑 `/ailab sync` + `/ailab promote` 統一處理。

## 1. Claude Code（多機：Windows / Ubuntu / WSL2 / macOS）

### 一次性安裝

詳細步驟見 [INSTALL.md](../skill/INSTALL.md)。簡言之：

**Linux / macOS / WSL2**（symlink，自動跟 git pull 同步）：

```bash
git clone https://github.com/gjj22622/jacky-wiki.git ~/jacky-wiki
mkdir -p ~/.claude/skills/ailab
ln -sf ~/jacky-wiki/wiki/ailab/skill/SKILL.md ~/.claude/skills/ailab/SKILL.md
```

**Windows PowerShell**（copy，每次 git pull 後手動同步）：

```powershell
git clone https://github.com/gjj22622/jacky-wiki.git $env:USERPROFILE\jacky-wiki
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\.claude\skills\ailab"
Copy-Item "$env:USERPROFILE\jacky-wiki\wiki\ailab\skill\SKILL.md" `
          "$env:USERPROFILE\.claude\skills\ailab\SKILL.md" -Force
```

### G2 mini Ubuntu 範例（針對 Jacky 的具體機器）

```bash
# 確認 wiki 已 clone（CLAUDE.md 提到 G2 mini Ubuntu 已有環境）
ls ~/jacky-wiki || git clone https://github.com/gjj22622/jacky-wiki.git ~/jacky-wiki

# 安裝 ailab Skill
mkdir -p ~/.claude/skills/ailab
ln -sf ~/jacky-wiki/wiki/ailab/skill/SKILL.md ~/.claude/skills/ailab/SKILL.md

# （可選）設環境變數
echo 'export JACKY_WIKI_HOME="$HOME/jacky-wiki"' >> ~/.bashrc
source ~/.bashrc

# 驗證：開 Claude Code 打 /ailab，應該有反應
```

### 日常更新

任一機器改了 wiki/ailab/skill/SKILL.md 並 push 後：

```bash
# 其他機器
cd ~/jacky-wiki && git pull --rebase origin main
# 完成（symlink 直接跟；copy 要重跑 cp）
```

## 2. Codex CLI

### 為什麼 Codex 不能直接用 Skill

Codex CLI 沒有 Claude Code 的 Skill 系統，但有 `~/.codex/AGENTS.md`（或類似的 system prompt 載入機制），可以放長期指引。

### 部署方式：把協定加進 `~/.codex/AGENTS.md`

```bash
# 把 ailab 協定段附加到 Codex AGENTS.md
cat <<'EOF' >> ~/.codex/AGENTS.md

---

# Jacky AI 實踐捕手協定（自動載入）

當 Jacky 說「實踐捕手」「ailab capture」「捕一份」「ailab session」「對話結尾總結」，
請用以下兩種模式之一輸出 markdown，**模型無關格式**，輸出後 Jacky 會手動貼到 OneDrive。

## Mode A：單一事件（9 欄位）

---
event: <30 字內>
date: <YYYY-MM-DD>
model: gpt-5.4 (codex-cli)
context: <用在哪個專案>
type: <突破|失敗|工具發現|模式發現|模型對比|工作流改進>
maturity: <想法|實驗中|已驗證|已穩定>
tags: <逗號分隔>
---

## 發生了什麼
## 為什麼重要
## 怎麼做的
## 對比與替代
## 連結與出處
## 升格目標

## Mode B：對話結尾總結（5 區塊）

---
session_date: <YYYY-MM-DD>
session_topic: <一句話>
model: gpt-5.4 (codex-cli)
context: <專案>
duration: <可選>
type: session-summary
tags: <逗號分隔>
---

## 最終做法（What Worked）
## 繞路紀錄（Detours）
## 錯誤與失敗（What Failed）
## 升格候選
## 待延伸（Next）

## 完整協定參考

`~/jacky-wiki/wiki/ailab/concepts/實踐捕手協定.md`
EOF
```

→ 之後在 Codex CLI 對話中只要說「捕一份」或「session 總結」，Codex 就會輸出標準格式。

### 把 Codex 輸出存到 inbox

Codex CLI 支援檔案系統，可以直接把輸出寫入：

```bash
# Codex 對話中
> 捕一份這次實踐：今天用 gpt-5.4 跑跨檔重構比 sonnet 4.6 更穩
# (Codex 輸出 Mode A 格式)

# Jacky 把輸出存進 wiki inbox
```

或讓 Codex 直接寫檔（如果 Codex 有寫檔能力）：

```
> 用 ailab Mode A 格式把剛才的事件寫到 ~/jacky-wiki/wiki/ailab/inbox/2026-05-02-codex-跨檔重構.md
```

### 同步回 wiki

不需要 sync——Codex 已直接寫到 wiki repo，下次 git commit + push 就同步到其他機器。

## 3. Gemini Gem

### 部署方式：建一個 Custom Gem

1. 開 Gemini Web，進入 Gem Manager（左側選單）
2. 「Create new Gem」
3. 命名：「Jacky AI 實踐捕手」
4. **Instructions** 欄位貼以下內容：

```
你是 Jacky 的 AI 實踐事件記錄助手。
當 Jacky 給你描述一個 AI 實踐（突破、失敗、工具發現、對話總結等），
請用以下兩種模式之一輸出 markdown 格式（其他話都不要說）：

## Mode A：單一事件（9 欄位）

當 Jacky 描述「一個事件」用此模式：

---
event: <30 字內>
date: <YYYY-MM-DD>
model: gemini-2.5-pro
context: <Jacky 用在哪個專案>
type: <突破|失敗|工具發現|模式發現|模型對比|工作流改進>
maturity: <想法|實驗中|已驗證|已穩定>
tags: <逗號分隔>
---

## 發生了什麼
## 為什麼重要
## 怎麼做的
## 對比與替代
## 連結與出處
## 升格目標

## Mode B：對話結尾總結（5 區塊）

當 Jacky 說「對話結尾」「session 總結」「總結這次對話」用此模式：

---
session_date: <YYYY-MM-DD>
session_topic: <一句話描述對話主題>
model: gemini-2.5-pro
context: <專案>
duration: <可選>
type: session-summary
tags: <逗號分隔>
---

## 最終做法（What Worked）
（具體做法、prompt、命令）

## 繞路紀錄（Detours）
（試 A 不通因為 X → 改試 B）

## 錯誤與失敗（What Failed）
（撞牆事件、假設錯誤）

## 升格候選
（建議升 ailab/tools 或 patterns，或留 inbox）

## 待延伸（Next）
（下次要做什麼）

## 何時用哪個 Mode

- 對話中突然發現「這個值得記」 → Mode A
- 對話結尾／要切換主題 → Mode B
- 多事件混合且要保留「最終做法 vs 繞路 vs 錯誤」對照 → Mode B

## 重要

- 用繁體中文
- 不要編造資訊——只記錄 Jacky 真的做過的事
- 如果某欄位不確定，用 `<待補>` 標記，不要亂猜
- 輸出純 markdown，不要加額外解釋
```

5. 儲存 Gem
6. 之後每次需要記錄時，**從 Gem 入口**進對話，自然會用協定格式輸出

### 把 Gemini Gem 輸出存到 inbox

Gemini Web 沒法直接寫本機檔案。流程：
1. Gem 對話中說「捕一份」或「session 總結」
2. Gem 輸出 markdown
3. **Jacky 複製整段 markdown**
4. 貼到 `OneDrive/ailab-inbox/<日期>-<event>.md`
5. 之後在 Claude Code 跑 `/ailab sync`

## 4. Web AI（Gemini Web / ChatGPT / Claude Web）

### 不持久版：每次貼 prompt

如果不想建 Gem / Custom GPT，每次直接貼 prompt 給 AI。

#### Mode A 完整 prompt

```
我要用「實踐捕手協定 v1.1 Mode A」記錄剛才這個 AI 實踐事件。
請用以下 9 欄位格式輸出（其他都不要說）：

---
event: <30 字內>
date: <YYYY-MM-DD>
model: <你是哪個模型>
context: <我用在哪個專案>
type: <突破|失敗|工具發現|模式發現|模型對比|工作流改進>
maturity: <想法|實驗中|已驗證|已穩定>
tags: <逗號分隔>
---

## 發生了什麼
## 為什麼重要
## 怎麼做的
## 對比與替代
## 連結與出處
## 升格目標
```

#### Mode B 完整 prompt

```
我們這次對話準備告一段落（或要切換主題）。
請用「實踐捕手協定 v1.1 Mode B」幫我總結這次對話的 AI 實踐紀錄。
重點要保留「最終做法／繞路／錯誤」三方對照——其他都不要說：

---
session_date: <YYYY-MM-DD>
session_topic: <一句話>
model: <你是哪個模型>
context: <用在哪個專案>
duration: <大致時間>
type: session-summary
tags: <逗號分隔>
---

## 最終做法（What Worked）
## 繞路紀錄（Detours）
## 錯誤與失敗（What Failed）
## 升格候選
## 待延伸（Next）
```

### 持久版：建 Custom GPT / Custom Project

#### ChatGPT Custom GPT

ChatGPT Plus/Team/Enterprise 可建 Custom GPT，把上面的 prompt 設成 instructions（同 Gemini Gem 邏輯）。

#### Claude Web Project

Claude Web 有「Project」功能，可在 Project 的「Custom Instructions」放協定，之後該 Project 內所有對話自動套用。

## 5. 手機／極簡記錄

### 場景

- 在路上想到一個方法
- 跟人聊天時 AI 跑一個快速測試
- 沒法開 Claude Code

### 三行極簡格式

打開任何能寫文字的 app（OneDrive 手機版、備忘錄、Telegram saved messages），寫：

```
@ailab YYYY-MM-DD <model> | <event 一句話>
why: <為什麼重要，一句話>
how: <怎麼做，一句話>
```

範例：

```
@ailab 2026-05-02 gemini-2.5-flash | Gemini 影像辨識比 Claude 快 3x 但細節差
why: 大量影像分類任務，Gemini 是 Niche 候選
how: 把同 100 張圖丟 Gemini Web 跟 Claude，計時+人工檢核
```

### 同步路徑

1. 寫進 `OneDrive/ailab-inbox/raw.md`（在末尾追加）
2. 回 Claude Code 跑 `/ailab sync`
3. Claude 會把 raw.md 的條目補完成 9 欄位 Mode A 格式，寫到 wiki/ailab/inbox/

## 各環境對照表

| 維度 | Claude Code | Codex CLI | Gemini Gem | Web AI | 手機 |
|---|---|---|---|---|---|
| 觸發成本 | 低（`/ailab`）| 低（自然語言）| 低（在 Gem 內對話）| 中（每次貼 prompt）| 極低（隨手寫）|
| 寫檔能力 | ✅ 直接寫 wiki | ✅ 直接寫 wiki | ❌ 要手動複製 | ❌ 要手動複製 | ⭕ 寫 OneDrive |
| 持久設定 | ✅ Skill | ✅ AGENTS.md | ✅ Gem | ⭕（Custom GPT/Project）| ❌ |
| 同步路徑 | git push | git push | 複製→OneDrive→sync | 複製→OneDrive→sync | OneDrive→sync |
| 完整度 | 100%（Skill 自動讀協定）| 90%（要看 Codex 是否會主動讀）| 80%（人讀 prompt）| 70%（每次貼）| 30%（極簡，sync 補完）|

## 維護紀律

| 規則 | 說明 |
|---|---|
| ❗ 主版只在一處 | `<jacky-wiki>/wiki/ailab/skill/SKILL.md`——不要在各機分頭改 |
| ❗ Codex / Gem 設定也版控 | 把 Codex AGENTS.md 與 Gemini Gem instructions 也存到 `wiki/ailab/skill/` 內，多機共用 |
| ✅ 改協定就升版號 | v1.0 → v1.1 → v1.2，避免新舊混雜 |
| ✅ 設定改完通知所有環境 | 改 SKILL.md → 各 Claude Code 機 git pull；改 Codex prompt → 各 Codex 機 git pull；改 Gem → 重設 Gem instructions |
