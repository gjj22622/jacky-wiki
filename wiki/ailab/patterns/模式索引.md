# 模式索引

> Jacky 在多個場景反覆驗證、已穩定的 AI 實踐模式。
> **規則**：本目錄**只放連結 + 個人視角註記**，**不複製內容**——共用模式在 cross-domain，學術應用在 nchu，多模態在 jwood。

---

## 為什麼模式不在 ailab 寫完整版

| 維度 | ailab/patterns/ | 其他域（cross-domain / nchu / jwood）|
|---|---|---|
| 視角 | 我個人怎麼用、心得 | 適用所有人的 SOP |
| 內容 | 連結 + 個人註記 | 完整描述 |
| 例 | 「9 階段我用得最順的時序」 | 「9 階段工作流 SOP」 |

> ailab 是**廚房**，cross-domain 是**菜譜**——廚房試完才升菜譜。

### 已成文 6：Admin Token 管理後台設計法
- **完整規格書**：[admin-token-backend 規格書](admin-token-backend-規格書.md)（Jacky 標準後台設計法）
- 配對 skill：`/admin後台`（`admin-token-backend`），注入規格＋複製 `templates/` 樣板
- 來源：双云 AI School（Node＋JSON）＋ SPEAKS Coach（Python＋Postgres）各做一半，本規格合一並補業界最佳實務
- 核心：token 即去個資化行為追蹤＋四來源算力成本歸戶＋兩層權限（Admin 密鑰 × RBAC）＋稽核＋統一後台 UX

### 已成文 7：Zeabur CLI 自動部署方法
- **通用方法論**：[Zeabur 自動部署方法](zeabur自動部署方法.md)（任何專案適用，非單一專案）
- 三條黃金鐵則：deploy 上傳當前目錄(要 cd 源碼夾)／改 env 後 restart 沒用必須 deploy／上傳逾時 transient 重試
- 來源：AI 大腦上線實戰（從 tools/members 誤 deploy 變 404、restart 不重載 env 等血淚）
- 具體案例：[`docs/ai-brain-部署操作手冊.md`](../../../docs/ai-brain-部署操作手冊.md)

### 已成文 8：高併發活動撐流量設計法
- **通用方法論**：[高併發活動撐流量設計法](高併發活動撐流量設計法.md)（單 Node LLM 串流服務，~100 人實體活動前壓力盤點）
- 核心：扛不住的不是人數是**四個地雷**——①請求熱路徑禁同步 fs ②會場共用 IP 把按-IP 限流打爆 ③共用 token "kick" 互踢 ④對上游 LLM 無限併發→排隊優雅降級
- 心法：**「同時數」≠「人數」**；併發閘 N 設「同時生成數」(20~30)不是人頭，全做 env 可一秒調
- 含可複製的「生成併發閘」code（acquire/release 交棒＋佇列上限＋等待逾時＋client 斷線即 abort）＋上線前檢查清單
- 來源：TBSA Agents（agents.tbsa.tw）6/26 說明會上線前盤點

### 已成文 9：上線前容量壓測體檢法（AGENTS【修】進階版）
- **通用方法論**：[上線前容量壓測體檢法](上線前容量壓測體檢法.md)（任何服務上線/demo 前的效能容量驗證）
- 定位：AGENTS【修】(E 評測修正) 把「殘酷測試」從**內容正確性**延伸到**效能/容量**——「雙軌品管」長出效能軌
- 與「已成文 8」成對：**8＝建（為流量而建）／9＝修（上線前驗證）**
- 核心：**四層瓶頸**逐層拿數字——L1 伺服器整台 VM RAM（**不是容器**）、L2 build 尖峰會不會 OOM、L3 親手測最重操作秒數＋≤5 併發、L4 上游額度（**以後台為準，API 推算不可當事實**）
- 心法：「**護欄設定對 ≠ 體驗順**」；沒實測＝結論「未驗證」；配對 skill `/壓測`
- 來源（反推）：2026-06-26 agents.tbsa.tw demo 因 2GB OOM 當眾轉圈圈跑不出來
- 配對個人視角事件 → [inbox/2026-06-27-session-壓測skill與容量體檢法](../inbox/2026-06-27-session-壓測skill與容量體檢法.md)

## 模式升格的反向流動

某些 ailab 模式可能反過來升到結晶層：

```
ailab/patterns/<Jacky 個人視角> 累積對照
   ↓ 跨場景驗證 ≥ 2 次
cross-domain/<可教學 SOP>
   ↓ 多場景反推 ≥ 3 場景
shuangyun/AGENTS v1.x（結晶方法論）
```

> 12 個 v1.0→v1.1 新元素就是這個流動的歷史證據。
