# SPEAKS 六步驟詳解

> 每個步驟的核心問題、操作工具、Jacky 的提問模式。

## S — Scan（場景掃描）

> **核心問題：你要在什麼戰場上開口？**
> **SOSTAC 對應：Situation — Where are we now?**

不同場景，邏輯完全不同。

### S-1：聽眾解碼（Audience Decoding）

| 維度 | 關鍵問題 | 操作工具 |
|---|---|---|
| **決策角色** | 聽眾是決策者、影響者、執行者？ | 利害關係人矩陣（Power-Interest Grid） |
| **認知起點** | 對議題已知道多少？ | 知識差距評估（Knowledge Gap Assessment） |
| **心理狀態** | 帶著什麼期待或偏見進場？ | 預設立場掃描（Bias Scan） |
| **文化語境** | 高語境 / 低語境？直接 / 委婉？ | Erin Meyer 文化地圖 |

### S-2：場景類型判定（7 種）

| 場景 | 時長 | 核心任務 | 成功定義 |
|---|---|---|---|
| 電梯演說 | 30-90 秒 | 引發興趣 | 對方說「再多說一點」 |
| 企劃提報 | 15-30 分鐘 | 取得信任與預算 | 決策者批准或進入下一輪 |
| 策略簡報 | 10-15 分鐘 | 對齊決策方向 | 高層共識 + Action |
| 競賽口報 | 8-15 分鐘 | 展現完整度與實力 | 評審高分 |
| 客戶月會 | 20-40 分鐘 | 展示成效與下期規劃 | 客戶續約信心 |
| 危機溝通 | 不定 | 穩定信心、控制損害 | 利害關係人情緒穩定 |
| 知識傳授 | 60-180 分鐘 | 學員真正學會 | 能獨立複製 |

### Jacky 的提問

> 「你要報告的對象是誰？他最在意什麼？他進場前心裡已經有結論了嗎？如果有，你怎麼翻盤？如果沒有，你怎麼植入？」

## P — Purpose（目的錨定）

> **核心問題：聽眾聽完之後，做什麼、想什麼、感受什麼？**
> **SOSTAC 對應：Objectives — Where are we going?**

### P-1：口說三層目標模型

```
┌─────────────────────────────────────┐
│  第三層：行動目標（Action）           │
│  聽完之後，他們會「做」什麼？         │
│  例：批准預算、簽約、進入下一輪       │
├─────────────────────────────────────┤
│  第二層：認同目標（Belief）           │
│  聽完之後，他們會「相信」什麼？       │
│  例：相信我們的策略方向是正確的       │
├─────────────────────────────────────┤
│  第一層：理解目標（Understanding）    │
│  聽完之後，他們會「知道」什麼？       │
│  例：了解市場現況和我們的競爭優勢     │
└─────────────────────────────────────┘
```

> **設計原則**：由下而上**建構**，由上而下**驅動**。

### P-2 / P-3：SMART + OKR 口說目標

P-2 用 SMART 框架設定具體目標；P-3 用 OKR 處理重大／系列性口說。

### Jacky 的提問

> 「你這次口說的 Call to Action 是什麼？如果聽眾只記得一件事，那件事是什麼？**不要跟我說你的目的是『讓大家了解我們的企劃』——那不是目的，那是過程。目的是讓他們做決定。**」

## E — Engineer（架構設計）

> **核心問題：用什麼敘事架構，讓聽眾從「不知道」走到「願意行動」？**
> **SOSTAC 對應：Strategy — How do we get there?**

### E-1：金字塔原理（結論先行）

```
          ┌──────────┐
          │  核心結論  │ ← 開場 30 秒就要說出來
          └─────┬────┘
        ┌───────┼───────┐
   ┌────┴────┐ ┌┴──────┐ ┌┴──────┐
   │ 理由一  │ │理由二 │ │理由三 │ ← 三個，不多不少
   └────┬────┘ └───┬───┘ └───┬───┘
    數據/事實   數據/事實   數據/事實
```

> **MECE 防護**：三個理由相互獨立、完全窮盡。

### E-2：SCQA 開場設計

| 步驟 | 心理功能 |
|---|---|
| **S** Situation | 建立共識起點 |
| **C** Complication | 製造認知衝突 |
| **Q** Question | 引導思考方向 |
| **A** Answer | 提出你的解方 |

### E-3：信任三角（口說的 STP）

```
        ┌────────────┐
        │   真實性    │
        │ Authenticity│
        └──────┬─────┘
               │
    ┌──────────┼──────────┐
    │                     │
┌───┴──────┐      ┌──────┴───┐
│   能力    │      │   善意    │
│Competence │      │ Goodwill  │
└──────────┘      └──────────┘
```

> **Jacky 的洞察**：初次見面**善意 > 能力**；長期信任**能力 > 善意**。

### E-4 / E-5：敘事弧線 + 場景專屬架構

| 場景 | 推薦架構 |
|---|---|
| 電梯演說 | 四步法：我是誰 → 解決什麼 → 獨特優勢 → 下一步 |
| 策略簡報 | 金字塔 + SCQA |
| 企劃提報 | SOSTAC 口說版（六階段依序展開） |
| 競賽口報 | SCQA + AIDAS 節奏 |
| 客戶月報 | KPI 儀表板 + 行動標題 |
| 危機溝通 | 承認 → 行動 → 承諾 |
| 知識傳授 | SECI + 體驗式教學 |

## A — Arm（戰術裝備）

> **核心問題：你帶了什麼武器上場？**
> **SOSTAC 對應：Tactics — How exactly do we get there?**

### A-1：口說 7 武器（對應 7Ps）

| 武器 | 7P 對應 | 內容 |
|---|---|---|
| 🗣 **語言力** | Product | 動作標題、數據包裝、類比、故事 |
| 🎵 **聲音力** | Price | 語速、音量、停頓、音調 |
| 📊 **視覺力** | Place | 投影片、數據視覺化、手勢 |
| 🤝 **信任力** | Promotion | Cialdini 六原則、社會證明 |
| 👤 **身體力** | Physical Evidence | 眼神、開放手勢、身體前傾 |
| 🔄 **互動力** | Process | 提問、沈默、Q&A、即時調整 |
| 🤖 **AI 力** | People | AI 稿件、語音分析、模擬對練 |

### A-2：Cialdini 六原則的口說應用

| 原則 | 應用 |
|---|---|
| 互惠 | 先提供有價值的洞察，再提出請求 |
| 承諾一致 | 讓聽眾先同意小前提 |
| 社會認同 | 引用同產業成功案例 |
| 權威 | 展示數據來源公信力 |
| 好感 | 先建立人際連結 |
| 稀缺 | 強調時間窗口 |

### A-4：AI 武器庫

| 工具 | 功能 |
|---|---|
| Claude / ChatGPT | AI 輔助架構與初稿 |
| Yoodli | 冗餘詞、語速、語調分析 |
| Poised | 線上會議即時隱形提醒 |
| Hyperbound | AI 角色扮演高壓對話 |
| Speeko | 音調、語氣起伏訓練 |

## K — Kindle（點燃現場）

> **核心問題：上場了，怎麼把準備好的一切完美傳達？**
> **SOSTAC 對應：Action — The details: who, when, how?**

### K-1：開場 30 秒設計

| 開場類型 | 範例 |
|---|---|
| 數據衝擊 | 「過去 90 天，我們流失了 23% 的高價值客戶。」 |
| 反直覺提問 | 「如果我告訴你，你的最大競爭對手不是同業？」 |
| 故事開場 | 「上週我接到一通電話，是我們最大的客戶——」 |
| 結論先行 | 「我的建議是：立刻將 30% 預算轉投影音內容。」 |

> **Jacky 的禁區**：「永遠不要用『大家好，我今天要報告的是...』開場。**先丟一個他們不得不聽的東西，再自我介紹。**」

### K-2：AIDAS 節奏控制

| 階段 | 時間佔比 | 技巧 |
|:--:|:--:|---|
| **A** Attention | 10% | 數據衝擊、反直覺提問、故事 |
| **I** Interest | 20% | SCQA 中的 S 和 C |
| **D** Desire | 40% | 金字塔結構的核心論述 |
| **A** Action | 20% | 具體下一步、時程、預算 |
| **S** Satisfaction | 10% | Q&A、建立長期信任 |

### K-3：Q&A 應對框架

| 問題類型 | 應對策略 |
|---|---|
| 確認理解型 | 直接回答 + 補充 |
| 挑戰質疑型 | 先認同 + 再引導 |
| 超出範圍型 | 承認 + 承諾追蹤（48 小時內回覆） |
| 敵意攻擊型 | 不防守 + 拉回框架 |

### K-5：壓力下的即時口說

> 沒有準備時間時：

- **PREP 框架**：Point → Reason → Example → Point
- **30 秒法則**：深呼吸 3 秒 → 一個核心觀點 → 一個具體例子 → 一句話收尾
- **承認法**：「我需要再想想，但初步的觀點是——」**比胡說八道好一萬倍**。

## S — Sharpen（持續精進）

> **核心問題：如何用數據驅動口說能力的持續進化？**
> **SOSTAC 對應：Control — How do we monitor performance?**

### S-1：口說 KPI 儀表板

| KPI | 衡量方式 | 基準值 |
|---|---|:--:|
| 結構清晰度（核心訊息回憶率） | 聽眾 30 秒內能複述結論 | ≥ 70% |
| 信任建立（提案通過率／續約率） | 口說後實際決策結果 | 逐季提升 |
| 語言效率（冗餘詞比率） | AI 工具偵測（Yoodli） | < 3% |
| 時間控制 | 實際 vs 計畫 | ±10% |
| 互動品質 | 會後問卷／AI 情感分析 | ≥ 4/5 |
| 非語言一致性 | AI 語音分析 + 錄影回看 | 持續改善 |

### S-2：90 天行為矯正計畫

| 階段 | 天數 | 焦點 |
|---|:--:|---|
| 覺察期 | 1-30 | 用 AI 錄製並分析所有口說場景 |
| 矯正期 | 31-60 | 針對弱項密集練習，弱項 KPI 改善 ≥ 20% |
| 內化期 | 61-90 | 真實高壓場景應用，信任指標提升 |

### S-3：從「會說」到「會教」的進化（4 級）

```
Level 1：會說（Speaker）  — 能用 SPEAKS 完成各類口說任務
Level 2：會評（Evaluator）— 能用 SPEAKS KPI 評估他人口說
Level 3：會教（Coach）    — 能將口說知識系統化傳授
Level 4：會建（Architect）— 能將口說知識轉化為 AI Skill / Agent 永久傳承
```

### SECI 在口說領域

| 階段 | 實踐 |
|---|---|
| **社會化** | 觀摩高手提報、實戰簡報、師父帶上場 |
| **外顯化** | 將經驗寫成 SPEAKS 框架文件 |
| **組合化** | 整合多位高手技巧為教材、AI Skill |
| **內在化** | 反覆練習直到 SPEAKS 成為本能反應 |
