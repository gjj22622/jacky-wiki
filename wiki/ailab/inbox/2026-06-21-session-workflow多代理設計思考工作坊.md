## 最終做法（What Worked）

把「帶團隊腦力激盪」這種模糊需求，落成一個**確定性編排的多代理 Workflow**，而不是自己一個人想或開一堆平行 Agent 各說各話。兩次都成立（一次策略研究、一次設計思考工作坊），同一套骨架：

**三輪骨架（關鍵）**：
1. **發想（parallel）**：N 個「人格代理」各自從不同視角獨立發想，schema 強制結構化輸出（提案名／解決哪個痛點／怎麼運作／可行性）。
2. **交鋒（parallel）**：把第一輪全部提案彙成一張「白板字串」，**回餵給每個人格**，要它對別人的提案表態（挺／戰／加碼）＋說「綜合後我會改成…」。**這一輪是把「平行獨白」變成「真討論」的關鍵**——沒有它，產出只是一堆並排點子；有它，才有觀點碰撞、互相修正、收斂。
3. **綜合（single）**：一個「引導師／首席」代理拿前兩輪全文，寫成**保留具名發言與交鋒過程的會議紀錄**＋收斂的方向＋待決問題。

**人格設計的兩個高價值招**：
- **第一性原理人格（Elon 化身）**：prompt 灌「質疑每條需求、能刪就刪(the best part is no part)、能自動化別用人、敢碰硬體」。它的價值是**把『怎麼教老師傅用電腦』直接改寫成『把這需求刪掉』**——逼出別人想不到的解。
- **對抗性用戶代理（老師傅本尊）**：放一個「真實使用者本人」人格進團隊（62歲、不打字、怕被取代），任務是**把每個方案放他身上壓力測試「這我才不要用」**。它一票否決了「桌遊卡牌」「廣角相機」等看似聰明其實落地會死的方案，是**落地性的守門員**。沒有它，整桌專家會自嗨。

**操作要點**：
- 用 Workflow 工具（背景跑、完成自動通知），不要自己開一堆 Agent 手動接力。
- schema 強制每輪結構化（IDEA/REACT/MINUTES），合成階段才有乾淨素材、不用解析自由文。
- 守線（定價停拍板／匿名／30%揭露）寫成 GUARD 常數，每個 prompt 都帶，DRY。
- 人格 desc 要寫足「信念＋禁忌＋他會戰什麼」，人格才立得住、交鋒才有火花。

## 繞路紀錄（Detours）

- 一開始「策略研究」workflow 用 5 個發想視角＋3 人評審團打分，產出不錯但**偏靜態（各自打分）**；第二個「工作坊」改成**交鋒輪（互相回應）**，討論感與收斂品質明顯更高——以後要「會議紀錄」型產出，用交鋒輪不用打分輪。
- 想過自己在主對話直接開 5 個平行 Agent，但那樣**沒有確定性的「彙總→回餵→再收斂」控制流**，且要手動接力；Workflow 的 parallel + 字串彙總 + 下一輪餵入，才把多輪討論自動化。

## 錯誤與失敗（What Failed）

- **子代理 WebSearch 在 workflow 內多次回傳不可用**：策略研究那場，市場數字一度撈不到，靠權威來源(Wikipedia/官方)交叉確認、把不確定的降權標註才沒翻車。教訓＝research 型 workflow 要在 prompt 內**強制「查不到就誠實標查無、別編」**，並讓合成階段保留「資料限制聲明」。
- 第一版若不放「對抗性用戶代理」，專家團容易產出**聰明但使用者不買單**的方案（桌遊、感測手套、冷錄製箱）——這是設計思考最常見的死法，靠 user-proxy 人格擋下。

## 升格候選

- ⭐ **「多代理設計思考工作坊」模式（三輪：發想→交鋒→綜合｜含第一性原理人格＋對抗性用戶代理）** → 強烈建議升 `ailab/patterns/`，這是可跨專案複用的硬方法。
- **「交鋒輪 > 打分輪」當『要討論紀錄型產出』的選型心法** → 併入上條 pattern 或 `tools/` workflow 心法。
- **workflow 內 WebSearch 不可靠 → research prompt 要強制誠實標查無** → `tools/claude-code.md` 或 workflow 雷區補一條。
- 本 session 屬 Mode B 脈絡，升格時把上述抽成獨立 Mode A 再升，原檔留 inbox。
