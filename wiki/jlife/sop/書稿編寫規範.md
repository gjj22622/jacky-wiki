# 書稿編寫規範

> Codex 編寫書稿時應遵守的格式、檔案位置、Facebook 來源標記、author:pending 補充標記。
> 來源：`CLAUDE.md`、`docs/editor/02_自傳型回憶錄編輯_v0.2.md`

---

## 檔案位置

```
docs/manuscript/
├── volumes/        ← D 版正式書稿（5 卷）
├── volumes-e/      ← E 版職業身分版書稿（6 卷）
└── archive/        ← 舊版書稿存檔
```

> 書稿持續更新中。任何 AI 協作都應**先讀 `CLAUDE.md` 與 `docs/editor/` 編輯文件**，再動手修改書稿。

## 命名規則

| 版本 | 格式 | 範例 |
|---|---|---|
| D 版 | `{序號}_{卷名}_{版本}.md` | `01_卷一_人生一開始誰知道後面會這麼複雜_v0.2.md` |
| E 版 | `E{序號}_{卷名}_{版本}.md` | `E01_別人的場我的練功房_v0.1.md` |

**版本號**：v0.1 為初版 → v0.2, v0.3...（修訂遞增）→ v1.0（正式定稿）。

> ⚠️ D 版檔名數字開頭，E 版字母+數字開頭。所有解析檔名的正則必須用 `^[A-Za-z]?\d{1,2}` 兼容兩種。

## fb:YYYY-MM-DD 標記規範

> 為了讓好讀版閱讀器能將書稿段落與原始 Facebook 貼文**精確對應**，書稿中引用或改寫自 Facebook 貼文的段落，應在該段落附近加上 HTML 註解標記。

### 格式

```markdown
<!-- fb:2009-08-15 -->
這一週康是美父親節業績成長六倍...

<!-- fb:2013-12-31 -->
佳醫的 Jacky 謝幕，另一舞台的 Jacky 上台...
```

### 多篇貼文合併

同一段引用多篇貼文時用逗號分隔：

```markdown
<!-- fb:2009-08-15,2009-08-16 -->
```

### 用途

- 標記不影響 Markdown 渲染
- 閱讀器可以用它**自動拉出原文與圖片**
- 日期對應該篇 Facebook 貼文的發布日期

## author:pending 補充標記規範

> Facebook 匯出資料**不包含**：
>
> 1. 別人在你貼文底下的留言（**HTML 版完全沒有**；JSON 版只有自己留的）
> 2. 很多沒有寫在 Facebook 上的記憶
>
> 書稿中需要作者補充的段落，使用此標記。

### 格式

```markdown
<!-- author:pending {提問內容} -->
```

### 範例

```markdown
<!-- author:pending 請補充這則貼文的留言回應。你記得有哪些人留言？ -->
```

```markdown
<!-- author:pending 這場會議當時氣氛如何？對方主管說了什麼讓你決定離開？ -->
```

### 用途

| 用途 | 說明 |
|---|---|
| **提示作者** | 讓 Jacky 知道哪些地方需要他補充 Facebook 上沒有的記憶 |
| **引導回憶** | 每個標記都包含**具體提問**，幫助 Jacky 回想細節 |
| **追蹤進度** | 可以 grep 所有 `author:pending` 標記，檢視還有多少地方等待補充 |
| **完成標記** | Jacky 補充完成後，將 `author:pending` 改為 `author:done` |

### 適用場景

- 貼文底下的留言回應（Facebook 匯出不包含別人的留言）
- 沒有寫在 Facebook 上的重要事件
- 需要作者確認的細節（人名、時間、地點）
- 需要作者提供的內心感受或幕後故事

## 圖片引用規範

> 書稿中嵌入 Facebook 照片時，**必須使用 `/api/media/` 路徑**。

### 正確寫法

```markdown
![圖片說明](/api/media/your_facebook_activity/posts/media/{相簿目錄}/{檔名}.jpg)
```

### 錯誤寫法（不要用）

```markdown
![圖片說明](../../../data/facebook_exports/extracted/facebook-gjj22622-2006-3-9%20-%202026-3-9_01/...)
```

### 為什麼

> Facebook 匯出資料分散在 `_01` 和 `_02` 兩個目錄中。相對路徑無法確定檔案在哪一個目錄。**`/api/media/` 端點會自動搜尋所有匯出目錄**，保證能找到檔案。
>
> （2026-03-19 卷二照片無法顯示的根因。）

## 寫作指引（5 條編輯原則）

> 來源：`docs/editor/02_自傳型回憶錄編輯_v0.2.md`

1. **自傳重場景，不重金句密度** — 每章至少一個可記憶的場景（哪一年、在哪、有誰、發生什麼、怎麼想）
2. **自傳重人，不重概念** — 優先補人名、關係、職務、對話、互動
3. **不能只有反思沒有事件** — 先有事，再有想法
4. **保留原始語氣的證據感**，但不讓碎片感拖垮閱讀
5. **「當時的我」與「現在回頭看的我」兩個視角要持續交替**

### 每章基本組成（7 點）

1. 這一章在講什麼人生階段
2. 這段時間最重要的角色是誰
3. 發生了哪些代表性事件
4. 主角當時在忙什麼、相信什麼、卡在哪裡
5. 有沒有出現轉折點
6. 這段經驗後來造成什麼改變
7. 現在回頭看，這段的真正意義是什麼

### 卷級規模目標

每卷收斂至 **8-12 章**。每章必須有：具體時段、明確事件、可辨識人物、一條主要推進線。

### 要避免的問題

- 抽象自我定義堆疊（大刪「我是一個...的人」式段落）
- 重複反思（多章講同一個感悟要合併）
- 只有情緒沒有場景
- 整段照抄 Facebook 貼文
