# 備課品質 Gate

## 每個 Gate 的內層監察 LOOP

```text
工作 AI → 產出 → 監察 AI → PASS → 人工 Gate／下一階段
                  │
                  └→ RETURN（缺陷＋證據）→ 工作 AI 修正 → 再監察
```

### 角色分離

- 工作 AI：根據輸入與退件清單產出／修正。
- 監察 AI：只評、不改；逐條比對標準，輸出 `PASS` 或 `RETURN`、證據與缺陷位置。
- 人：決定目標、標準、例外、公開與最終交付。

### 監察輸出格式

| 欄位 | 要求 |
|---|---|
| Verdict | `PASS` 或 `RETURN` |
| Round | 第幾輪，預設上限 3 |
| Criteria | 每條標準的通過／失敗 |
| Evidence | 具體頁碼、段落、檔案或數值 |
| Defects | 只列未通過問題，按阻斷／重大／輕微分級 |
| Return target | 退回 A 行動、T 工具、S 策略、O 目標或 S 現況 |

監察 AI 不得直接重寫成品；否則評審與執行混在一起，無法確認缺陷是否真正被修正。

| Gate | 通過條件 | 失敗時回退 |
|---|---|---|
| G0 Intake | 日期、受眾、時長、場合、可檢查目標、限制已知 | 補資料或重寫 O |
| G1 Route | 主課型唯一；必需、選配、禁止模組明確 | 重判五維，不以主題猜課型 |
| G2 Retrieve | 1–3 前例及可借／不可借項；或明示無前例 | 擴查索引或採最近骨架 |
| G3 Design | 記憶錨點、時間表、互動、收束與交付定義一致 | 先修策略，不做投影片 |
| G4 Validate | 關鍵假設有證據、主辦確認或明示風險 | 補最低成本驗證 |
| G5 Build | 路由要求的交付齊；來源、版本、最終版清楚；中文微軟正黑體、英數 Arial | 修缺件，不擴張範圍 |
| G6 Rehearse | 時間、視覺、內容、權限、Demo、備援、隱私全過；可見文字皆 ≥16pt | A→T→S 逆推修正 |
| G7 Deliver | 偏差與現場證據已記錄 | 課後補記，不改寫事實 |
| G8 Review | 結果對 O、根因及下一輪改動有證據 | 區分感想與因果 |
| G9 Promote | 有兩場證據或一次高風險明確因果；已去重 | 留候選，不升格 |

## 共同紅線

- 演講與工作坊素材不得直接互套。
- 客戶、學員個資與未授權案例不得進公開教材。
- 重大改動後必須全程串讀；T-24 小時凍結非必要視覺改動。
- Demo 沒有導航與備援，不得視為通過。
- 自動生成檔若已被人工修改，重生前必須確認或另存新版本。
- 所有中文一律微軟正黑體；英文、數字、日期、網址與 Email 一律 Arial。
- 所有可見文字不得低於 16pt；一般內文以 20–24pt 為目標。
- 迴圈必須有至少兩個終止條件，且不得自行修改規則。

## 逆推除錯

從成本最低處倒查：`A 行動 → T 工具／環境 → S 策略 → O 目標 → S 現況真偽`。同層最多重試三次；沒有新證據不得原樣重跑。

## 終止條件

- `PASS`：進人工 Gate 或下一階段。
- 同一 Gate 最多 3 輪。
- 連續兩輪沒有改善：停止原路重跑，啟動逆推。
- 涉及目標改寫、公開發布、客戶承諾、個資或預算：強制人工決策。
